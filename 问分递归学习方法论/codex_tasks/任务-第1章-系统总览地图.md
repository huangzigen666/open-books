## 2. 本条目定位（一句话）

把「模拟一台计算机系统去学习」讲清楚成一张可勾选的**地图＋体检表**：七环节系统模型（输入→编译→存储→调度→执行→调试→迭代）＋其下显式命名的递归五机制脊柱，让读者**当场定位自己卡在哪一环**。**本章只做两件可验证的事——给一张架构图、给一张每环配可观察症状句的自诊断表——其余方法全部锚到各承接章，本章一概不展开。**

## 3. 必含结构（章节＝八段模板，见 CODEX 第三节）

> 本章是章节，套八段模板。但第 3、4 段在本章是「地图段」与「体检段」，**绝不是方法段**——写法见下方每段的硬约束。

1. **本章结论**（加粗一段，结论先行）：一句话点破——分数停滞往往不是不够努力/不够聪明，而是「学习这台机器从没被当成一个系统体检过」；本章不教任何一环怎么修，只帮你画出整台机器的图、并定位你卡在哪一环。核心动作：对照七环节自诊断表，勾出自己最红的那一环，然后翻到它负责的那一章。
2. **先拆穿一个迷思**：点破「学习好＝单点努力堆够了（多背、多刷、多熬）」这类「头痛医头」迷思——把学习当成一堆孤立动作，而不是一个**会自我调用的循环系统**；给一个可验证现象（如「每一环你都在花时间，分却不涨，因为你从没把它们当成一条会互相喂数据的流水线来看」）。迷思必须与「系统模型＋定位卡点」严格对应，不得耸动脱节。
3. **这个概念对应你学习里的哪个真实困境 → 给七环节系统模型 + 一张学习系统架构图**（三层落法的前两层在本章的形态）：
   - 先讲真实困境：学习的七件事（吸收新内容、把它变成考场能用、记住、分配时间精力、考场上调用、错了改、长期升级）从来是各管各的，没人告诉你它们是**一条流水线 + 一个会回流的循环**。
   - 再给隐喻命名：把这七件事命名为七环节（输入→编译→存储→调度→执行→调试→迭代），并显式说明「调试→迭代」会**回流**喂给下一轮（这就是递归，不是装饰），点明「这是一个会自我调用的循环」。
   - **一张学习系统架构图（文字/表均可，不画 mermaid 也行）**：用文字流程或一张表呈现「七环节顺序 + 调试/迭代回流箭头 + 每环一行『它管什么（一句话职责）』 + 它对应哪一章」。**职责列只写『这一环负责什么』，绝不写『怎么做好这一环』。**
   - 紧跟着显式命名**递归五机制脊柱**：拆解为子问题、base case、自引用、回溯与迭代、收敛判据。**每条只一句话，且这句话只做两件事：(a) 一句话说它是什么定位坐标，(b) 锚到负责章——「见第 N 章」。绝不解释怎么做。** 章归属硬性如下，不许写错：拆解为子问题→第2章；base case→第2章；自引用→第4章；回溯与迭代→第7章；收敛判据→第7章（考前承接）。统一术语只在此**锚定不展开**：base case＝能不查任何资料独立做对的最小单元；收敛＝连续 N 次独立做对／能讲给别人听懂。
4. **一个可上手的「体检」练习（三层落法第三层）＝七环节自诊断表**：这是本章唯一的「当天能做」动作——对照表给自己每一环打个红/黄/绿。**硬要求：表里每一环必须配一个「可观察症状句」，是「你最近是不是……」式的自测题，不是定义。** 示范句式（你按七环节各写一条，对应正确的承接章，不要照抄我的措辞、要写成你自己的、贴学习现场的）：
   - 输入环：「你最近是不是上课听着都懂、笔记记得很全，回头自己做却调不出来？」
   - 编译环：「你最近是不是一看就会、一做就废＝知识没编译成考场能执行的指令，见第3章。」
   - 存储环：「你最近是不是背的时候很顺、过两天默写就断＝存进去取不出来，见第5章。」
   - 调度环：「你最近是不是学一会儿就刷手机、多科横跳后回不到状态，见第6章。」
   - 执行环：「你最近是不是平时会做、考场上调不出来或时间不够，见第3/4章。」
   - 调试环：「你最近是不是同一类题反复错、错题本很厚分不涨，见第4章。」
   - 迭代环：「你最近是不是每周都在学、却说不出这周比上周修好了哪个 bug，见第7章。」
   表后给一句**唯一动作指令**：勾出你最红的那一环，翻到它对应的章——本章到此结束，不在这里教你修。并引用 [附录B·学习系统七环节自诊断表](../appendix/附录B_学习系统七环节自诊断表.md)（完整可勾版）与 [附录A·计算机概念学习动作对照总表](../appendix/附录A_计算机概念学习动作对照总表.md)（全书速查锚）。
5. **给家长的一段（话术微结构·写死）**：严格按「**这一环家长能观察到的信号（孩子学习时的具体表现）＋ 一句该说的话 ＋ 一句别说的话**」三件套写，全部具体到可照说。本章家长段聚焦「先帮孩子定位卡在哪一环，而不是笼统催」。
   - 信号示例（你细化）：孩子说「我每天都在学啊」但说不清今天修好了什么；一遇到分不涨就笼统归因「我太笨/不够努力」。
   - 该说的话（照说级）：例「先别急着加时间。咱们对着这七环，看你这周最卡的是哪一环——是听懂了做不出，还是背了忘，还是错题反复错？」
   - 别说的话（照说级）：例「你就是不够努力，再多刷两套就好了。」
   - **禁止出现「理解孩子／营造氛围／多鼓励／多沟通」类抽象词。**
6. **不泄密红线**：本章天然低危（只给公开框架地图＋体检表），但仍须写一小段守边界——明确：本章只帮你**自己**画图、自己粗定位卡在哪一环；至于「为什么偏偏卡在这一环、你这一科的精准根因、怎么从现在这档升到下一档、按周持续纠偏到收敛」一律不在本章（也不在本书免费层）展开，指向问分。章末写一句收口（措辞见第 4 节）。
7. **避免玄学的硬要求**：本章每个比喻是否落到了一个可勾选的体检项／一个当天动作？若只是「系统很重要」「要把自己当机器」却勾不动，删。
8. **本章自查**：`- [ ]` 复选框清单（沿用前两本格式，见第 6 节）。

## 4. 红线（写死本条目硬约束·违反即不通过）

**最高纪律（本章定义性约束）：本章是「体检表」不是「方法展览」。**

1. **禁止展开任何一个环节/机制的「怎么做」。** 七环节、递归五机制，本章一律只给「是什么 + 它负责什么 + 见第几章」，方法（base case 怎么找、错题怎么归因、缓存怎么清、注意力怎么调度、版本怎么迭代）**全部留给各承接章**。凡你写出任何一环的操作步骤、实操模板、判据细则、对照修法，即越界。
2. **删形容词测试（硬验收）：把本章所有形容词与修饰性句子删掉后，本章必须仍是一份能勾选的体检表**（一张架构图 + 一张每环配可观察症状句的自诊断表 + 五条锚点）。若删完只剩抒情、剩不下可勾选的项，即写空，重写。
3. **递归五机制每条只一句话，且只做「定位 + 锚章」**，不得解释做法；章归属不许写错（拆子问题/base case=第2章、自引用=第4章、回溯与迭代/收敛=第7章）。
4. **统一术语只锚定不展开**：base case＝能不查任何资料独立做对的最小单元；收敛＝连续 N 次独立做对／能讲给别人听懂。本章只引用这两句定义并指向第2/7章，**不在本章演示怎么用**。
5. **自诊断表每一环必须是「可观察症状句」（你最近是不是……式），不是定义、不是方法。** 缺任一环的症状句，或把症状句写成「应该怎么做」，即不通过。
6. **不泄密红线（CODEX 第四/七节）**：本章不给个性化精准诊断、不给精准根因、不给「如何升档」的升级路径、不给持续陪跑/完整执行系统；这些一律指向问分。**强自检**：写完回读——若读者照本章就能「无需任何外部诊断、自己跑出从定位到精准根因到持续升级的完整闭环」，就是写过头，收回到「框架 + 自查 + 找问分」层。章末显式收口一句：**「公开安全层到此·精准根因定位与持续个性化升级指向问分」**（本章是地图章，收口比第4章可略轻，但这句必须在）。
7. **三层落法 / 不生搬硬凑**：每个计算机概念必须先绑一个真实学习困境、再给隐喻、再落到一个可勾选体检项；落不到的删。**不许带寄存器/中断向量/指针这类落不到当天动作的二级术语**。
8. **不写玄学大词**：禁「成长型思维／潜能／聪明药／速成提分／玄学记忆术」；提分是方法跑通的结果，不是承诺。
9. **迷思不标题党**：开场迷思（「单点努力堆够了就行/学习不是一个系统」）必须与本章「系统模型＋定位卡点」严格对应，不得耸动脱节。
10. **不越界写规划/志愿**：本书只管平时怎么学；规划/志愿一句话交棒前两本即可，不展开（本章一般无需交棒，第11章统一交棒；若提及只一句话指向 `../../新疆高中三年升学规划书/README.md`、`../../新疆家长志愿风险识别手册/README.md`）。
11. **案例脱敏或合成**：禁可识别学生信息（真实成绩、学校、获奖）。
12. **双读者**：正文写给考生（能自己对照体检表上手）；章末给家长段按第 3 段-5 写死。
