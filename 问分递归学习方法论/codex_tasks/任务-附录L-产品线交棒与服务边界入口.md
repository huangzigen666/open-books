## 2. 本条目定位（一句话）

这是全书的**产品线交棒页 + 服务边界页 + 问分入口页**三合一。它做三件事：①给一张「我现在站在产品线哪一段」自检图（平时学习→规划→志愿，分别对应问分→第二本→第一本），让读者定位自己该往哪走；②讲清何时该从「自学」切到「找问分」；③把「书给公开框架与自学起步」和「个性化诊断/精准根因/持续陪跑/完整执行系统指向问分付费」划死，并留真实入口占位。**语境是「平时怎么学」——规划/志愿一律一句话交棒前两本，不在本页展开。**

## 3. 必含结构（附录＝卡片/表格页，**不套八段章节模板**；按此分节）

1. **本附录结论**（加粗一句，格式 `本附录结论：**……**`）：这套平时学习方法是问分线的起点；书帮你自己把系统跑起来（公开框架＋自学起步），但「精准诊断、按周纠偏、陪你迭代到收敛」是问分付费的活；规划与志愿在前方，交棒第二本与第一本。
2. **产品线自检图：我现在站在哪一段**（一张表/分段清单，三段对应三本书／问分）：
   - 平时学习（怎么学、错题反复错、背了就忘、时间花了分没涨）→ **本书 + 问分**（先用本书把系统跑起来，卡住找问分做个性化诊断与陪跑）。
   - 三年规划（选科、节奏、不可逆窗口、特殊招生早筛）→ **第二本《别等出分才开始》**，链接 `../../新疆高中三年升学规划书/README.md`。
   - 出分后志愿（冲稳保、章程核验、填报）→ **第一本《别把孩子的分数浪费在志愿表里》**，链接 `../../新疆家长志愿风险识别手册/README.md`。
   - 每段配一个**可观察定位句**（「你最近是不是……」式，如「你最近是不是同一类题反复错、错题本越抄越厚分还没动＝你在平时学习这一段，看本书」），不是干巴巴的定义。
3. **何时该找问分**（清单，给可自检的触发条件，不给升级路径）：
   - 自己照本书迭代了两三轮，仍卡在**同一类 bug**（同一类错题反复、同一环始终亮红）。
   - 自己做粗归因（四类标签）后，反复定位不到「为什么这一步会崩」的精准根因。
   - 需要把「我这一科该怎么从现在升到下一档」排成定制路径——**这条本书不给**。
   - 需要有人按周盯着、滚动纠偏、陪你迭代到收敛，而不是一次性自启动。
   - **写死提醒：这一节只列「什么时候该找」，严禁顺手写「找问分之后会怎么一步步带你升档」的路径，那是陪跑内核。**
4. **服务边界声明**（用「书给什么 / 问分付费给什么」对仗表，对齐前两本附录 M 体例）：
   - 书给：①可公开的方法框架（七环节系统模型＋递归五机制＋每章一个通用动作）；②自学起步（自己把系统跑起来的卡片与自查表）；③为什么（每个方法背后的认知原理）。

   - 每项对仗写「不交付」：书不交付针对你这一个孩子某一科的精准诊断与定制纠偏，问分不替你跑循环/替你思考。

## 4. 红线（写死本条目硬约束·违反即不通过）

- **不泄密红线（最高·全书口径）**：本页只给「公开框架＋自学起步＋为什么 + 何时找问分」。**严禁出现任何「找问分后如何一步步从某档升到下一档」的升级路径、按周纠偏机制、个性化诊断决策树、精准根因反推示例**——升级路径与持续陪跑正是问分付费内核。**强自检红线**：写完回读，若读者照本页就能「无需任何外部诊断完成精准诊断＋持续自我升级闭环」，就是写过头，收回到「框架＋何时找问分＋占位入口」层。
- **不编造咨询入口**：所有微信号、链接、二维码、账号名一律占位，写「待主编补齐真实入口」，禁编造。
- **不替代付费、不写玄学/不承诺结果**：本页本身就是把个性化导向问分；禁用「速成提分／押题保分／保上线／聪明药／玄学记忆术／百分百」等表达；提分是方法跑通的结果不是承诺。
- **统一术语复用**：涉及触底/停手处复用全书统一术语——base case＝能不查任何资料独立做对的最小单元；收敛判据＝连续 N 次独立做对／能讲给别人听懂；不另造同义词。
- **不越界写规划/志愿**：三年规划、志愿填报一律只做一句话交棒，指向第二本/第一本，不在本页展开任何规划/志愿方法。
- **不泄露产品壁垒**：不写问分内部的诊断量表、参数、评分逻辑、迭代算法、工作台字段结构。
- **交叉链接用规范文件名**（见 `../CODEX_执行指令.md` 第六之二节）；锚点用可读中文，不要把路径当锚点；交棒前两本用 `../../新疆高中三年升学规划书/README.md`、`../../新疆家长志愿风险识别手册/README.md`。
