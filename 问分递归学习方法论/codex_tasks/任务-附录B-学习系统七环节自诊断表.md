## 2. 本条目定位（一句话）

**一张全书入口体检表**：把七环节系统模型（输入 / 编译 / 存储 / 调度 / 执行 / 调试 / 迭代）做成可勾选的自测，每环配一个**可观察症状句**（「你最近是不是……」式，如「一看就会、一做就废＝编译环卡」）＋ **绿 / 黄 / 红三档自评** ＋ **对应去读哪一章**。读者用它在 5 分钟内定位「我最该先修哪一环」，然后跳到对应章。**它只做定位与分流，不讲任何方法。**

## 3. 必含结构（附录＝表页，不套八段）

按以下顺序写，全部用短句、清单、Markdown 表格：

1. **标题**：一级标题 `# 附录 B：学习系统七环节自诊断表`。
2. **结论先行的用途说明（2–4 句）**：这是一张**入口体检表**，不是方法清单；它只回答「你现在最该先修哪一环」，方法在对应章里。一句话给使用方式：照每一环的症状句对号入座，给自己打绿/黄/红，红的那一环就是本周入口。**点明它配合第1章使用**（回链用规范名 `../chapters/01_系统总览地图.md`）。
3. **三档自评图例（写死，先于主表）**：用一小段或小表把绿/黄/红定义成**可观察的判据**，不写形容词空话。建议口径（可微调用词但必须是可观察的）：
   - 🟢 绿＝这一环我最近基本没卡，症状句基本不成立。
   - 🟡 黄＝偶尔卡，症状句一半成立。
   - 🔴 红＝经常卡，症状句一看就中——这就是你本周的入口环。
4. **七环节自诊断主表（核心，一张 Markdown 表）**：**每一行＝一个环节**，至少五列：`环节(计算机名)` | `它管学习里的哪件事(一句话)` | `可观察症状句（你最近是不是……）` | `我的自评(🟢/🟡/🔴)` | `去读哪一章`。**症状句是本表的命脉**：必须是**可勾选的自测题**（「你最近是不是……」式），不是定义、不是方法。七环逐行写死，症状句参考（可润色但语义对齐，且必须可观察）：

   | 环节 | 管的事 | 症状句（你最近是不是……） | 去读 |
   |---|---|---|---|
   | 输入 | 你怎么接收新知识 | 课听了、书看了，却说不清今天到底学了哪几条？ | 第1章 / 配第2章 |
   | 编译 | 学过 → 会用 | 一看就会、一做就废，例题懂了一动笔就卡？ | 第3章 |
   | 存储 | 记得住 → 取得出 | 背的时候顺，过几天默写就空，考场上想不起来？ | 第5章 |
   | 调度 | 时间和注意力怎么分配 | 学一会儿就刷手机，几科横跳后回不到状态？ | 第6章 |
   | 执行 | 考场上真把题跑完 | 平时会做，一到限时考试就来不及、跑不完？ | 第6章 / 配第3章 |
   | 调试 | 错题怎么处理 | 错题抄得很整齐，同类题还是反复错，说不清错在哪？ | 第4章 |
   | 迭代 | 这周比上周强在哪 | 复盘只剩一句「下次认真点」，下周又踩同一个坑？ | 第7章 |

   注：「去读哪一章」一列用规范名链接（见第5节文件名表），锚点用可读中文。`执行`环可挂在第6章并配第3章；不要为「输入」「执行」硬造一个本书没写的章，挂到最贴近的现有章即可，**以本表实际语义为准、与第1章七环命名保持一致**。
5. **怎么用这张表（3–5 条短动作，只做分流不给方法）**：例：①先全表打完绿/黄/红；②**只挑一个最红的环**当本周入口，别七环一起改；③翻到那一环「去读哪一章」对应的章，按那一章的当天动作做起；④一两周后回到本表重打一次分，看红的有没有转黄。**严禁在这里写任何具体修法**（修法全在各章），这里只给「怎么用表」。
6. **给家长的一段（话术微结构·写死）**：严格按「**家长能观察到的信号（具体表现）＋ 一句该说的话 ＋ 一句别说的话**」三件套写，具体到可照说。信号扣住「家长不必懂学科，只需看孩子能不能自己指出卡在哪一环」。例：该说的话——「这张表你打完了吗，最红的是哪一环？」；别说的话——「你就是不努力，每一项都得管起来。」**禁「理解孩子/营造氛围/多鼓励/多沟通」类抽象词。**
7. **边界标注（写死一行）**：本表只做**入口定位与分流**，不给诊断结论；**它告诉你卡在哪一环，不告诉你为什么卡、怎么从这一环升上去**——精准定位某科某状态的真实根因与持续升级，指向问分。结尾写死一行：**「公开安全层到此·本表只做粗定位；精准根因诊断与持续纠偏指向问分。」**

## 4. 红线（写死本条目硬约束，违反即不通过）

> 下列把全书全局口径里**与本条目相关**的几条焊进来，逐条照做。

1. **每环必须是可勾选的「症状自测题」，不是定义**（最高优先）：每一环那一格写的是「你最近是不是……」式的**可观察现象**，读者能直接打勾对号入座。**禁止把症状句写成名词解释 / 概念定义 / 方法步骤**（如禁「编译就是把理解转成可执行」这种定义句；要写成「一看就会、一做就废？」这种自测句）。
2. **只做定位、不给方法**：本表的职责是把读者分流到对应章。**任何具体怎么做（拆解法、提取练习、错题四分法、调度表、更新日志……）一律不写在本表**，只写「去读哪一章」。强自检：若读者照本表就能在不翻任何章节的情况下知道「具体该怎么修」，就是越界，收回到「症状句＋分档＋去哪章」。
3. **删形容词后仍是体检表**：删掉所有修饰性形容词后，本表必须**仍是一份能勾选的表**（七行环节 × 症状句 × 绿黄红 × 去哪章俱在）。写完做一次「删形容词」自检。
4. **不泄密红线（全书最高）**：本表是公开框架层的入口工具——只给「七环 + 症状句 + 自评分档 + 分流到章」。**不写**：个性化精准诊断、精准根因、「如何从某一环/某一档升到下一环/下一档」的升级路径、持续陪跑机制、完整自动跑的执行系统。这些一律指向问分。**结尾必须有第3.7条那行「公开安全层到此·…指向问分」标注。**
5. **统一术语复用（出现即对齐）**：若本表说明文字里碰到 base case，用全书统一口径＝「能不查任何资料独立做对的最小单元」；碰到收敛，用「连续 N 次独立做对／能讲给别人听懂」。**但本表以「定位分流」为主，不要硬塞这些术语**——能不写就不写，写了必须对齐口径，且不展开方法。
6. **不写玄学 / 结论先行 / 不标题党**：禁「成长型思维 / 潜能 / 聪明药 / 速成提分 / 玄学记忆术」等大词；用途说明结论先行；症状句必须与它分流到的那一章方法严格对应（如「一看就会一做就废」对应第3章编译，不许张冠李戴）。提分是结果不是承诺。
7. **双读者 + 家长段写死**：本表正文写给考生（读者自己能打分对照）；家长段严格按「信号＋该说的话＋别说的话」微结构，禁抽象词（见 3.6）。
8. **不越界写规划/志愿**：本书只管平时怎么学；本表不出现三年规划、志愿填报内容；如确需提及前两本，仅一句话交棒，用 `../../新疆高中三年升学规划书/README.md`、`../../新疆家长志愿风险识别手册/README.md`（本附录通常无需交棒，能不写就不写）。
9. **案例脱敏**：症状句用通用学习场景，不得出现可识别学生信息（真实成绩/学校/获奖）。
