## 2. 本条目定位

- 类型：**正文章节**，套八段模板（见 CODEX 第三节）。
- 标题（第一行一级标题，照抄）：`# 第二章：递归调用——把一道不会的题，拆到你会做的那一层为止`
- 它在书里的角色：**全书递归脊柱的根章**。「能自己做对的最小一步」这个概念由本章定义并给硬判据；第3章「不懂」类报错、第4章自引用、第7章回溯迭代都回指本章。本章给的是人人能套的「往下拆、再往回拼」的通用方法，写到读者照着能自己拆通一道题。
- 三层落法（必须三层都落齐，缺一层即不合格）：
  - **真实困境**：一道综合题/大题，看例题时秒懂，自己一动笔就卡在第一步、不知从哪下手；一卡就整道放弃，或者直接抄答案；抄完这道，换一道同类还是不会。投入了「刷」和「抄」的时间，却从没把真正卡住的那一小步单独拆出来练过。
  - **隐喻（递归调用）**：把一道不会的大题，拆成几个更小、你已经会的子问题；一路往下拆，拆到「不查任何资料、自己就能独立做对的最小一步」；触底之后，再顺着自己的思路一层层往回拼回整道题。这就是递归——不是一口吞下整道题，是拆到你能下口的那一步，再搭回去。
  - **当天动作**：挑一道你今天真不会的题（只挑一道），逐层问「这一步，我卡在哪个更小的、我已经会的东西上」，一直拆到能自己做对的最小一步，把它写下来，再往回拼一遍。配[附录C·递归拆解卡](../appendix/附录C_递归拆解卡.md)当天套用。
- 统一术语强制复用（**本章是这两个词的定义源头，要给足大白话**）：
  - **「能自己做对的最小一步」**：本章主词用这句大白话；指「不查任何资料、不看答案，自己就能独立做对（独立写出）的那一步」。英文叫 base case，**只在本章首次出现时括注一次**（如「（英文叫 base case，后面就用大白话）」），之后一律用大白话，不再出现英文。
  - **触底硬判据**：能不查任何资料、独立做对（独立写出）的那一步＝拆到底。还要瞄答案、还讲不出为什么这步成立＝没到底，继续往下拆。
  - 若提及「练到什么程度算够」，用全书统一的**收敛判据＝连续 N 次独立做对／能讲给别人听懂**，一句话带过、锚到第7章，不展开。

## 3. 必含结构（章节＝八段模板，按 CODEX 第三节逐段写；段标题用 `##`）

1. **本章结论（加粗一段，结论先行）**：第一行加粗写出——拆穿「刷题越多越好」；建起「把一道不会的题往下拆到能自己做对的最小一步、再往回拼」的能力；一句话核心动作（挑一道不会的题，逐层拆到能自己做对的最小一步，写下来再往回搭；配附录C）。

2. **先拆穿一个迷思**：1-2 句点破「刷题越多越好」。点出真相：遇到不会就狂刷/抄答案，刷的多半是「你已经会的」，真正卡住的那一小步从没被单独拎出来练过，所以刷一百道、那一步还是过不去。给可验证现象：看例题秒懂、自己动笔卡在第一步；抄完这道答案，换个数字/换道同类又不会。迷思与本章方法严格咬合，不标题党。

3. **这个概念对应你学习里的哪个真实困境**（三层落法前两层）：先讲困境（大题不知从哪下手、一卡就整道放弃或抄答案、抄完下次还不会），再给「递归调用」隐喻命名。讲清：解一道不会的题，不是硬憋整道，是把它拆成更小、你已经会的子问题，一路拆到「能自己做对的最小一步」，再往回拼。把「我不会这道题」重命名为「我还没把它拆到我会的那一步」。

4. **怎么算「拆到底」——能自己做对的最小一步（硬判据）**：给触底的硬判据，可自测、不许是形容词：
   - 到底＝不查任何资料、不看答案，这一步你能独立做对（独立写出）。
   - 没到底的信号：还要瞄一眼答案才敢写、还讲不出这一步为什么成立——就继续往下拆。
   - 点明：拆到底不是拆到「最简单」，是拆到「你自己确实能独立做对」的那一层；每个人的那一层可能不同，这正常。
   - 在此首次给出「能自己做对的最小一步（英文叫 base case，后面就用大白话）」的括注。

5. **一题拆到底·分层示范（实操模板）**：拿**一道合成的综合题**（自造、脱敏，别用真实题目/学生），具体示范怎么逐层往下拆、再往回拼。建议用一条看得见的「拆解链」展示：整道题 → 要用到哪几步 → 哪一步卡住 → 这一步又卡在哪个更小的、我已经会的东西上 → 一直拆到能自己做对的最小一步 → 触底 → 再一层层往回拼回整道题。示范要具体到能照着仿做，但**只演示「怎么拆/怎么判断触底/怎么往回拼」这套通用方法本身**，把方法给透。

6. **当天动作 + 空白拆解卡（三层落法第三层）**：给当天就能做的动作 + 引到附录C：
   - 今天挑一道你真不会的题（只挑一道，别挑十道）。
   - 写下整题，逐层问「这一步我卡在哪个更小的、我已经会的东西上」，往下拆。
   - 拆到能自己做对的最小一步为止，把那一步单独写出来、确认能不查资料独立做对。
   - 再顺着往回拼一遍，看整道题能不能接上。
   - 引用[附录C·递归拆解卡](../appendix/附录C_递归拆解卡.md)：一张把题逐层拆到「能自己做对的最小一步」的空白模板，卡只给框架与空格、不替你下结论。
   - 显式承接递归：把拆出来的那个最小卡点喂进练习、带进下一轮（呼应第4章的「错题喂下一轮」、第7章迭代），一句话带过即可，不展开。

7. **给家长的一段（话术微结构·写死，对齐 04 章第6段）**：严格写成三件套，全部具体到可照说，禁「理解孩子/营造氛围/多鼓励/多沟通」类抽象词：
   - **这一环家长能观察到的信号**：孩子一遇到大题/难题就卡住不动、半天不下笔；要么直接抄答案，要么一句「不会做」就整道跳过；同类题抄过一次，下次还是不会。
   - **一句该说的话**：例如「先别抄答案——这道题里，哪一小步是你已经会的？从你会的那步往上，一点点往下拆看看。」
   - **一句别说的话**：例如「这么简单都不会？多刷几道不就会了。」
   - 补一句：家长不用会做这道题，只盯一个动作——孩子有没有把题往下拆、找到自己会的那一步，而不是整道放弃或直接抄答案。

8. **本章自查（结尾自然收口 + 一句软服务邀请，不开防御节）**：
   - 先用一两句自然收束：这套「往下拆到能自己做对的最小一步、再往回拼」的方法，书里都给透了，你一个人就能照着拆。
   - **全章唯一一处软服务邀请**（自然一句，邀请口气，不划界）：例如「真到自己反复拆不下去、总卡在同一类题上，问分会盯着你这一科一直往下挖、按周陪你调节奏。」——这是长期陪着一个人才做得了的事，跟看书自学是两码事。
