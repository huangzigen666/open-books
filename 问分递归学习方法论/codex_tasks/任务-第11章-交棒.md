## 2. 本条目定位（一句话）

带读者把七环节（输入→编译→存储→调度→执行→调试→迭代）在**自选的一科上跑通一圈**，亲手摸到「这套方法是一个会自我调用的递归循环」；然后给一张「我现在站在产品线哪一段」自检图（**平时学习→规划→志愿**），说明这套**平时学习方法是问分线的起点**，往后接第二本《别等出分才开始》（三年规划）与第一本《别把孩子的分数浪费在志愿表里》（志愿）。**本书只把「平时怎么学」讲透；规划与志愿只一句话交棒、绝不展开。**

## 3. 必含结构（章节＝八段模板，按 CODEX 第三节）

> 本章是「收口/交棒」章，但仍**套八段模板**，不是附录卡片页。第 2 段的「迷思」要选一个与交棒方法严格对应的——**建议迷思：「方法学完一遍就算结束了 / 看完这本书就到头了」**（对应本章「把方法真正跑通一圈、再交棒去下一段」，不得耸动脱节）。

1. **本章结论**（加粗一段，结论先行）：拆穿「看完方法就算学会了」；本章建起的能力＝把七环节在一科上**真跑通一圈**、并认清自己在产品线的位置；一句话核心动作（**今天选一科，把输入→编译→存储→调度→执行→调试→迭代走一遍，留下痕迹**）。
2. **先拆穿一个迷思**：点破「方法读懂了＝方法跑通了 / 这本书读完就到头了」，给一个可验证现象（如「七环节每章都点头，真上手选一科跑，卡在第二环就停了」）。迷思与本章方法严格对应，不标题党。
3. **这个概念对应你学习里的哪个真实困境**：先讲困境（方法零散记了一堆、从没在一科上连起来跑过整圈，所以分还是不动），再给隐喻命名——**前十章是装好的各个模块，本章是第一次「整机通电跑一遍」**（completed run / 一次完整运行）。显式点出这一圈本身就是递归：这一轮的输出（错题、更新日志、收敛判据）就是下一轮的输入。
4. **一个可上手的递归练习**（本章两件干货，都要落到当天可做）：
   - ① **「选一科跑通一圈」清单**：给一张七环节走一遍的可填表，每环只写**一个当天能留下的痕迹**（动词开头），如：输入→拆出一个 base case；编译→合书复述一条；存储→默写一次；调度→一个番茄钟划正字；执行→独立做一题；调试→给一道错题贴粗标签；迭代→写一条更新日志。**复用全书统一术语**：base case＝能不查任何资料独立做对的最小单元；收敛判据＝连续 N 次独立做对／能讲给别人听懂。各环用 CODEX 第六之二节**规范章节名**一句话回指（如「拆 base case 见第2章」「贴粗标签见第4章」），**不展开方法**。
   - ② **「我站在产品线哪一段」自检图**：一张三段图——**平时学习（本书）→ 三年规划（第二本）→ 志愿填报（第一本）**，每段一句「这一段在解决什么」+ 一句「我现在是不是在这一段」自测句。**规划/志愿两段只各一句话定位，绝不展开方法。**
   - 引用 [附录L·产品线交棒与服务边界入口](../appendix/附录L_产品线交棒与服务边界入口.md)（边界页，只给自检图与入口，不给个性化结论）。
   - **显式链接前两本 README**：第二本 `../../新疆高中三年升学规划书/README.md`、第一本 `../../新疆家长志愿风险识别手册/README.md`（锚点用可读中文，不要把路径当锚点）。
5. **给家长的一段**（话术微结构·写死）：严格按「**这一环家长能观察到的信号（孩子在「跑通一圈/面对下一段」时的具体表现）＋ 一句该说的话 ＋ 一句别说的话**」三件套，具体到可照说。**禁「理解孩子／营造氛围／多鼓励／多沟通」类抽象词。**（信号示例：孩子方法听了一堆但从没在一科上完整跑过、一提规划/志愿就焦虑想跳过平时学习；该说示例：「先把这一科按七环节跑一圈，规划和志愿后面有专门的书，不用现在慌」；别说示例：「这些方法先别管了，赶紧想想报哪个学校、选哪个专业」。）
6. **不泄密红线**：本章只给「自己起步把系统跑起来」的公开框架 + 一句「**何时该找问分**」。**何时找问分写死成可判定的触发条件**：自己照本书把一圈跑通、**独立迭代两三轮后仍卡在同一类 bug**、需要**个性化精准诊断/精准根因/如何从当前档升到下一档的升级路径/持续按周陪跑到收敛**时 → 指向问分。**严禁**在本章给出「如何升档」的路径、持续纠偏机制、完整自动跑的执行系统（升级路径＝陪跑内核，按 CODEX 第四节「第7、8、11 章只给自助模板不给陪跑机制」执行）。章末显式标注一句：**「公开安全层到此·持续诊断、纠偏、陪跑与完整执行系统指向问分」。**
7. **避免玄学的硬要求**：本章每个比喻（整机通电/完整运行/产品线地图）必须落到一个真实动作或一张可填表，落不到就删；不写「速成提分/玄学」大词，提分是方法跑通的结果不是承诺。
8. **本章自查**：`- [ ]` 复选框清单（沿用前两本格式）。

## 4. 红线（写死本条目硬约束）

1. **不泄密红线（最高）**：本书只给「公开方法框架 + 自学起步 + 为什么」。本章对「**如何从某一档升到下一档的升级路径**」「持续监控/按周纠偏」「陪你迭代到收敛」「完整自动跑的执行系统」一律**不写**，全部指向问分。**强自检红线**：写完回读——若学生照本章就能**无需任何外部诊断、自己完成持续自我升级闭环**，就是写过头，必须收回到「框架 + 自助模板 + 一句话找问分」层。何时找问分写成可判定触发条件（迭代两三轮仍卡同一类 bug、需个性化诊断/升档路径/持续陪跑）。
2. **边界·不越界写规划/志愿**：本书只管「平时怎么学」。三年规划、志愿填报**只各一句话交棒**，指向第二本/第一本，**不在本章展开任何规划/志愿方法**（不谈选科策略、不谈冲稳保、不谈批次/位次/平行志愿）。
3. **三层落法**：「完整运行」「产品线地图」必须①绑真实困境（方法零散没跑通整圈／不知道平时学习之后是什么）②给隐喻（整机通电／产品线地图）③落到当天动作（七环节走一遍的可填表＋产品线自检句）。落不到动作的比喻删。
4. **统一术语强制复用**：base case＝能不查任何资料独立做对的最小单元；收敛判据＝连续 N 次独立做对／能讲给别人听懂。各环回指其它章节一律用 CODEX 第六之二节**规范章节名**，禁自造简写。
5. **双读者**：正文写给考生；章末「给家长段」写死成「信号＋该说＋别说」话术微结构，禁抽象词。
6. **结论先行加粗；迷思与方法严格对应不标题党；不写玄学大词；提分是结果不是承诺；案例脱敏或合成。**
7. **交叉链接防断链**：引用附录 L 用规范名 `../appendix/附录L_产品线交棒与服务边界入口.md`；交棒前两本用 `../../新疆高中三年升学规划书/README.md`、`../../新疆家长志愿风险识别手册/README.md`；锚点用可读中文。
