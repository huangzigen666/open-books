# 目录设计（第三本）

> ⚠ 口径更新（2026-06-29，最高优先）：本文档中第4/7/8章、附录E/I 等处的「不泄密硬上限 / 公开安全层到此 / 五阶段禁升档路径 / 按周纠偏指向问分」均为**旧口径，已被 `CODEX_执行指令.md`「一之二节②」覆盖**。最终模型＝**方法层完全写透，问分只卖服务（针对个人的持续诊断＋按周陪跑），章末仅一处软性服务邀请、无防御式标注**。读本文档定位时，凡涉及不泄密/边界的旧表述，一律以 CODEX 一之二节为准。

主线：以「计算机系统」为骨架（七环节：输入/编译/存储/调度/执行/调试/迭代），以「递归五机制」为显式脊柱，每章取一个系统概念，先拆穿一个主流学习迷思，再给递归解法，最后落到一个当天就能上手的真实学习动作或一张卡。每章只回答两个问题——**这章拆穿什么迷思 / 建起什么能力** + **一个能立刻上手的递归动作**——最后把读者交棒给第二本。全书自身就是一次「学习如何学习」的递归示范。

## 完整目录

### 前言：你不是内存不够，是没人教你怎么给自己编程
立框架：分数停滞往往不是「不够努力/不够聪明」，而是学习这台机器从没被调试过；声明四条边界（不写玄学/不承诺押题保分/AI 只做辅助不替你思考/个性化诊断与陪跑指向问分），预告交棒第二本。先点破最大的隐形 bug：把「投入时间」当成「学会」的死循环。**给家长一条独立最短阅读路径：你可以只读前言＋第9章＋各章末「给家长段」。**
- 真干货：用「假勤奋＝程序一直在跑但没产出」这个可验证现象开场，给一个 5 分钟自测（你上周学的东西现在能默写几条），让读者当场看到自己的 bug，而非讲「要相信努力」。

### 第1章：把自己当成一台计算机——递归学习的总览地图
全书脊柱与指南针，**但本章是「体检表」不是「方法展览」**。讲清「模拟计算机系统去学习」是什么：七环节系统模型（输入→编译→存储→调度→执行→调试→迭代）＋其下显式命名的递归五机制脊柱，点明这是一个会自我调用的递归循环。**强制纪律：本章不准展开任何一个环节/机制的具体方法（方法全部留各章），只做两件可验证的事。**
- 真干货：①一张可勾选的「七环节自诊断表」，每环必须配一个**可观察症状句**（不是定义，是「你最近是不是……」式自测题，如「你最近是不是一看就会、一做就废＝编译环卡」）；②递归五机制每条只在它后续负责的章节锚点处一句话带过（如「base case 见第2章」），本章不解释怎么做。验收标准：删掉所有形容词后，本章是否仍是一份能勾选的体检表。

### 第2章：递归调用——把一道不会的题，拆到你会做的那一层为止
拆穿迷思「刷题越多越好」。映射递归与 base case：遇到不会的不是抄答案，而是不断问「这一步我卡在哪个更小的、我已经会的东西上」，拆到触底再往回拼。
- 真干货：给「一题拆到底」实操模板，示范把一道综合题逐层拆成「我已经会的最小单元」，给空白拆解卡当天套用；明确 base case 硬判据（**能不查任何资料独立写出＝触底**）。

### 第3章：编译——学过 ≠ 会用，知识要先翻译成考场能执行的指令
拆穿迷思「听懂了/看会了就等于会了」。映射编译/源码 vs 可执行：老师讲的是源代码，考场上能调用的才是可执行文件。教把被动理解「编译」成主动可执行。
- 真干货：给「编译通过」硬判据清单（合书能复述/能讲给别人/能独立做出变式），给「编译报错对照表」，把「一看就会一做就废」当成具体报错信息来定位，而非笼统说「要理解」。

### 第4章：调试——错题不是用来罚抄的，是用来定位 bug 的（全书最锋利钩子·最高危泄密点）
拆穿迷思「错了就改对答案/订正＝抄一遍」。把错题从罚抄升级为「定位这次思维在哪一步崩的」，并显式承接递归的**自引用**机制：错题不是用来改的，是用来喂下一轮的。按 bug 类型分类而非题型分类。
- 真干货：给「错题归因四分法卡」（看错/不懂/记错/没时间），**每类只配「一句话识别问句＋一句通用原则」**（如记错→去查它该长什么样再重背），附一个错题调试示例——**示例只演示归因到四类标签为止**。
- **不泄密硬上限（对齐第一本「采集卡只给空白框架不给结论」）：严禁出现「如果是 X 且 Y 则用 Z 修法」的二级修法分支；严禁完整反推根因示例；精准根因（为什么这一步会崩）显式断在此处。本章末显式标注「公开安全层到此·精准根因定位与定制纠偏指向问分」。**

### 第5章：内存与缓存——为什么背了就忘，以及怎么让知识可被提取
拆穿迷思「死记硬背最保险/背得多就考得好」。映射内存/缓存/读写：「存进去」和「取得出」是两件事，考场失分常是提取失败而非没存。教主动提取＋间隔重复，把知识搬进考场能秒取的缓存，并清掉脏缓存（记错的、过时的）。
- 真干货：给可执行的提取练习设计法（自测优先于重读、间隔节奏起步参数），给「脏缓存自查」动作（定期默写核心概念、标记记错的清理掉）；用「重读手感顺≠记得住」这个反直觉点纠正最常见误区。

### 第6章：进程调度——时间和注意力是 CPU，别让它一直在切换中空转
拆穿迷思「全都想抓/多学就是好」。**真实困境先行**：学一会儿就刷手机、多科横跳后回不到状态。映射进程调度/上下文切换：频繁切换的真正代价是上下文切换开销。教单任务时间块、优先级调度（先修高频失分项）。
- 真干货：**把比喻落到一个糙而可做的动作——「一个番茄钟内被自己中断几次，划一笔正字」**（替代偏实验室、学生不会真做的「记录被打断后多久回到状态」）＋一张按「失分频率×可提升空间」排优先级的调度表；明确这是排序方法不是排满时间表。**若 CPU 比喻落不到这个划正字动作，砍掉比喻直接讲「一次只做一件事＋先修最常错的」。**

### 第7章：版本迭代——每周给自己发一个新版本，复盘是写更新日志
拆穿迷思「复盘＝反思一下/感动自己」。显式承接递归的**回溯与收敛**：把复盘升级为「发版本」，回退到最近做对处带修正重跑，每周记录这版改了什么、修了哪个 bug、下版试什么，最终收敛到稳定可用的个人方法。
- 真干货：给「每周更新日志模板」（本版改动/修复的 bug/遗留问题/下版实验一项）＋「收敛判据」（**连续 N 次独立做对／能讲给别人听懂＝该停，别刷到考试为止**）＋「一次只改一个变量」的验证法，避免复盘沦为情绪宣泄。
- **护栏：只给自助模板；按周持续纠偏、陪你迭代到收敛是问分的活，本章一句话指向问分。**

### 第8章：自举与元认知——学习如何学习，让系统自己升级自己（次高危泄密点）
全书递归的收口与最高层。**迷思改为「学会了就不用管怎么学了」**（直接对应元认知/学习如何学习，比「越聪明越好」与方法咬合更紧）。映射自引用/自我编程/自举：把前面所有方法本身当成可被同样方法优化的对象，教学生监控自己的学习过程。给五阶段刻度（模仿/调试/重构/自我编程/进化）让读者自测在哪一档。
- 真干货：给「元认知三问」当堂练习（我在系统哪一环/这方法这周有效吗/下一步我自己会怎么调），三问必须配**当堂可填的具体对象**（本周哪个方法、哪道题），不许停在「监控自己的学习」这类元话术；用一个「学生自己发现一个学习 bug」的合成示例。
- **不泄密硬上限：五阶段刻度只作「自测我在哪档」的静态标尺（每档一个识别信号），严禁写「如何从这档升到下一档」的任何路径化指引（升级路径＝问分陪跑内核）；元认知三问只做自检问句、不配「问出答案后该怎么调」的决策树；合成示例只演示「发现一个 bug」，持续按周纠偏到收敛显式断开。本章末显式标注「公开安全层到此·持续个性化升级与陪跑指向问分」。**

### 第9章：给家长的运维手册——你是供电和散热，不是超频
双读者落地章，整章写给家长。把前八章的系统隐喻翻译给家长：孩子是那台机器，家长的角色是稳定供电（情绪/后勤/信任）与散热（降焦虑），而不是超频（加压加时长）或替 CPU 运算（代替思考）。
- 真干货：给「家长该做/不该做对照表」＋「话术替换」（把「你怎么又错」换成「这道题 bug 定位到哪一步了」），**话术替换覆盖到每一环而非只给一两句样板**；给一个「家庭运维例会」轻量模板；明确家长不需要懂学科，只需维护系统运行环境，个性化诊断交给孩子自己＋问分。

### 第10章：AI 是你的副驾，不是你的大脑
讲 AI 在递归学习里的正确位置：AI 可做整理/查漏/讲解/生成练习/陪练复现 bug，但不能替你跑循环、替你思考、替你做诊断决策。沿用第一本「AI 降级、不拍板」原则。
- 真干货：给「安全提示词 vs 危险提示词」对照（可直接复制的提问范式）；明确 AI 能做调试辅助，但「个性化精准诊断与持续陪跑」指向问分，划清边界。

### 第11章：交棒——方法跑通之后，规划与志愿在前方等你
产品线衔接与收口章。带读者把前面所有环节串成一次完整运行（选一科跑通一圈），说明这套平时学习方法是问分线的起点，往后接第二本（三年规划）与第一本（志愿）；划清边界：本书给「自己起步把系统跑起来」的公开框架，持续诊断、纠偏、陪跑、完整执行系统指向问分。
- 真干货：给一张「我现在站在产品线哪一段」自检图（平时学习→规划→志愿）＋一句话指明何时该找问分（自己迭代两三轮仍卡在同一类 bug、需要个性化诊断时），与前两本「交棒章」体例一致。

- 附录 E：错题调试归因四分法卡（看错/不懂/记错/没时间×对应**通用原则**，本书最锋利钩子，**输出边界写死：只给四类粗分类＋每类一句话识别＋一句通用原则，严禁二级修法决策树，精细诊断指向问分**，对应第4章）

每章另配 1 个「当章就能做」的最小练习（如对今天一道错题跑一遍分层调试），杜绝只读不练。

## 分阶段写作范围

**第一阶段（可发布 MVP，先用最锋利的认知钩子获客）**：前言＋第1章（系统总览地图）＋第3章（编译·学过≠会用）＋第4章（调试·错题归因四分法）＋第5章（内存缓存·背了就忘）＋第11章（交棒）＋附录 A/B/D/E/F/L。
理由：编译/调试/缓存这三章直击「一看就会一做就废、错题反复错、背了就忘」三大最普遍痛点，是天然的搜索与分享钩子，且最能展示「公开框架」与「个性化诊断留给问分」的边界。错题归因四分法（附录 E）是全书最易搜索分享、且天然守住边界的获客卡——**前提是其输出边界已按硬上限收窄到四类标签＋一句话原则**。

**第二阶段（补全递归骨架与双读者）**：第2章（递归拆解）＋第6章（进程调度）＋第7章（版本迭代）＋第8章（自举元认知）＋第9章（给家长的运维手册）＋第10章（AI 副驾）＋附录 C/G/H/I/J/K。
理由：递归拆解与自举是脊柱但更抽象、需配大量示例打磨；家长运维章在核心方法成形后再写，衔接更准。**第8章自举因最接近「自我升级闭环」、最贴近问分付费内核，留到第二阶段并由主编校准边界（尤其五阶段不给升档路径）后发布**，避免先放最接近泄密的工具、后放约束它的框架（沿用第二本分阶段红线逻辑）。
