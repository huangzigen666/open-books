## 一之二、内容口径调整（主编 2026-06-29 通读 MVP 后 · 最高优先）

> 本节为最新口径。**与后文第四节「不泄密硬约束」、第七节禁止事项第 2/3 条冲突处，一律以本节为准。** 适用于 MVP 各章的修订与第二阶段的新章。

### ① 计算机术语减负 + 硬词翻译
- 保留核心心智模型（学习＝一台你能自己调试的机器）与各章「拆穿迷思」的标题隐喻（编译/调试等），这些不动。
- **每个计算机术语第一次出现，必须紧跟一句大白话解释**（如「编译，就是把听懂的东西变成考试时自己能写出来的」）。
- **「base case」不再做主词**：正文一律用大白话「**能自己做对的最小一步**」（指不查任何资料就能独立做对的那一步）；英文 base case 至多在某章首次出现时括注一次，之后不用。
- **砍二级隐喻与术语堆叠**：源码/可执行、脏缓存、上下文切换、进程调度等，凡读者要停下来想的，换成大白话（「背了取不出来」「注意力往哪放」）或删。一段里不要堆超过一个新术语。
- 验收：一个完全不懂计算机的家长/学生，从头读不卡壳。

### ② 不泄密线：方法层完全放开，只卖「执行＋陪跑」
> 主编 2026-06-29 拍板：**问分卖的是「针对每个孩子的个性化执行＋按周陪跑」这个服务，不是方法本身。** 所以方法可以写透——书越慷慨，越多人信、越多人来找问分做「在我身上落地」那一段。

- **方法层完全写透。** 凡是「一个人能学会、能自己做的方法」——通用方法、自跑诊断流程、怎么自己找薄弱链条、怎么自我归因、进阶方向——全部写进书，**不再为问分保留任何方法**。目标：读者照书能把整套方法自己学会、自己跑通一轮。
- **留给问分的是「服务」不是「方法」**：①针对你这一个孩子的**持续诊断**（有人一直盯着你的具体情况往下挖）；②**按周动态陪跑**（一周一周跟着你调节奏、用工具帮你跑）。这两件是「替你／陪你长期做」，书写得再透也不冲突。
- **指向问分＝软性、偶尔的服务邀请，不是防御式「此处不给」。** 去掉一切「公开安全层到此·X 指向问分」式硬标注（那会让书读起来像在藏东西）。改成自然的一句话邀请，且**不必每章都有**——只在「自己做起来确实累、需要有人长期陪」处点一句，如「方法都在书里了；想要有人针对你自己的情况持续诊断、按周陪你调，那是问分在做的事」。
- 落地：第4章/附录E 已是范例（四类标签＋每类通用修法＋自跑诊断流程；问分＝持续诊断＋按周陪跑；软入口）。第8章五阶段：连「怎么升档」的通用方法也写透，问分只做「针对你个人的持续陪跑」。
- **强自检（反过来做）**：读者照书应能把整套方法自己学会、自己跑通一轮；问分提供的是「替你/陪你长期做」的服务。**若你发现某段在藏方法、读着发虚，就放开它。**

### ③ 去 AI 味 · 顺中文语感
- **把连续的一句一段并成自然段**：同一个意思的几句短句，组织成一两个自然段，不要一句一行连排往下堆；单句成段只在真正要强调处偶尔用、不连续堆。少机械排比；长短句搭配，有自然停顿和口语节奏。
- 去翻译腔、对仗腔；像一个懂行的人坐你对面讲话，不像生成的文档。
- 不为整齐牺牲自然：该连成一段就连，不要每句都换行。

## 二、写作总规则

沿用前两本基线：

1. **结论先行**：每章第一行用加粗写出「本章结论」——这章拆穿什么迷思 / 建起什么能力 + 一句话核心动作。
2. **不写玄学**：禁用「押名校」「保上线」「速成提分」「聪明药」「玄学记忆术」等表达；提分是方法跑通的结果，不是承诺。
3. **短句清单卡片**：多用短句、清单、对照表、卡片；Rex 式直白判断，不写鸡汤大词。
4. **AI 降级**：AI 只做材料整理、查漏、陪练、生成练习，不替考生跑循环/思考/做诊断决策。

本书特有 4 条：

5. **三层落法纪律（防生搬硬凑·硬要求）**：每个计算机/递归概念必须走三层——①先回答「它对应学习里哪个真实困境」（如调试＝错题反复错却不知错在哪），②再给隐喻，③再落一个**当天就能做的动作或一张可填卡**。凡只能讲概念、给不出动作的比喻一律不要写。每章至少 1 个可上手练习。**裁剪令：凡 Codex 顺手带出的二级计算机术语（寄存器/中断向量/指针等），若在附录 A 填不出对应当天动作，一律删，只保留能落动作的核心概念。**
6. **每章开场拆穿一个迷思**：用「先拆穿一个主流学习迷思，再给递归解法」的开场结构提升反常识张力。迷思主张必须与本章方法严格对应，不得耸动脱节（防标题党）。各章迷思已校准：第2章=刷题越多越好、第3章=听懂了就等于会了、第4章=订正＝抄一遍、第5章=死记硬背最保险、第6章=全都想抓/多学就是好、第7章=复盘＝感动自己、**第8章=「学会了就不用管怎么学了」（不要写成「越聪明越好/天赋决定一切」，那与元认知方法咬合不紧）**。
7. **递归脊柱显式**：第2章（拆子问题/base case）、第4章（自引用·错题喂下一轮）、第7章（回溯与收敛）、第8章（自举/元认知）四处必须显式承接「递归」，让递归是明写的脊柱而非装饰。**第1章总览章只锚定不展开。** base case 统一硬判据＝「能不查任何资料独立做对的最小单元」；收敛判据统一＝「连续 N 次独立做对／能讲给别人听懂」——这两条作为全书统一术语强制各章复用。
8. **双读者**：考生能自己上手，家长能看懂。每章正文主要写给考生；第9章整章写给家长。**前言必须给家长一条独立最短阅读路径**（前言＋第9章＋各章末给家长段）。**各章末「给家长段」写死成同一张话术微结构（见三-5），禁抽象词。**

## 三、章节内统一模板

每章按以下结构写（标题层级与前两本一致，正文短句、多清单、多卡片/表格）：

1. **本章结论**（加粗一段）：结论先行，拆穿什么迷思 / 建起什么能力 + 核心动作。
2. **先拆穿一个迷思**：1-2 句点破本章针对的主流学习迷思，给一个可验证的具体现象（不讲空话）。
3. **这个概念对应你学习里的哪个真实困境**：先讲困境，再给计算机/递归隐喻命名（三层落法的前两层）。
4. **一个可上手的递归练习**：给一张可填卡或一套当天能做的动作（三层落法第三层）；卡片只给框架/自查，不给个性化结论。
5. **给家长的一段（话术微结构·写死）**（第1-8、10、11 章每章末尾一小段；第9章为整章）：**严格按 `本环家长能观察到的信号（孩子做题/学习时的具体表现）＋ 一句该说的话 ＋ 一句别说的话` 三件套写，全部具体到可照说的话术；禁止出现「理解孩子/营造氛围/多鼓励/多沟通」类抽象词。**
6. **不泄密红线**：涉及个性化诊断/精准根因/持续陪跑处，写「自己怎么做粗归因/自启动」，然后一句「精准定位与持续纠偏指向问分」，高危章（第4、8章）章末显式标注「公开安全层到此」。
7. **避免玄学的硬要求**：本章每个比喻是否落到了一个真实动作？若否，删。
8. **本章自查**：复选框清单（沿用前两本 `- [ ]` 格式）。

## 四、不泄密硬约束（所有涉及个性化处的章节必读）

本书给三层公开安全层：①可公开的方法框架（七环节系统模型＋每章一个通用动作＋递归五机制）；②自学起步（自己把系统跑起来的卡片与自查表）；③为什么（每个方法背后的认知原理）。守住不外泄的是问分付费内核：

- **个性化诊断**：书只教「自己怎么给错题/学习状态做粗归因」，把「精准定位某个学生某科的真实根因、给定制纠偏路径」留给问分。第4、8 章显式标注此边界。
- **教练/陪跑**：书是一次性自启动指引；「持续监控、按周纠偏、陪你迭代到收敛」是问分的活；**尤其「如何从某一档/某状态升到下一档」的升级路径＝陪跑内核，一律不写。** 第7、8、11 章只给自助模板不给陪跑机制。
- **完整执行系统与最深专有机制**：不写问分内部的诊断量表、参数、评分逻辑、迭代算法；凡涉及处一律留「此处可由问分做个性化升级」的钩子，由主编/产品后补，**架构师/Codex 不编造专有深度**。
- **强自检红线**：若某章已能让学生照抄出一套**无需任何外部诊断就能持续自我升级的完整闭环**，就是写过头，必须收回到「框架+自查问句+为什么+找问分」层（逐章用此条质检）。

**两处写成硬上限（不是提示，是上限）：**

### 第4章错题归因四分法（最高危）
- 只给「看错/不懂/记错/没时间」**四类标签的定义 ＋ 每类一句话识别问句 ＋ 每类一句话通用原则**（如：记错→去查它该长什么样、再重背一次；没时间→把这类题拆到 base case 提速，见第2章）。
- **严禁出现「如果是 X 且 Y 则用 Z 修法」的二级修法分支；严禁针对某学生某科的精细诊断决策树。**
- 调试示例**只演示归因到四类标签为止**，到「精准根因＝为什么这一步会崩」显式断在此处，写「精准根因定位指向问分」。
- 章末显式标注：**「公开安全层到此·精准根因定位与定制纠偏指向问分」。**

### 第8章自举元认知五阶段（次高危）
- 五阶段（模仿/调试/重构/自我编程/进化）**只作「自测我现在在哪档」的静态标尺，每档给 1 个识别信号即可**。
- **严禁写「如何从这档升到下一档」的任何路径化指引——升级路径正是问分陪跑内核。**
- 元认知三问只做「自检问句」，配当堂可填的具体对象（本周哪个方法/哪道题），**不配「问出答案后该怎么调」的决策树。**
- 合成示例只演示「学生发现一个 bug」，到「持续按周纠偏到收敛」显式断开，写「指向问分」。
- 章末显式标注：**「公开安全层到此·持续个性化升级与陪跑指向问分」。**

- 表格用 Markdown 表格；自查清单用 `- [ ]`；卡片用小标题 + 表格/清单。
- 章末更新 `book.md` 目录链接与「当前写作状态」。

## 七、禁止事项

1. **不泄露问分付费核心**：不给「针对你这个孩子的精准诊断、定制纠偏路径、持续陪跑节奏、完整自动跑的执行系统」；所有个性化产出停在「框架+自查问句+需找问分」。**强自检红线**：若某章已能让学生照抄出一套无需任何外部诊断就能持续自我升级的完整闭环，就是写过头，必须收回到框架层（逐章用此条质检）。
2. **第4章错题归因四分法（已按第一之二节②放宽为中等尺度）**：给四类标签＋每类识别问句＋每类**通用修法步骤**＋一个学生能自己跑的「归类→定位缺口→针对修→隔几天重做验证」诊断流程。**只把「针对某个具体学生某科的自适应/个性化诊断」与「持续按周陪跑、动态调整的纠偏节奏」留给问分**；不再禁通用修法与自跑流程。
3. **第8章五阶段（已按第一之二节②放宽为中等尺度）**：除静态自测标尺外，可给**每档到下一档的通用进阶方向（人人适用的一般做法）**；**只把「针对你个人当前状态的自适应升级路径」与「持续按周陪跑」留给问分**。
4. **不写玄学大词**：不堆「成长型思维」「潜能」「聪明药」式空话；不承诺名次、不用押中/保过类表达；不写玄学记忆术。
5. **不生搬硬凑比喻**：每个计算机/递归概念必须先绑定一个真实学习困境、再落到一个当天可做动作；落不到动作的比喻一律删（写正文时逐章自检）；二级术语（寄存器/中断向量/指针）落不到附录 A 当天动作的一律砍。
6. **各章家长段禁抽象词**：必须写成「信号＋该说的话＋别说的话」话术微结构，禁「理解孩子/营造氛围/多鼓励」。
7. **反常识不标题党**：章节迷思钩子与正文方法必须严格对应（第8章迷思用「学会了就不用管怎么学了」），耸动而脱节即标题党，禁。
8. **案例必须脱敏或合成**：禁止可识别学生信息（真实成绩、学校、获奖）。
9. **不越界写规划/志愿内容**：三年规划、志愿填报一律只做一句话交棒，指向第二本/第一本，不在本书展开。
