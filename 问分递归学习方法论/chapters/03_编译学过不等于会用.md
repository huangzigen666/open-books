# 第三章：编译——学过 ≠ 会用，知识要先翻译成考场能执行的指令

**本章结论：听懂了、看会了，不等于会用。本章要拆穿的迷思是「老师一讲我懂了＝我会了」；要建立的能力是自己当编译器（说白了，就是自己合上书检查一遍：学过的东西，到了考场没人帮忙，你能不能自己跑出来），用硬判据看知识有没有真正变成考场上能用的本事。核心动作：合上书，对今天学的一个点跑三道检查——能复述、能讲懂、能做变式；任一条卡住，就找出是哪一步没过。**

先拆穿一个迷思：听懂了不等于会了。

最常见的现象是这样：老师讲题时你一直点头，答案一看也顺，可一合上书自己写，到第二步就卡住了；原题明明会，换个数字、换个问法，立刻就空白。这不是「再认真点」能解释的。这叫学过了，但还没真正变成你自己的东西——借用计算机里的一个说法，叫「没编译通过」。所谓编译，就是把别人讲的内容，翻译成你自己能直接拿来用的版本。

## 1. 真实困境：一看就会，一做就废

很多学生把「听课」当成学习完成了。可上课听懂，只说明你接收到了材料；看答案看懂，也不过是你顺着别人铺好的路走了一遍。

考试要的不是顺着别人走。它要你在没有答案、没有提示、没有老师停下来提醒的情况下，把知识自己调出来，按步骤一步步跑完。这中间差了一步，这一步就叫编译——把听来的东西，变成自己能独立用出来的本事。

## 2. 老师讲的版本，和你考场上能用的版本

老师讲的、笔记上抄的、答案里写的，都是「别人的版本」。这些当然有价值，但它们不能直接替你考试。真正能在考场上不查资料就调出来、自己跑完的，才是「你自己的版本」。

「听懂」只是拿到了老师那版；「会用」是你已经把它改写成了自己考场上能直接用的那版。

所以，「一看就会、一做就废」不是你笨，也不是一句含糊的「理解不透」就能打发。它是一个很具体的毛病：那段知识还停在老师的版本，没翻译成你自己能用的版本。

这么说有个好处——它把问题从情绪里拉出来。你不是整个人不行，只是这一段知识还没生成你自己能用的版本而已。

## 3. 编译通过的硬判据

判断一个知识点到底有没有变成你自己的，别看手感，看三个硬动作。

| 判据 | 合格表现 |
|---|---|
| 能复述 | 合上书和笔记，能把关键步骤用自己的话完整说一遍。 |
| 能讲懂 | 能讲给一个没学过的人听懂，对方能照着你的说法做对。 |
| 能做变式 | 换数字、换条件、换问法后，能独立做对一道同类题。 |

三条全过，才算编译通过。只觉得「我懂了」不算数，重看一遍很顺、跟着答案能抄下来，也都还不算。

为什么这里非要合上书？因为开着书的时候，你其实是在借外部提示做题：答案的下一步、老师的板书、笔记里的关键词，都在悄悄替你补空。考场不会给这些提示。合上书以后，缺哪一步会立刻露出来——这不是坏事，它恰恰说明你终于看见了，从「听懂」到「会用」之间，到底缺的是哪一段。

这里的「能讲懂」对齐全书的收敛判据：连续 N 次独立做对、或能讲给别人听懂；在编译这一环里，它就是验收口。「能做变式」也有条底线：至少要能不查任何资料、独立做对一道。如果连第一步都下不去，就退回第 2 章说的那个「能自己做对的最小一步」——也就是不查任何资料就能独立做对的那一步——从那里重新搭起。

下面这张表故意做得很粗。粗，才安全：它只帮你把「一做就废」归进几类，而不替你判断为什么偏偏你这一科总在这里栽跟头。

| 报错现象 | 一句话识别问句 | 一句话通用方向 |
|---|---|---|
| 看着会，合上书复述不出来 | 我能不能合上书把关键步骤说全？ | 这是没编译，回去把这一步当新内容重读，再合书自测。 |
| 能复述，讲不给别人听懂 | 我能不能让没学过的人听懂为什么这么做？ | 讲不通的那一步就是没过的地方，先把它标出来。 |
| 原题会，换个数字或问法就废 | 同类变式我能不能独立做对一道？ | 这是只记住了原题的套路，去找变式重跑。 |
| 某一步根本讲不出为什么 | 这一步我能不能讲出它为什么成立？ | 这是「没真懂」，退回第 2 章那个能自己做对的最小一步，标出不会的最小单元。 |

要说清楚：这张表不是诊断书，它只告诉你现在大致卡在哪个方向。至于针对你个人的精准根因，下面先讲通用怎么自己修。

## 5. 每类怎么修：四种通用修法

知道卡在哪类还不够，得能自己动手修。下面这四类，是大多数人都能照着跑的通用修法——不针对某一个人，但每一类都给到能落地的一步。

**第一类：看着会，合上书复述不出来。**
这一类的根子，多半是「看过」被错当成了「记住」。背是背了，可一到要用就取不出来，因为你从头到尾都是在看、在认，从没自己调出来过一次。修法就一条：把这一段当成全新内容重读一遍，读完立刻合上书，逼自己从零讲一遍关键步骤；卡在哪一步，就回去只补那一步，再合书重讲。重复到能一口气说全为止。别整页重看，那只会让「眼熟」继续冒充「会」。

**第二类：能复述，但讲不给别人听懂。**
能背步骤、却讲不通，说明你记住的是动作，没记住每一步背后的「为什么」。修法：找出你讲到哪一句会打结、会开始绕——那一句就是你自己也没想明白的地方。盯住它，回去把它的来由弄清楚（这一步凭什么能这么做、它到底解决什么），然后假装对面坐着一个没学过的人，重讲一遍，直到对方能照着做对。讲得顺不顺，比你自己感觉懂没懂靠谱得多。

**第三类：原题会，换个数字或问法就废。**
这一类是只背下了这一道题的固定走法，没抓住背后那条通用思路。修法：拿同一个知识点找两三道变式，先别急着做，先问自己「这几道题，哪一步是共用的、哪一步随题目在变」。把那条不变的主线找出来，再动手；做完回头看，是不是每道都踩在同一条主线上。能把变的和不变的分开，这一类就算修好了。

**第四类：某一步根本讲不出为什么。**
这是真的没懂，不是没编译。这一类别在这一层硬磕，直接退回第 2 章那个「能自己做对的最小一步」，把不会的部分一路往小里拆，拆到某一步你不查资料也能独立做对为止。从那一步往上，一步一步重新搭，每搭一步都讲一遍为什么。地基补上了，上面那步自然就讲得出了。

## 6. 一个你能自己跑的小流程：判断 → 定位 → 修 → 验证

上面四类是分开讲的，连起来其实就是一个你随时能自己跑一遍的环。不用谁帮你诊断，照着跑就行。

1. **判断**：合上书，对今天这个点跑三道判据——复述、讲懂、变式。每条只记「过」或「卡」。三条全过，今天这点收工；只要有一条卡，进下一步。
2. **定位缺口**：拿卡住的那一条，对照报错表归进一类，然后再往里抠一层——到底是哪一句话讲不下去、哪一步换了数字就不会、哪一步说不出为什么。把这一处具体写下来，别只写「这道题不会」。
3. **针对修**：照上一节对应那类的修法去补，只补卡住的那一处，别整章重来。补完当场再合书自测一遍，过了再往下走。
4. **隔几天验证**：当天能讲通，不等于真留住了。给自己定个三五天后的时间，到点把同一个点的变式重做一遍——不许翻之前的记录。还能独立做对，才算真编译过了；又卡住，就把它当成新一轮的输入，重新跑这个环。

## 7. 当天编译练习

今天同样只挑一个（原因见第 2 章）刚学的知识点，或者一道刚听懂的例题。

一个点跑完整、留下记录，比十个点都扫一眼有用得多。本章宁可慢一点，也要让你亲眼看见，到底是哪一步没真正变成你自己能用的东西——看见了这一点，后面的练习才不会白跑。

按下面顺序做：

1. 合上书和答案。
2. 跑三道判据：复述、讲懂、变式。
3. 每条只记「过 / 卡」。
4. 卡住的地方，对照报错表归到一类。
5. 照对应修法补，补完合书重测。
6. 写下隔几天重做验证的时间。

可填卡如下：

| 知识点/题号 | 能复述（过/卡） | 能讲懂（过/卡） | 能做变式（过/卡） | 报错类别 | 下次重做验证时间 |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

这张卡只采集两件事：编译有没有通过，卡在哪一类。它不替你判断「为什么我老是这样」。把没通过的点写进下一轮，就是这套方法里说的自我引用——上一轮的卡点，正好成了下一轮的输入。

## 8. 给家长的一段

这一环里，家长能观察到的信号大概是这样：孩子嘴上说「这个我会了」「老师讲的时候我听懂了」，可一让他不看书讲一遍就卡壳；原题会做，换个数字就空着；复习时一直在重看课本和答案，很少合上书自己测一遍。

一句该说的话：「这个你先别看书，合上给我讲一遍，讲得通就算真会了。」

一句别说的话：「你不是说听懂了吗？怎么还不会，那就是没认真听。」

家长不用懂这道题，只盯一个动作就够了：孩子到底有没有合上书自测，还是在反复重看。

这一整套自查你一个人就能跑通；要是某一科总在这里老编译不过，问分能陪你盯着你本人那一处反复卡的地方。
