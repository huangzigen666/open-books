# 第一章：把自己当成一台计算机——递归学习的总览地图

**本章结论：分数停在那儿不动，很多时候不是你不够努力，也不是你不够聪明，而是学习这台机器从来没被当成一个整体好好查过一遍。这里说的「机器」，就是把你从看书到考试的整个过程，当成一台你自己能拆开、能检查、能修的设备。本章不教任何一环具体怎么修，只把整台机器的图画给你看，再帮你认出自己卡在哪一环。核心动作：对着后面那张七环节自诊断表，圈出最红的一环，然后翻到管它的那一章。**

先拆穿一个迷思：学习好，靠的不是把某一件事堆够量。

多背一点，多刷一点，多熬一点，看着都像在使劲。可问题是，如果你真正卡住的地方在别处，那光在这一件事上加量，往往一点用都没有。

这事其实是看得出来的。你每天都在学，笔记有，错题本有，计划也排了，分数却偏偏不动；真要问一句到底卡在哪，你又只能含糊地说一句「都不太行」。这说明你缺的不是一句打气的话，而是一张能让你看清自己的系统地图。

## 1. 先把学习看成一条流水线

学习从来不是一个单独的动作，它至少是七件事连在一起：

- 接收新内容——把课和书里的东西收进来。
- 把看懂的变成考场上真能用的。
- 让记住的东西，需要时能拿得出来。
- 安排好时间，也安排好注意力往哪放。
- 在限定时间里把题做完。
- 用错题看清自己到底错在哪一类。
- 让这一轮的结果，能喂进下一轮。

这七件事不是各干各的。它们串成一条流水线，而且是一条会拐回来的循环：前面收进来的东西乱，后面做题就跟着卡；学过的没变成会用的，同一类错题就一遍遍冒出来；错题查完不带进下一轮，下次还在原地摔一跤。

所以这本书借一台计算机来给这七件事起名字。不是为了显得高级，是因为这样你才好对着名字找自己卡在哪。

## 2. 七环节系统模型

下面这张图只回答一件事：这一环是干什么的。至于「怎么把它做好」，全留给后面对应的那一章。

| 顺序 | 环节 | 这一环负责什么（大白话） | 对应章节 |
|---|---|---|---|
| 1 | 输入 | 新知识怎么进到你脑子里 | 本章定位；配[第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 2 | 编译 | 学过的东西，能不能变成考场上真能用的（「编译」=把看懂的翻成会做的） | [第3章·编译](../chapters/03_编译学过不等于会用.md) |
| 3 | 存储 | 记住的内容，需要时能不能取出来 | [第5章·内存与缓存](../chapters/05_内存与缓存.md) |
| 4 | 调度 | 时间和注意力，怎么分配 | [第6章·进程调度](../chapters/06_进程调度.md) |
| 5 | 执行 | 限时的时候，能不能把题做完 | [第6章·进程调度](../chapters/06_进程调度.md) + [第4章·调试](../chapters/04_调试错题当bug.md)（限时提速见第4章「没时间」类） |
| 6 | 调试 | 错题暴露出你错的是哪一类（「调试」=像查机器故障一样查错题） | [第4章·调试](../chapters/04_调试错题当bug.md) |
| 7 | 迭代 | 这一轮的结果，怎么带进下一轮 | [第7章·版本迭代](../chapters/07_版本迭代.md) |

连起来就是这么一条线：

```text
输入 → 编译 → 存储 → 调度 → 执行 → 调试 → 迭代
                                      ↑          ↓
                                  下一轮输入 ← 回流
```

最后那条拐回来的线，就是这本书要讲清楚的「递归」。这里先把它理解成一句话：上一轮不是到头了，它会变成下一轮的开头——这一轮的错题、这一轮的结果，都会接着喂进下一轮。

本书章节不按这条流水线先后排，而是按你最常先卡的入口排——错题调试提前到第4章先讲，背了忘放第5章；对着右边「去读哪一章」翻即可。

## 3. 撑起这套循环的五个底层机制

七环节是你看得见的外层地图。底下还有五个更基础的机制，撑着整套循环在转。这里同样只点个位置，不讲做法。

| 底层机制 | 一句话说清它是什么 | 去哪一章 |
|---|---|---|
| 拆成小问题 | 把笼统的「不会」，拆成一个个更小、更具体的卡点。 | [第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 能自己做对的最小一步 | 不查任何资料就能独立做对的那一步。 | [第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 上一轮喂下一轮（自引用） | 这一轮的错题和结果，会成为下一轮的起点。 | [第4章·调试](../chapters/04_调试错题当bug.md) |
| 退回去再往前（回溯） | 出错了就退回到最近一个还做得对的地方，改完再往下一版走。 | [第7章·版本迭代](../chapters/07_版本迭代.md) |
| 算「够好了」的标准 | 比如连续 N 次不靠帮助做对，或者能讲明白让别人听懂。 | [第7章·版本迭代](../chapters/07_版本迭代.md) |

说清楚边界：本章不讲怎么拆、不讲怎么查错题、不讲怎么复盘，它只告诉你这几件事分别归哪一章管。

## 4. 七环节自诊断表

现在做本章唯一的动作。先别急着改计划，对着下面每条症状，给那一环打个红、黄或绿。

| 环节 | 可观察症状句 | 我的标记 | 去读 |
|---|---|---|---|
| 输入 | 你最近是不是课听了、书看了，却说不清今天到底学了哪几条？ | 绿 / 黄 / 红 | 本章；配[第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 编译 | 你最近是不是一看就会、一做就废，例题懂了，一动笔就卡？ | 绿 / 黄 / 红 | [第3章·编译](../chapters/03_编译学过不等于会用.md) |
| 存储 | 你最近是不是背的时候很顺，过几天默写就空，考场上想不起来？ | 绿 / 黄 / 红 | [第5章·内存与缓存](../chapters/05_内存与缓存.md) |
| 调度 | 你最近是不是学一会儿就刷手机，几科横跳后回不到状态？ | 绿 / 黄 / 红 | [第6章·进程调度](../chapters/06_进程调度.md) |
| 执行 | 你最近是不是平时会做，一到限时考试就来不及、跑不完？ | 绿 / 黄 / 红 | [第6章·进程调度](../chapters/06_进程调度.md) + [第4章·调试](../chapters/04_调试错题当bug.md)（限时提速见第4章「没时间」类） |
| 调试 | 你最近是不是错题抄得很整齐，同类题还是反复错，说不清错在哪？ | 绿 / 黄 / 红 | [第4章·调试](../chapters/04_调试错题当bug.md) |
| 迭代 | 你最近是不是复盘只剩一句「下次认真点」，下周又踩同一个坑？ | 绿 / 黄 / 红 | [第7章·版本迭代](../chapters/07_版本迭代.md) |

红黄绿怎么打：

| 颜色 | 判定 |
|---|---|
| 绿 | 这条症状最近基本不出现。 |
| 黄 | 这条症状偶尔出现，但没有反复卡住。 |
| 红 | 这条症状反复出现，一看就中。 |

别一冲动把七环全打成红——全红，就等于没定位。本章要你做的不是把自己从头到尾批一遍，而是找出这一周最明显的那个入口，一个就够。入口找准了，剩下的具体动作，交给对应章节。

打完别想偏了：红不是差评，它只是一个入口；黄不等于安全，它说明这一环已经开始往后面拖了；绿也不是从此不用管，只是这周先不从这儿动刀而已。

完整可勾版见[附录B·学习系统七环节自诊断表](../appendix/附录B_学习系统七环节自诊断表.md)。

全书每个概念对应的当天该做什么，见[附录A·计算机概念学习动作对照总表](../appendix/附录A_计算机概念学习动作对照总表.md)。

本章唯一动作指令：圈出你最红的一环，翻到它对应的章。到这里就停，先别在本章里找修法。

## 5. 给家长的一段

这一环，家长能看到的信号是：孩子每天都在学，书桌上卷子笔记都有，可你一问「你现在最卡哪一环」，他说不清；分数一不动，他就一句「我太笨」或者「我就是不够努力」。

该说的一句话是：「先别急着加时间。咱们对着这七环看一眼——你这周最红的是哪一环？是听懂了做不出，是背了忘，还是错题反复错？」

别说的一句话是：「你就是不够努力，再多刷两套就好了。」

家长不用替孩子去修。你先帮他把那句「我不行」，换成「我卡在某一环」，就已经做了最要紧的一步。

## 6. 这本书给到哪，剩下的归问分

本章给你的是一张完整的地图，外加一次粗定位。

照着它，你能看出自己这周大概卡在输入、编译、存储、调度、执行、调试、迭代里的哪一环——这是你自己就能跑通的第一步。后面每一章会接着给你那一环的通用修法，还会配一套你能自己跑的自查流程：先判断卡在哪，再找出缺口在哪，然后针对着补，过几天重新做一遍验证。这套通用方法和自查，书里都会写透。

那剩下的是什么？是这本书给不了、得交给问分的那部分：为什么偏偏是你、在这一科上反复栽进这一环，你那条最弱的链子到底断在哪里，以及一周周贴着你的状态去调节奏和重点。

这部分书里不展开——它本来就得贴着某个具体的人、一周周地做。

## 7. 避免玄学的硬要求

本章所有比喻只有一个用处：帮你圈出卡点。

一句话要是落不进上面那些表格，就删掉；一个概念要是对不上[附录A·计算机概念学习动作对照总表](../appendix/附录A_计算机概念学习动作对照总表.md)，就不进这本书。
