# 第二章：递归调用——把一道不会的题，拆到你会做的那一层为止

**本章结论：刷题不是越多越好，真正有效的是把一道不会的题往下拆，拆到你能自己做对的最小一步，再顺着这一步往回拼回整道题。今天只挑一道真不会的题，逐层拆到能自己做对，写下来，再用[附录C·递归拆解卡](../appendix/附录C_递归拆解卡.md)重跑一遍。**

## 1. 先拆穿一个迷思

很多人相信：题刷得越多，成绩自然会上去。

问题在于，遇到不会就狂刷，刷到的常常是你已经会的部分；真正卡住的那一小步，一直没被单独拎出来练过。于是你看例题秒懂，自己动笔卡在第一步；抄完这道答案，换个数字、换个同类问法，又不会了。

这不是题量不够，是你没有拆到真正卡住的那一层。

## 2. 真实困境：你不会的不是整道题

一道综合题摆在面前，很多学生第一反应是「这题我不会」。这句话太粗。

一整道题里，通常有读题、识别条件、调用公式、代入、计算、写结论好几层。你说整道都不会，可能只是其中一小步没接上。可你没有把那一步拆出来，只能整题放弃，或者直接抄答案。

抄答案的问题是：它帮你把整条路走完了，却没告诉你哪块砖是你自己缺的。下次换一题，答案不在旁边，你还是从同一个地方断。

递归调用，用大白话说，就是别一口吞整道题。先把大问题拆成小问题；小问题还不会，再拆成更小的问题；一直拆到「能自己做对的最小一步（英文叫 base case，后面就用大白话）」。到那一步以后，再反过来一层层往回拼。

所以，「我不会这道题」要改成一句更有用的话：「我还没把它拆到我会的那一步。」

## 3. 怎么算拆到底

拆到底不是拆到看着简单，而是拆到一个硬判据：不查任何资料、不看答案，这一步你能独立做对、独立写得出来。还要瞄一眼答案才敢写，不算到底；能写出结果却讲不出这一步为什么成立，不算到底；只是觉得「这个应该会」、一盖住答案就空白，也不算到底。

每个人的底不一样。别人一步能跨过去，你可能要拆成三步，这不丢人——真正浪费时间的，是明明底下没接住，还硬逼自己做整道题。

拆到底，是为了找到一个自己确实能站住的地方。站住了再往上搭，这一步别省。

练到什么程度算够，后面第7章会讲收敛判据：连续 N 次独立做对，或者能讲给别人听懂，这一小块才算暂时稳了。

## 4. 一题拆到底：分层示范

下面用一道合成题，不对应任何真实学生。

题目：一次函数图像经过点 A(2,5)，并且与 y 轴交于 3。求这个函数表达式，并判断点 B(4,7) 是否在这条直线上。

不要急着看答案。先拆。

```text
整道题：
求函数表达式，并判断 B 点是否在直线上。

第一层：
要先写出一次函数的一般形式 y = kx + b。

第二层：
题目说与 y 轴交于 3，所以 b = 3。

第三层：
图像经过 A(2,5)，说明 x = 2、y = 5 可以代入 y = kx + b。

第四层：
代入后得到 5 = 2k + 3。

第五层：
解方程：5 = 2k + 3。
```

假设你卡在第五层。

这时别说「一次函数我不会」。你已经会了前面几层：知道形式，知道 b，知道点能代入。真正卡住的是一个更小的东西：解 `5 = 2k + 3`。

继续拆：

```text
5 = 2k + 3
两边同时减 3，得到 2 = 2k
两边同时除以 2，得到 k = 1
```

如果这一步你能不看答案自己写出来，这就是你这道题里的底。

触底以后往回拼：

```text
k = 1，b = 3
所以函数表达式是 y = x + 3
判断 B(4,7)：把 x = 4 代入，y = 4 + 3 = 7
所以 B 在这条直线上
```

这道题的关键不是一次函数本身，而是你看见了：整道题不会，最后可能只是一个小方程没接住。把那一小步单独拎出来，它就能练；能练，就能往回拼。

如果你卡在第三层，不知道「点在图像上」为什么能代入，那底就在更下面：你要先弄明白「一个点在函数图像上，意思是这个点的 x、y 同时满足表达式」。这就是另一个人的底。

所以拆题没有统一深度，深度由你自己能不能独立做对决定。

仿做时别急着问「这题属于什么题型」。先问更小的问题：这题第一步要干什么？第一步我会不会？会，就往上接；不会，就继续拆。拆题的顺序不是从整本书的目录出发，而是从你眼前卡住的那一步出发。

也别把拆题写成很长的反思。每一层只写一句话就够：这一层要做什么、我卡在哪里、下一层要拆到哪。写得越长，越容易重新变成抄答案。写得短，才看得出哪一步真断了。

拆完以后，一定要往回拼。只会拆，不往回拼，等于把题拆散了没装回去。你要亲眼看见：那个小方程接回一次函数，函数表达式再接回判断点的位置，整道题才重新跑通。拼回去这一下，也是在确认你不是背下了答案，而是真把这条路自己走通了一遍。

## 5. 今天就做：一张空白拆解卡

今天只挑一道你真不会的题，别挑十道——十道会让你重新掉回刷题计数，那又绕回了刷量，没拆到点子上。

按这个顺序写：

1. 写下整道题。
2. 写第一层：这题要先做哪件事。
3. 问自己：这一步我卡在哪个更小的东西上。
4. 继续往下拆，直到有一步你不查资料也能独立做对。
5. 把这一步单独写出来，确认自己真能做。
6. 从这一步往回拼，一层层接回整道题。
7. 最后写一句：这道题真正卡住的是哪一小步。

可填模板见[附录C·递归拆解卡](../appendix/附录C_递归拆解卡.md)。那张卡只给空格，不替你下结论；题到底要拆到哪一层，要靠你用「不查资料独立做对」这个判据来验。

拆出来的最小卡点，不要写完就扔。把它放进下一轮练习：今天练这一小步，过几天盖住答案重做。这样它就接上了第4章的错题喂下一轮，也接上了第7章的版本迭代。

## 6. 给家长的一段

这一环家长能观察到的信号：孩子一遇到大题或难题就卡住不动，半天不下笔；要么直接抄答案，要么说一句「不会做」就整道跳过；同类题抄过一次，下次还是不会。

一句该说的话：「先别抄答案，这道题里哪一小步是你已经会的？从那一步开始往上接。」

一句别说的话：「这么简单都不会？多刷几道不就会了。」

家长不用会做这道题，只盯一个动作：孩子有没有把题往下拆、找到自己会的那一步，而不是整道放弃或直接抄答案。

## 7. 结尾：能拆通，才叫方法开始跑

这套方法不神秘：往下拆，拆到能自己做对的最小一步；再往回拼，拼回整道题。你一个人就能照着跑。

真到自己反复拆不下去、总卡在同一类题上，问分能帮你定位是哪一层老接不住。
