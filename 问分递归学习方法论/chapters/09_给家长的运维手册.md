# 第九章：给家长的运维手册——你是供电和散热，不是超频

**本章结论：你不用懂学科，也不用替孩子做题。你的活是把孩子学习的运行环境维护好：稳定供电，意思是情绪、后勤、信任别断；做好散热，意思是把家里的焦虑降下来；不超频，别一味加压加时长；不替 CPU 运算，别代替孩子思考。核心动作就一句：盯住环境，把手从题上拿开。**

## 1. 先点破一个家长误区

很多家长的直觉是：我越使劲，孩子提分越快。于是你坐旁边盯题、盯分、盯手机，临时加卷子，替他整理错题，替他决定先学哪科。

可你可能已经见过反效果：你一坐到旁边，孩子写得更慢；你把错题本整理得整整齐齐，下次同类题他还是不会；你越催，他越烦，最后学习变成你们俩对抗。问题不在于你不关心，而是你把力气用到了题上。家长该使劲的地方在环境上，不在孩子脑子里那一步推导上。

还有一种更隐蔽的场景：你陪读时，孩子明明平时能做的题也开始写错，草稿纸越擦越乱；你一离开，他反而能安静把前几步跑完。不是孩子故意和你对着干，而是你站在旁边那一刻，学习现场多了一个「随时会被评价」的压力源。你越想把他按进正确步骤，他越忙着防备你的反应，真正用于思考的那点力气就被挤走了。力气用错地方，关心也会变成干扰。

## 2. 把系统隐喻翻译成家里动作

孩子才是那台真正跑学习循环的机器。前八章写给孩子，是让他自己学会拆题、编译、存储、调度、执行、调试、迭代。家长的位置不是上去替他跑，而是做运维：让机器能稳定工作。

供电，就是情绪、后勤、信任这三路电别断。在家里具体就是：吃睡尽量稳，桌面尽量安静，孩子说自己在修一个问题时，你别每天用分数把他打回原点。电一断，机器就重启，前面刚搭起来的节奏全丢。

比如孩子考差那天晚上，原本已经愿意说「我这次主要是时间不够」，你一句「早跟你说过别玩手机」，他很可能立刻闭嘴。那一刻最该做的不是追责，而是把话缩短：「先说一件，这次最卡的是哪一步。」你把入口留小，孩子才有可能继续讲；你把入口扩大成整个人的问题，他只能防守。

散热，就是降焦虑。孩子已经因为错题、排名、作业热起来了，家长再把自己的焦虑倒上去，机器只会更慢。在家里具体就是：先把话说短，先问卡在哪一步，别在孩子卡住时连带翻旧账。

散热最常见的失败，是家长自己先急起来：孩子刚说「我不会」，你脑子里已经跑到中考、高考、以后怎么办。接下来你说出口的就不是眼前这道题，而是一整串后果。更稳的做法是先把后果放下，只问眼前：「现在是读题没看懂，还是某一步接不上？」焦虑被压回到一步，孩子才有空间处理一步。

超频，就是给机器强行加压、加时长、加任务。放到家里，就是一考差就加一套卷、报一个班、熬到更晚。短期看起来转得快，长期更容易崩。

替 CPU 运算，就是你替孩子思考。CPU 是电脑里真正干活的那块，放到学习里，就是孩子自己的脑子。你直接讲答案、替他归错题、替他决定本周先练什么，看起来帮了忙，实际把他自己那块「处理器」闲置了。

## 3. 家长该做／不该做对照表

这张表先列最常踩的坑。照着把几条做稳，就已经能少很多冲突。

| 场景 | 该做（供电/散热） | 不该做（超频/替孩子算） |
|---|---|---|
| 情绪 | 先问「卡在哪一步」，话说短。 | 一上来骂态度、骂分数、骂别人家孩子。 |
| 后勤 | 固定吃饭、睡觉、桌面、手机摆放。 | 临时加任务，孩子刚坐下又被你打断。 |
| 信任 | 让孩子说本周自己在修什么。 | 每天追问「你到底有没有学」。 |
| 一道不会的题 | 问他哪一小步会，哪一步断。 | 直接讲答案，或说「这么简单都不会」。 |
| 一次考砸 | 只挑一个最影响的 bug 说清。 | 把所有科、所有旧账一次翻出来。 |
| 报班加量 | 先看现有问题有没有被定位。 | 一焦虑就加课、加卷、加到睡眠被挤掉。 |
| 错题本 | 让孩子自己归类，你只看有没有留痕。 | 替他整理得很漂亮，替他下结论。 |
| 家长焦虑 | 自己先停十分钟，再开口。 | 把「我怕你以后吃亏」变成当场施压。 |

家长不需要判断这题到底怎么做。你只判断一件事：我现在是在供电散热，还是在超频、替他算。

有一个很实用的自测：你说完一句话以后，孩子更愿意继续讲卡点，还是更想把本子合上？前者多半是在供电，后者多半是在断电。你不需要把自己变成温柔老师，也不用把所有话都说得漂亮；先把最伤人的三句话停掉，把「你怎么又这样」换成「卡在哪一步」，家里的学习对话就会先降一档火。

## 4. 话术替换：七环节逐环换嘴

下面这张表把前八章翻成家长能说出口的话。bug，就是一做就出错的那一处；你问的是「卡在哪」，不是「你怎么又不行」。

| 环节 | 孩子常见表现 | 别说这句 | 该说这句 |
|---|---|---|---|
| 输入 | 上完课说不清今天学了什么。 | 「你上课到底听没听？」 | 「今天先说三条新东西，说不全就翻第1章那张表定位。」 |
| 编译 | 听懂了，自己一做就卡。 | 「懂了还不会，就是没认真。」 | 「先合上书讲一遍，讲到哪句卡住，我们就标哪句。」 |
| 存储 | 反复读，合上书写不出。 | 「多读几遍自然会。」 | 「先合上书写三条，写不出再翻书查。」 |
| 调度 | 学一会儿摸手机，多科横跳。 | 「你怎么这么坐不住！」 | 「这 25 分钟只做这一件，手机先放这，结束看被打断几次。」 |
| 执行 | 平时会，限时写不完。 | 「怎么又磨蹭到没时间？」 | 「哪一步最慢？先把那一步单独练顺。」 |
| 调试 | 错题抄了，同类还错。 | 「你怎么又错，抄三遍。」 | 「这道题先归一类：看错、不懂、记错，还是没时间？」 |
| 迭代 | 复盘只写「下次认真」。 | 「反思有什么用，分还这样。」 | 「这周只改了哪一个动作？下周先验这一处。」 |
| 元认知 | 一套老办法用很久，没效果还继续用。 | 「方法有什么好想的，多做就行。」 | 「这周哪个学法没用了？下周换一个动作试试。」 |

遇到不会的题，可以回指[第2章](../chapters/02_递归调用拆到会做.md)那句「拆到能自己做对的最小一步」。遇到错题，可以回指[第4章](../chapters/04_调试错题当bug.md)的四类归因。你不用重教方法，只负责把话从刺激孩子，换成帮他定位。

这张表最重要的不是「说得高级」，而是把家长的本能反应往后撤半步。孩子不会、做错、考砸的时候，你最容易抢着先开口——先忍住，让他把卡在哪说出来。你少抢一步，孩子才有机会把自己的那一步说出来。

## 5. 家庭运维例会：一周一次，十几分钟

家庭例会不是查岗会。每周一次，15 分钟以内，固定时间。孩子先说，家长主要听和记；会上不讲题、不翻旧账、不临时加任务。完整可填版在[附录J·家长运维对照表与家庭运维例会模板](../appendix/附录J_家长运维对照表与例会模板.md)。

为什么一定让孩子先说？因为真卡点通常藏在孩子自己的几句话里：他会说「我不是不会，是写到第三步就慌」，也会说「我知道该背，就是一合书就空」。家长先开口，往往会直接给结论，孩子就只剩点头、解释、辩解三种反应，真正的卡点反而出不来。你只记，不是装沉默，而是在给他留出把问题说完整的时间。

孩子说完以后，你可以只复述一句：「我听到的是，下周先改这一处。」这句话比立刻提建议更有用。它会让孩子确认自己被听见，也会把本周的动作压回到一个点上。动作越少，越容易真的做完。下周回看时，也只看这一处有没有变化，别把短会扩成全科总结。

第一次开会最容易翻车的地方，是家长忍不住把小会开成审判会：孩子刚说一个 bug，你马上追问为什么早不说；孩子刚说下周只改一个动作，你又加三件「顺便」。稳住的办法很简单：会前先写一句提醒放在手边：「今晚不讲题、不翻旧账、不加任务。」忍住这三件事，第一次小会就算成功一半。

| 环节 | 谁说 | 说什么 |
|---|---|---|
| 本周最卡 | 孩子 | 这周哪一科、哪一类题最难受。 |
| 本周修好 | 孩子 | 我修好了哪个 bug，怎么知道它比上周好。 |
| 还卡着的 | 孩子 | 哪个 bug 反复卡，自己拆不动。 |
| 下周一件事 | 孩子定，家长记 | 下周只先改哪一个动作。 |
| 后勤一件事 | 家长 | 我这周帮你做哪一件环境上的事。 |

会后只留一张小表，不写长总结。表越短，越容易下周继续开。先保证能坚持，别追求好看。

| 本周孩子说卡在哪 | 家长这周供电散热做得如何 | 下周环境上只调一件事 |
|---|---|---|
|  |  |  |

硬纪律再说一遍：例会上家长不讲题、不判对错、不把小会变成批评会。你说得越多，孩子越少说；孩子说得越少，真正的卡点越藏起来。

例会开不下去，先把它缩短。十分钟也行，甚至五分钟也行，只保留三件事：孩子说一个已经修好的 bug，一个还卡着的 bug，下周只先改一个动作。家长只记，不点评。这个形式跑稳以后，再慢慢加后勤项。短会的价值不在仪式感，而在固定给孩子一个「说卡点不会立刻挨骂」的窗口。

## 6. 把这一章收住

你的活说到底就八个字：供稳电，做好散热；别超频，别替他算。这些你照着这一章就能做起来，不用懂任何一科。提分是孩子那套学习机器跑稳之后可能出现的结果，不是你多催几句就自动发生。

你把供电和散热做稳就够了；针对孩子个人的持续诊断和长期陪跑，是问分的事。
