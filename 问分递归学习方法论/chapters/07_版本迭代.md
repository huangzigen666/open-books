# 第七章：版本迭代——每周给自己发一个新版本，复盘是写更新日志

**本章结论：复盘不是反思一下，也不是写几句「下次更努力」感动自己。本章要建起的能力是：像发版本一样，一周给自己的学习方法更新一次，写清本版改动、修复的毛病、遗留问题、下版只试一件事，并用收敛判据判断哪一类可以收工。核心动作：今天开一份更新日志，按四栏填一版，再用[附录H·每周学习更新日志模板](../appendix/附录H_每周学习更新日志模板.md)每周续上。**

版本迭代，就是一版一版改；放到学习里，就是别每次都推倒重来，而是在上一版基础上只改一处，观察它有没有用。更新日志，就是把这次到底改了什么写下来，方便下次回头看。

## 1. 先拆穿一个迷思

很多人把复盘理解成「考完想一想」「错题抄一遍」「写一句下次要细心」。这些都可能有情绪价值，但不一定能让方法变好。

真正的问题是：下周你还会不会错同一类？你这周到底改了哪一处？这一处有没有起作用？要是这些都没留下记录，复盘就只是叹气，或者把自己感动一遍。

最常见的现象是：每次都复盘，每次都写「更努力」，一连几周错的还是同一类题。另一种更隐蔽：一不顺就整科推倒重来，前几周已经做对的东西也一起扔掉。

这不叫迭代。它只是重启。

## 2. 真实困境：你没有上一版，也没有下一版

学习方法很容易变成一团糊。今天觉得自己背得不够，明天觉得题刷少了，后天觉得时间安排有问题。每次都想改，每次都改一堆，过一周一看，不知道到底是哪一处让情况变好，也不知道是哪一处添了乱。

还有一种情况：你一道题做错，就从头翻课本；一类题没做稳，就把整章重学一遍。你没有回到最近一个做对的地方，而是把已经跑通的部分也扔了。

版本迭代要解决的就是这两件事：第一，别靠感受复盘，要留下更新日志；第二，别推倒重来，要回退到最近做对处，带着修正从那里重跑。回溯，用大白话说，就是退回最近一个你确实做对的节点，再往下走。收敛，就是练到稳了、不再忽对忽错，可以放下这一类。

## 3. 发版本：回退到最近做对处重跑

发一个学习版本，不是从头重学。

更好的做法是：先找最近一个你确实能自己做对的节点。这个说法接上[第2章·递归调用](../chapters/02_递归调用拆到会做.md)里的「能自己做对的最小一步」：不查资料、不看答案，你能独立做对的那一步。找到它以后，只带上这次要修的那一处，从那里继续往下重跑。

举个合成场景：一类数学大题，你第一步能读懂条件，第二步能列出式子，第三步变形总卡。那就不要从整章开始重学。你回到「能独立列式」这个节点，把第三步变形单独修，再从列式那里往后重跑。

这能保住前面已经做对的东西。回退还省时间：你不用把整章从头重学，只在真正卡住的那一点上花力气，前面跑通的部分直接接着用。学习最怕两种浪费：一种是错了不修，另一种是明明前半段会了，还把它一起推倒。

## 4. 硬规矩：一次只改一个变量

一版只动一处。

这个规矩很硬。你可以换一个解题方法，可以调一个学习节奏，也可以改一个习惯，但一版里只选一个。不要同时换资料、换时间、换练法、换错题本格式。

原因很简单：一次改五样，下周变好了，你不知道是哪一样起作用；下周更差，你也不知道是哪一样添了乱。一次只改一个变量，下一版回看才有判断价值。

所以更新日志里最重要的不是写得漂亮，而是写得能验证。下周回头看，只问一句：我只改的这一处，有没有让那个毛病少犯？

## 5. 每周更新日志四栏模板

每周只填一版。四栏固定，不要自己加一堆情绪栏。

| 栏目 | 填什么 |
|---|---|
| 本版改动 | 这一版我只改了哪一处。 |
| 修复的毛病 | 这一版把上一版哪个反复犯的毛病修掉了，或正在修。 |
| 遗留问题 | 这版没解决、留到以后的是哪一个卡点。 |
| 下版实验一项 | 下一版只试这一件事。 |

合成示范：

| 栏目 | 示例填写 |
|---|---|
| 本版改动 | 数学大题第二步变形前，先把已知条件重新圈一遍。 |
| 修复的毛病 | 第二步经常漏掉一个条件，导致后面式子错。 |
| 遗留问题 | 变形以后计算速度还是慢。 |
| 下版实验一项 | 下周只练变形后的计算验算动作。 |

这张表的用法很直接：一周后回看「漏条件」有没有少。如果少了，这版改动有效；还在漏，就把它继续放进遗留问题，下一版换一个更小的动作试。完整空白模板见[附录H·每周学习更新日志模板](../appendix/附录H_每周学习更新日志模板.md)。

## 6. 收敛判据：练到什么程度算够

很多人不会停。练了两遍就觉得会了，结果考试还错；另一类人一直不敢停，同一类题刷到考试前一天还在刷。

这里要用收敛判据。收敛，就是一类题练到稳了，不再忽对忽错，可以把主要精力挪走。硬定义是：连续 N 次独立做对，或者能讲给一个没学过这块的人也听懂。「讲给别人听懂」里的别人，得是没学过的人——你能把它讲到他点头，才说明你不是只在自己脑子里觉得会。N 自己定，建议 3 次起。满足任一条，就给这一类打一个「稳定版」标记，先收工。

这不是偷懒。没有收敛判据，复盘会变成死循环：错题喂进下一轮，下一轮再下一轮，永远不知道哪天该停。[第4章·调试错题](../chapters/04_调试错题当bug.md)讲错题喂下一轮，本章给停手线：该重跑的继续重跑，已经稳定的就别继续占用主要注意力。

今天就做两件事：开一份更新日志，按四栏填下这一版；再挑手头一类题，用收敛判据自测一次：连续独立做对几次了？能不能讲给别人听懂？能，就标稳定版；不能，就写进遗留问题，带进下一版。

## 7. 给家长的一段

这一环家长能观察到的信号：孩子每次考完或周末都说「复盘了」，但复盘就是对着卷子发呆、写一句「下次要更努力」；看不到他这周具体改了哪一处、修了哪个毛病；同一类题连着几周还在错，或者一不顺就把整科推倒从头来。

一句该说的话：「这周你只改了哪一处？写下来：改了什么、修掉了哪个毛病、下周只试哪一件事，下周我们对着看它有没有用。」

一句别说的话：「天天说复盘，怎么还错同一道？复盘有什么用，多做几套不就好了。」

家长不用懂这门学科，只盯一个动作：孩子有没有把「这周只改了一处、改了什么」写下来留痕，而不是空喊更努力或推倒重来。

## 8. 把这一章收住

一周发一版、写更新日志、一次只改一个变量、按收敛判据收工，这套方法前面已经写透了。你一个人就能照着一版一版往下迭代。

方法都在书里了，你自己就能一周周发版本；真到自己迭代两三轮还总卡在同一类毛病、判断不准这一版到底改好没有，可以让问分帮你看一眼。

本章不承诺分数。提分是方法一版版跑稳之后的结果，不是写完一张日志就自动发生。
