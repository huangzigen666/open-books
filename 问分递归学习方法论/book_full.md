# 前言：你不是脑子不够用，是没人教你怎么给自己排错

**分数卡住，多半不是你脑子不够用，也不是没使劲，而是「学习」这台机器从来没人帮你检查过哪里出了毛病。最大的隐形毛病，是把「花了时间」当成「学会了」。**

很多学生每天都在学。课听了，题刷了，错题也抄了，看起来一直没闲着。问题是，没什么产出。上周学过的东西，今天合上书就说不出来；例题看的时候懂了，自己一动笔就卡住；错题本越抄越厚，同一类题下次照样错。

这不是简单一句「不努力」能盖过去的。说白了，是这台机器一直带着毛病在转，没人停下来查过。

> 这本书反复用一个比方：把你的学习看成一台机器，一台你自己能拆开、能检查、能修的机器。后面会借用几个跟电脑有关的词来打比方——出现一个，我就用大白话解释一句，你不懂电脑也能一路读下去。

## 先做个 5 分钟自测

现在就做。合上书和笔记，拿一张纸，挑你上周学过的某一章。计时 5 分钟，凭记忆把你能想起来的要点写下来。不要翻书，不要看答案，不要中途补救。

时间到，数一数：你能独立写出几条？

写不出来，不代表你没听课。更准确的说法是：东西可能进去了，但取不出来——学是学过了，可没在你脑子里留下一个能随时调出来用的结果。

写不出来的那种「卡」，往后会分别对应几个入口：

- 一看就会、一做就废，见[第3章·编译（把「看懂」变成「能用」）](../chapters/03_编译学过不等于会用.md)。
- 错题反复错，说不清到底错在哪，见[第4章·调试（把错题当成要查的毛病）](../chapters/04_调试错题当bug.md)。
- 背的时候挺顺，过几天又取不出来，见[第5章·记不住和想不起来](../chapters/05_内存与缓存.md)。

这一节只帮你看见「哪里有毛病」，不替你下结论说到底是哪根筋的问题。

## 这本书不是什么

它不是押题书，也不承诺保分。它不是一张排满时间的刷题计划表——书名已经把话说明白了：别刷题，去调试。

它也不替你做长期陪跑。这套方法本身——怎么把问题查出来、怎么动手修、为什么这样修——书里都写透了，你照着就能自己跑起来。至于「为什么偏偏是你这个学生、这一科、这个阶段反复栽在同一处」那种贴着你一个人来的判断，得有人长期跟着你才做得了，那是另一回事。

它更不靠神化记忆、天赋故事或者打鸡血的口号。分数不是喊出来的，机器跑通了，结果才有机会冒出来。

## 这本书是什么

它是一套平时怎么学的方法。你可以把自己看成一台能给自己写指令的机器，一条学习流水线从头到尾走七步：把东西输进去、把「看懂」变成「能用」、存下来、安排注意力往哪放、真正动手做、把错的地方查出来修掉、再喂回去重跑一遍。这七步串起来，就是你每天学习真正在干的事。

这条流水线底下还垫着一层更基本的东西，我管它叫「递归脊柱」——说人话就是：碰到一道难题，先拆成几个小问题；一路拆到「不查任何资料、自己就能独立做对的最小一步」；再顺着自己的思路往回搭，看结果到底对没对，不对就再来一轮。

这些只是先在前言里点个名，让你心里有个数，具体怎么做各章会一个一个讲。

说到底，这是一套平时学习的方法。提分是方法跑通之后的结果，不是我跟你打的包票。读完你不一定马上涨分，但应该能看清两件事：自己到底卡在哪一步，今天能动手做的是哪一个动作。

## 最重要的一句话

**真正决定分数的，不是你今天又坐了几个小时，而是你有没有把上一轮的错题和卡点，重新喂回去再跑一遍。**

真在学，是会留下痕迹的：今天卡在哪儿，下一轮拿什么重跑，重跑出来又是什么结果。而感动自己，往往只剩一句话——「我今天学了很久。」这两件事，差得远着呢。

## 四条边界

读这本书之前，先记住四条边界。

1. **不写玄学**：不承诺押题、保分，不靠神化记忆和天赋故事；提分是方法跑通的结果，不是承诺。
2. **AI 只帮忙，不替你想**：AI 可以帮你整理、查漏、讲解、出练习、陪你把当初做错的地方再现一遍；但它不能替你跑这套学习循环，不能替你思考，也不能替你拿主意，详见[第10章·AI副驾](../chapters/10_AI副驾.md)。
3. **方法自己跑，陪跑是另一回事**：怎么把这套方法跑起来、怎么自己跑一轮自查、为什么这样做，书里都讲透了，你照着就能上手。真到自己反复卡在同一处、怎么调都还是错，问分会盯着你这一科一直往下挖、按周陪你调节奏——这是长期陪着一个人才做得了的事，跟看书自学是两码事。
4. **只管平时怎么学**：三年怎么规划、志愿怎么填，不在这本书里展开，到该交棒的地方会一句话指向前两本。

## 交棒前两本

这本书只管一件事：平时学习这台机器，怎么自己查、自己修。

等方法跑顺了，下一棒是三年怎么规划，见[《别等出分才开始》](../../新疆高中三年升学规划书/README.md)。再往后，出分之后志愿怎么填，见[《别把孩子的分数浪费在志愿表里》](../../新疆家长志愿风险识别手册/README.md)。

这本书只负责交棒，不替你展开规划和志愿。

## 怎么用这本书

如果你是考生，先读[第1章·系统总览地图](../chapters/01_系统总览地图.md)，照着把那七步给自己过一遍，看看哪一步最弱。哪一步最红，就先翻对应那一章。

三个最常见的入口：

- 一看就会、一做就废：先读第3章。
- 错题反复错：先读第4章。
- 背了就忘：先读第5章。

每一章都只要求你最后落到一个当天能做的动作，读完就去做，别光划线。

如果你是家长，不用把书里那些跟电脑有关的比方都搞懂。最短的路是：读「前言＋第9章《给家长的运维手册》＋各章末尾的『给家长段』」。第9章《给家长的运维手册》就是写给你的主轴章，系统讲你该怎么供电、散热、不超频。你要看的不是学科细节，而是自己该说什么、别说什么，以及孩子真卡住、长期靠自己调不动的时候，可以找问分这样长期盯着一个人的人搭把手。

# 第一章：把自己当成一台计算机——递归学习的总览地图

**本章结论：分数停在那儿不动，很多时候不是你不够努力，也不是你不够聪明，而是学习这台机器从来没被当成一个整体好好查过一遍。这里说的「机器」，就是把你从看书到考试的整个过程，当成一台你自己能拆开、能检查、能修的设备。本章不教任何一环具体怎么修，只把整台机器的图画给你看，再帮你认出自己卡在哪一环。核心动作：对着后面那张七环节自诊断表，圈出最红的一环，然后翻到管它的那一章。**

先拆穿一个迷思：学习好，靠的不是把某一件事堆够量。

多背一点，多刷一点，多熬一点，看着都像在使劲。可问题是，如果你真正卡住的地方在别处，那光在这一件事上加量，往往一点用都没有。

这事其实是看得出来的。你每天都在学，笔记有，错题本有，计划也排了，分数却偏偏不动；真要问一句到底卡在哪，你又只能含糊地说一句「都不太行」。这说明你缺的不是一句打气的话，而是一张能让你看清自己的系统地图。

## 1. 先把学习看成一条流水线

学习从来不是一个单独的动作，它至少是七件事连在一起：

- 接收新内容——把课和书里的东西收进来。
- 把看懂的变成考场上真能用的。
- 让记住的东西，需要时能拿得出来。
- 安排好时间，也安排好注意力往哪放。
- 在限定时间里把题做完。
- 用错题看清自己到底错在哪一类。
- 让这一轮的结果，能喂进下一轮。

这七件事不是各干各的。它们串成一条流水线，而且是一条会拐回来的循环：前面收进来的东西乱，后面做题就跟着卡；学过的没变成会用的，同一类错题就一遍遍冒出来；错题查完不带进下一轮，下次还在原地摔一跤。

所以这本书借一台计算机来给这七件事起名字。不是为了显得高级，是因为这样你才好对着名字找自己卡在哪。

## 2. 七环节系统模型

下面这张图只回答一件事：这一环是干什么的。至于「怎么把它做好」，全留给后面对应的那一章。

| 顺序 | 环节 | 这一环负责什么（大白话） | 对应章节 |
|---|---|---|---|
| 1 | 输入 | 新知识怎么进到你脑子里 | 本章定位；配[第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 2 | 编译 | 学过的东西，能不能变成考场上真能用的（「编译」=把看懂的翻成会做的） | [第3章·编译](../chapters/03_编译学过不等于会用.md) |
| 3 | 存储 | 记住的内容，需要时能不能取出来 | [第5章·内存与缓存](../chapters/05_内存与缓存.md) |
| 4 | 调度 | 时间和注意力，怎么分配 | [第6章·进程调度](../chapters/06_进程调度.md) |
| 5 | 执行 | 限时的时候，能不能把题做完 | [第6章·进程调度](../chapters/06_进程调度.md) + [第4章·调试](../chapters/04_调试错题当bug.md)（限时提速见第4章「没时间」类） |
| 6 | 调试 | 错题暴露出你错的是哪一类（「调试」=像查机器故障一样查错题） | [第4章·调试](../chapters/04_调试错题当bug.md) |
| 7 | 迭代 | 这一轮的结果，怎么带进下一轮 | [第7章·版本迭代](../chapters/07_版本迭代.md) |

连起来就是这么一条线：

```text
输入 → 编译 → 存储 → 调度 → 执行 → 调试 → 迭代
                                      ↑          ↓
                                  下一轮输入 ← 回流
```

最后那条拐回来的线，就是这本书要讲清楚的「递归」。这里先把它理解成一句话：上一轮不是到头了，它会变成下一轮的开头——这一轮的错题、这一轮的结果，都会接着喂进下一轮。

本书章节不按这条流水线先后排，而是按你最常先卡的入口排——错题调试提前到第4章先讲，背了忘放第5章；对着右边「去读哪一章」翻即可。

## 3. 撑起这套循环的五个底层机制

七环节是你看得见的外层地图。底下还有五个更基础的机制，撑着整套循环在转。这里同样只点个位置，不讲做法。

| 底层机制 | 一句话说清它是什么 | 去哪一章 |
|---|---|---|
| 拆成小问题 | 把笼统的「不会」，拆成一个个更小、更具体的卡点。 | [第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 能自己做对的最小一步 | 不查任何资料就能独立做对的那一步。 | [第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 上一轮喂下一轮（自引用） | 这一轮的错题和结果，会成为下一轮的起点。 | [第4章·调试](../chapters/04_调试错题当bug.md) |
| 退回去再往前（回溯） | 出错了就退回到最近一个还做得对的地方，改完再往下一版走。 | [第7章·版本迭代](../chapters/07_版本迭代.md) |
| 算「够好了」的标准 | 比如连续 N 次不靠帮助做对，或者能讲明白让别人听懂。 | [第7章·版本迭代](../chapters/07_版本迭代.md) |

说清楚边界：本章不讲怎么拆、不讲怎么查错题、不讲怎么复盘，它只告诉你这几件事分别归哪一章管。

## 4. 七环节自诊断表

现在做本章唯一的动作。先别急着改计划，对着下面每条症状，给那一环打个红、黄或绿。

| 环节 | 可观察症状句 | 我的标记 | 去读 |
|---|---|---|---|
| 输入 | 你最近是不是课听了、书看了，却说不清今天到底学了哪几条？ | 绿 / 黄 / 红 | 本章；配[第2章·递归调用](../chapters/02_递归调用拆到会做.md) |
| 编译 | 你最近是不是一看就会、一做就废，例题懂了，一动笔就卡？ | 绿 / 黄 / 红 | [第3章·编译](../chapters/03_编译学过不等于会用.md) |
| 存储 | 你最近是不是背的时候很顺，过几天默写就空，考场上想不起来？ | 绿 / 黄 / 红 | [第5章·内存与缓存](../chapters/05_内存与缓存.md) |
| 调度 | 你最近是不是学一会儿就刷手机，几科横跳后回不到状态？ | 绿 / 黄 / 红 | [第6章·进程调度](../chapters/06_进程调度.md) |
| 执行 | 你最近是不是平时会做，一到限时考试就来不及、跑不完？ | 绿 / 黄 / 红 | [第6章·进程调度](../chapters/06_进程调度.md) + [第4章·调试](../chapters/04_调试错题当bug.md)（限时提速见第4章「没时间」类） |
| 调试 | 你最近是不是错题抄得很整齐，同类题还是反复错，说不清错在哪？ | 绿 / 黄 / 红 | [第4章·调试](../chapters/04_调试错题当bug.md) |
| 迭代 | 你最近是不是复盘只剩一句「下次认真点」，下周又踩同一个坑？ | 绿 / 黄 / 红 | [第7章·版本迭代](../chapters/07_版本迭代.md) |

红黄绿怎么打：

| 颜色 | 判定 |
|---|---|
| 绿 | 这条症状最近基本不出现。 |
| 黄 | 这条症状偶尔出现，但没有反复卡住。 |
| 红 | 这条症状反复出现，一看就中。 |

别一冲动把七环全打成红——全红，就等于没定位。本章要你做的不是把自己从头到尾批一遍，而是找出这一周最明显的那个入口，一个就够。入口找准了，剩下的具体动作，交给对应章节。

打完别想偏了：红不是差评，它只是一个入口；黄不等于安全，它说明这一环已经开始往后面拖了；绿也不是从此不用管，只是这周先不从这儿动刀而已。

完整可勾版见[附录B·学习系统七环节自诊断表](../appendix/附录B_学习系统七环节自诊断表.md)。

全书每个概念对应的当天该做什么，见[附录A·计算机概念学习动作对照总表](../appendix/附录A_计算机概念学习动作对照总表.md)。

本章唯一动作指令：圈出你最红的一环，翻到它对应的章。到这里就停，先别在本章里找修法。

## 5. 给家长的一段

这一环，家长能看到的信号是：孩子每天都在学，书桌上卷子笔记都有，可你一问「你现在最卡哪一环」，他说不清；分数一不动，他就一句「我太笨」或者「我就是不够努力」。

该说的一句话是：「先别急着加时间。咱们对着这七环看一眼——你这周最红的是哪一环？是听懂了做不出，是背了忘，还是错题反复错？」

别说的一句话是：「你就是不够努力，再多刷两套就好了。」

家长不用替孩子去修。你先帮他把那句「我不行」，换成「我卡在某一环」，就已经做了最要紧的一步。

## 6. 这本书给到哪，剩下的归问分

本章给你的是一张完整的地图，外加一次粗定位。

照着它，你能看出自己这周大概卡在输入、编译、存储、调度、执行、调试、迭代里的哪一环——这是你自己就能跑通的第一步。后面每一章会接着给你那一环的通用修法，还会配一套你能自己跑的自查流程：先判断卡在哪，再找出缺口在哪，然后针对着补，过几天重新做一遍验证。这套通用方法和自查，书里都会写透。

那剩下的是什么？是这本书给不了、得交给问分的那部分：为什么偏偏是你、在这一科上反复栽进这一环，你那条最弱的链子到底断在哪里，以及一周周贴着你的状态去调节奏和重点。

这部分书里不展开——它本来就得贴着某个具体的人、一周周地做。

## 7. 避免玄学的硬要求

- [ ] 我有没有把「能自己做对的最小一步」和「够好了的标准」只做定义锚定，没有展开用法？
- [ ] 我有没有让每一环都有一句「你最近是不是……」的症状句？

- [ ] 我有没有让读者只做一个动作：圈最红一环，再翻对应章？
- [ ] 我有没有把通用方法和自查流程的边界说清——书里给透，针对个人的持续诊断和按周陪跑才归问分？

# 第二章：递归调用——把一道不会的题，拆到你会做的那一层为止

**本章结论：刷题不是越多越好，真正有效的是把一道不会的题往下拆，拆到你能自己做对的最小一步，再顺着这一步往回拼回整道题。今天只挑一道真不会的题，逐层拆到能自己做对，写下来，再用[附录C·递归拆解卡](../appendix/附录C_递归拆解卡.md)重跑一遍。**

## 1. 先拆穿一个迷思

很多人相信：题刷得越多，成绩自然会上去。

问题在于，遇到不会就狂刷，刷到的常常是你已经会的部分；真正卡住的那一小步，一直没被单独拎出来练过。于是你看例题秒懂，自己动笔卡在第一步；抄完这道答案，换个数字、换个同类问法，又不会了。

这不是题量不够，是你没有拆到真正卡住的那一层。

## 2. 真实困境：你不会的不是整道题

一道综合题摆在面前，很多学生第一反应是「这题我不会」。这句话太粗。

一整道题里，通常有读题、识别条件、调用公式、代入、计算、写结论好几层。你说整道都不会，可能只是其中一小步没接上。可你没有把那一步拆出来，只能整题放弃，或者直接抄答案。

抄答案的问题是：它帮你把整条路走完了，却没告诉你哪块砖是你自己缺的。下次换一题，答案不在旁边，你还是从同一个地方断。

递归调用，用大白话说，就是别一口吞整道题。先把大问题拆成小问题；小问题还不会，再拆成更小的问题；一直拆到「能自己做对的最小一步（英文叫 base case，后面就用大白话）」。到那一步以后，再反过来一层层往回拼。

所以，「我不会这道题」要改成一句更有用的话：「我还没把它拆到我会的那一步。」

## 3. 怎么算拆到底

拆到底不是拆到看着简单，而是拆到一个硬判据：不查任何资料、不看答案，这一步你能独立做对、独立写得出来。还要瞄一眼答案才敢写，不算到底；能写出结果却讲不出这一步为什么成立，不算到底；只是觉得「这个应该会」、一盖住答案就空白，也不算到底。

每个人的底不一样。别人一步能跨过去，你可能要拆成三步，这不丢人——真正浪费时间的，是明明底下没接住，还硬逼自己做整道题。

拆到底，是为了找到一个自己确实能站住的地方。站住了再往上搭，这一步别省。

练到什么程度算够，后面第7章会讲收敛判据：连续 N 次独立做对，或者能讲给别人听懂，这一小块才算暂时稳了。

## 4. 一题拆到底：分层示范

下面用一道合成题，不对应任何真实学生。

题目：一次函数图像经过点 A(2,5)，并且与 y 轴交于 3。求这个函数表达式，并判断点 B(4,7) 是否在这条直线上。

不要急着看答案。先拆。

```text
整道题：
求函数表达式，并判断 B 点是否在直线上。

第一层：
要先写出一次函数的一般形式 y = kx + b。

第二层：
题目说与 y 轴交于 3，所以 b = 3。

第三层：
图像经过 A(2,5)，说明 x = 2、y = 5 可以代入 y = kx + b。

第四层：
代入后得到 5 = 2k + 3。

第五层：
解方程：5 = 2k + 3。
```

假设你卡在第五层。

这时别说「一次函数我不会」。你已经会了前面几层：知道形式，知道 b，知道点能代入。真正卡住的是一个更小的东西：解 `5 = 2k + 3`。

继续拆：

```text
5 = 2k + 3
两边同时减 3，得到 2 = 2k
两边同时除以 2，得到 k = 1
```

如果这一步你能不看答案自己写出来，这就是你这道题里的底。

触底以后往回拼：

```text
k = 1，b = 3
所以函数表达式是 y = x + 3
判断 B(4,7)：把 x = 4 代入，y = 4 + 3 = 7
所以 B 在这条直线上
```

这道题的关键不是一次函数本身，而是你看见了：整道题不会，最后可能只是一个小方程没接住。把那一小步单独拎出来，它就能练；能练，就能往回拼。

如果你卡在第三层，不知道「点在图像上」为什么能代入，那底就在更下面：你要先弄明白「一个点在函数图像上，意思是这个点的 x、y 同时满足表达式」。这就是另一个人的底。

所以拆题没有统一深度，深度由你自己能不能独立做对决定。

仿做时别急着问「这题属于什么题型」。先问更小的问题：这题第一步要干什么？第一步我会不会？会，就往上接；不会，就继续拆。拆题的顺序不是从整本书的目录出发，而是从你眼前卡住的那一步出发。

也别把拆题写成很长的反思。每一层只写一句话就够：这一层要做什么、我卡在哪里、下一层要拆到哪。写得越长，越容易重新变成抄答案。写得短，才看得出哪一步真断了。

拆完以后，一定要往回拼。只会拆，不往回拼，等于把题拆散了没装回去。你要亲眼看见：那个小方程接回一次函数，函数表达式再接回判断点的位置，整道题才重新跑通。拼回去这一下，也是在确认你不是背下了答案，而是真把这条路自己走通了一遍。

## 5. 今天就做：一张空白拆解卡

今天只挑一道你真不会的题，别挑十道——十道会让你重新掉回刷题计数，那又绕回了刷量，没拆到点子上。

按这个顺序写：

1. 写下整道题。
2. 写第一层：这题要先做哪件事。
3. 问自己：这一步我卡在哪个更小的东西上。
4. 继续往下拆，直到有一步你不查资料也能独立做对。
5. 把这一步单独写出来，确认自己真能做。
6. 从这一步往回拼，一层层接回整道题。
7. 最后写一句：这道题真正卡住的是哪一小步。

可填模板见[附录C·递归拆解卡](../appendix/附录C_递归拆解卡.md)。那张卡只给空格，不替你下结论；题到底要拆到哪一层，要靠你用「不查资料独立做对」这个判据来验。

拆出来的最小卡点，不要写完就扔。把它放进下一轮练习：今天练这一小步，过几天盖住答案重做。这样它就接上了第4章的错题喂下一轮，也接上了第7章的版本迭代。

## 6. 给家长的一段

这一环家长能观察到的信号：孩子一遇到大题或难题就卡住不动，半天不下笔；要么直接抄答案，要么说一句「不会做」就整道跳过；同类题抄过一次，下次还是不会。

一句该说的话：「先别抄答案，这道题里哪一小步是你已经会的？从那一步开始往上接。」

一句别说的话：「这么简单都不会？多刷几道不就会了。」

家长不用会做这道题，只盯一个动作：孩子有没有把题往下拆、找到自己会的那一步，而不是整道放弃或直接抄答案。

## 7. 结尾：能拆通，才叫方法开始跑

这套方法不神秘：往下拆，拆到能自己做对的最小一步；再往回拼，拼回整道题。你一个人就能照着跑。

真到自己反复拆不下去、总卡在同一类题上，问分能帮你定位是哪一层老接不住。

本章只管平时怎么把不会的题拆到会；三年规划看[《别等出分才开始》](../../新疆高中三年升学规划书/README.md)，志愿填报看[《别把孩子的分数浪费在志愿表里》](../../新疆家长志愿风险识别手册/README.md)。

# 第三章：编译——学过 ≠ 会用，知识要先翻译成考场能执行的指令

**本章结论：听懂了、看会了，不等于会用。本章要拆穿的迷思是「老师一讲我懂了＝我会了」；要建立的能力是自己当编译器（说白了，就是自己合上书检查一遍：学过的东西，到了考场没人帮忙，你能不能自己跑出来），用硬判据看知识有没有真正变成考场上能用的本事。核心动作：合上书，对今天学的一个点跑三道检查——能复述、能讲懂、能做变式；任一条卡住，就找出是哪一步没过。**

先拆穿一个迷思：听懂了不等于会了。

最常见的现象是这样：老师讲题时你一直点头，答案一看也顺，可一合上书自己写，到第二步就卡住了；原题明明会，换个数字、换个问法，立刻就空白。这不是「再认真点」能解释的。这叫学过了，但还没真正变成你自己的东西——借用计算机里的一个说法，叫「没编译通过」。所谓编译，就是把别人讲的内容，翻译成你自己能直接拿来用的版本。

## 1. 真实困境：一看就会，一做就废

很多学生把「听课」当成学习完成了。可上课听懂，只说明你接收到了材料；看答案看懂，也不过是你顺着别人铺好的路走了一遍。

考试要的不是顺着别人走。它要你在没有答案、没有提示、没有老师停下来提醒的情况下，把知识自己调出来，按步骤一步步跑完。这中间差了一步，这一步就叫编译——把听来的东西，变成自己能独立用出来的本事。

## 2. 老师讲的版本，和你考场上能用的版本

老师讲的、笔记上抄的、答案里写的，都是「别人的版本」。这些当然有价值，但它们不能直接替你考试。真正能在考场上不查资料就调出来、自己跑完的，才是「你自己的版本」。

「听懂」只是拿到了老师那版；「会用」是你已经把它改写成了自己考场上能直接用的那版。

所以，「一看就会、一做就废」不是你笨，也不是一句含糊的「理解不透」就能打发。它是一个很具体的毛病：那段知识还停在老师的版本，没翻译成你自己能用的版本。

这么说有个好处——它把问题从情绪里拉出来。你不是整个人不行，只是这一段知识还没生成你自己能用的版本而已。

## 3. 编译通过的硬判据

判断一个知识点到底有没有变成你自己的，别看手感，看三个硬动作。

| 判据 | 合格表现 |
|---|---|
| 能复述 | 合上书和笔记，能把关键步骤用自己的话完整说一遍。 |
| 能讲懂 | 能讲给一个没学过的人听懂，对方能照着你的说法做对。 |
| 能做变式 | 换数字、换条件、换问法后，能独立做对一道同类题。 |

三条全过，才算编译通过。只觉得「我懂了」不算数，重看一遍很顺、跟着答案能抄下来，也都还不算。

为什么这里非要合上书？因为开着书的时候，你其实是在借外部提示做题：答案的下一步、老师的板书、笔记里的关键词，都在悄悄替你补空。考场不会给这些提示。合上书以后，缺哪一步会立刻露出来——这不是坏事，它恰恰说明你终于看见了，从「听懂」到「会用」之间，到底缺的是哪一段。

这里的「能讲懂」对齐全书的收敛判据：连续 N 次独立做对、或能讲给别人听懂；在编译这一环里，它就是验收口。「能做变式」也有条底线：至少要能不查任何资料、独立做对一道。如果连第一步都下不去，就退回第 2 章说的那个「能自己做对的最小一步」——也就是不查任何资料就能独立做对的那一步——从那里重新搭起。

下面这张表故意做得很粗。粗，才安全：它只帮你把「一做就废」归进几类，而不替你判断为什么偏偏你这一科总在这里栽跟头。

| 报错现象 | 一句话识别问句 | 一句话通用方向 |

| 能复述，讲不给别人听懂 | 我能不能让没学过的人听懂为什么这么做？ | 讲不通的那一步就是没过的地方，先把它标出来。 |

| 某一步根本讲不出为什么 | 这一步我能不能讲出它为什么成立？ | 这是「没真懂」，退回第 2 章那个能自己做对的最小一步，标出不会的最小单元。 |

要说清楚：这张表不是诊断书，它只告诉你现在大致卡在哪个方向。至于针对你个人的精准根因，下面先讲通用怎么自己修。

## 5. 每类怎么修：四种通用修法

知道卡在哪类还不够，得能自己动手修。下面这四类，是大多数人都能照着跑的通用修法——不针对某一个人，但每一类都给到能落地的一步。

**第一类：看着会，合上书复述不出来。**
这一类的根子，多半是「看过」被错当成了「记住」。背是背了，可一到要用就取不出来，因为你从头到尾都是在看、在认，从没自己调出来过一次。修法就一条：把这一段当成全新内容重读一遍，读完立刻合上书，逼自己从零讲一遍关键步骤；卡在哪一步，就回去只补那一步，再合书重讲。重复到能一口气说全为止。别整页重看，那只会让「眼熟」继续冒充「会」。

**第二类：能复述，但讲不给别人听懂。**
能背步骤、却讲不通，说明你记住的是动作，没记住每一步背后的「为什么」。修法：找出你讲到哪一句会打结、会开始绕——那一句就是你自己也没想明白的地方。盯住它，回去把它的来由弄清楚（这一步凭什么能这么做、它到底解决什么），然后假装对面坐着一个没学过的人，重讲一遍，直到对方能照着做对。讲得顺不顺，比你自己感觉懂没懂靠谱得多。

**第三类：原题会，换个数字或问法就废。**
这一类是只背下了这一道题的固定走法，没抓住背后那条通用思路。修法：拿同一个知识点找两三道变式，先别急着做，先问自己「这几道题，哪一步是共用的、哪一步随题目在变」。把那条不变的主线找出来，再动手；做完回头看，是不是每道都踩在同一条主线上。能把变的和不变的分开，这一类就算修好了。

**第四类：某一步根本讲不出为什么。**
这是真的没懂，不是没编译。这一类别在这一层硬磕，直接退回第 2 章那个「能自己做对的最小一步」，把不会的部分一路往小里拆，拆到某一步你不查资料也能独立做对为止。从那一步往上，一步一步重新搭，每搭一步都讲一遍为什么。地基补上了，上面那步自然就讲得出了。

## 6. 一个你能自己跑的小流程：判断 → 定位 → 修 → 验证

上面四类是分开讲的，连起来其实就是一个你随时能自己跑一遍的环。不用谁帮你诊断，照着跑就行。

1. **判断**：合上书，对今天这个点跑三道判据——复述、讲懂、变式。每条只记「过」或「卡」。三条全过，今天这点收工；只要有一条卡，进下一步。
2. **定位缺口**：拿卡住的那一条，对照报错表归进一类，然后再往里抠一层——到底是哪一句话讲不下去、哪一步换了数字就不会、哪一步说不出为什么。把这一处具体写下来，别只写「这道题不会」。
3. **针对修**：照上一节对应那类的修法去补，只补卡住的那一处，别整章重来。补完当场再合书自测一遍，过了再往下走。
4. **隔几天验证**：当天能讲通，不等于真留住了。给自己定个三五天后的时间，到点把同一个点的变式重做一遍——不许翻之前的记录。还能独立做对，才算真编译过了；又卡住，就把它当成新一轮的输入，重新跑这个环。

## 7. 当天编译练习

今天同样只挑一个（原因见第 2 章）刚学的知识点，或者一道刚听懂的例题。

一个点跑完整、留下记录，比十个点都扫一眼有用得多。本章宁可慢一点，也要让你亲眼看见，到底是哪一步没真正变成你自己能用的东西——看见了这一点，后面的练习才不会白跑。

按下面顺序做：

1. 合上书和答案。
2. 跑三道判据：复述、讲懂、变式。
3. 每条只记「过 / 卡」。
4. 卡住的地方，对照报错表归到一类。
5. 照对应修法补，补完合书重测。
6. 写下隔几天重做验证的时间。

可填卡如下：

| 知识点/题号 | 能复述（过/卡） | 能讲懂（过/卡） | 能做变式（过/卡） | 报错类别 | 下次重做验证时间 |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

这张卡只采集两件事：编译有没有通过，卡在哪一类。它不替你判断「为什么我老是这样」。把没通过的点写进下一轮，就是这套方法里说的自我引用——上一轮的卡点，正好成了下一轮的输入。

## 8. 给家长的一段

这一环里，家长能观察到的信号大概是这样：孩子嘴上说「这个我会了」「老师讲的时候我听懂了」，可一让他不看书讲一遍就卡壳；原题会做，换个数字就空着；复习时一直在重看课本和答案，很少合上书自己测一遍。

一句该说的话：「这个你先别看书，合上给我讲一遍，讲得通就算真会了。」

一句别说的话：「你不是说听懂了吗？怎么还不会，那就是没认真听。」

家长不用懂这道题，只盯一个动作就够了：孩子到底有没有合上书自测，还是在反复重看。

这一整套自查你一个人就能跑通；要是某一科总在这里老编译不过，问分能陪你盯着你本人那一处反复卡的地方。

# 第四章：调试——错题不是用来罚抄的，是用来定位 bug 的

**本章结论：错题不是用来罚抄的，是用来找出这次思路在哪一步崩了的。核心动作就一句话：每道错题先归到「看错 / 不懂 / 记错 / 没时间」四类之一，再针对那一类去修，过几天盖住答案重做一遍，看修没修好。**

先拆穿一个迷思：订正，不等于把正确答案抄一遍。

很多孩子的错题本又厚又花，步骤抄得工工整整，可分数就是不动。最常见的情形是：同一类题这周错、下周还错；这次对着答案看会了，隔几天盖住答案又卡住；你问他到底错在哪，他只会说「粗心」「没想起来」「当时脑子一下蒙了」。这其实算不上订正完成——他只是把出错的地方抄了一遍，却没搞清楚错在哪、为什么错、下次怎么不错。

我们这本书把学习当成一台你能自己调试的机器。「调试」是个借来的词——程序员写的程序出了毛病，他们不会瞎改，而是先弄清楚是哪一步出了岔子，再对着那一步去修。错题，就是你这台机器给你的报错信息。报错不是拿来罚站的，它是在告诉你：问题出在这儿。

## 1. 错题真正暴露的不是答案，是卡在哪一步

一道题做错，表面看是结果错了；可更要紧的，是你的脑子在某一步卡住了。卡在哪儿，各有各的样：可能是读题时把条件看岔了，可能是某个概念你压根没真正弄懂、只是背了个壳，可能是公式记成了另一个样子，也可能是其实会做、但考试时间不够没做完。

这四种错，扣分的时候长得一模一样，可处理它们的办法，绝不能糊成一句「下次认真点」。看错了，得回去把题重读一遍；真没弄懂，就得退回去补那个概念；记岔了，那就翻课本对一对原样；至于会做但来不及的，只能靠把那几步练熟、把速度提上来。四种错，四副药，抓错药等于白吃。

## 2. 错题归因四分法：先贴标签，再对症修

每道错题，先贴一个粗标签，而且只贴一个。别一上来就写长篇反思，也别急着改训练计划。第一步，是把这个错从一团糊里捞出来，看清它属于哪一类。

下面这张表，每一类都给了三样东西：怎么认出它、认出后照着改的通用办法、以及一句要守住的原则。

| 粗标签 | 怎么认出它 | 认出后照着改（通用修法步骤） |
|---|---|---|
| 看错 | 我是不是把题干、符号、条件、单位、范围看岔了？ | ① 回到原题，把看岔的那个字、那个符号圈出来；② 完整重读一遍题，确认条件没漏；③ 想一句下次提醒自己的话，比如「看到『至多』先停一秒」；④ 过几天拿同类题验证，看还会不会再看岔。 |
| 不懂 | 我能不能合上答案，把这一步为什么这么做，自己讲一遍？ | ① 找出讲不出来的那一步，就是缺口所在；② 退回到你能自己做对的最小一步——就是不查任何资料、自己就能独立做对的那一步；③ 从那一步往上，一步一步把中间补回来，每补一步都要能自己讲出为什么；④ 过几天盖住答案重讲，讲得顺了才算补上。 |
| 记错 | 这个公式、概念、步骤，跟课本或笔记里的原样，对得上吗？ | ① 翻回课本或笔记，查它到底长什么样；② 把记错的地方和正确的并排放，找出差在哪个字、哪个符号；③ 重背一遍，背完默写出来核对，别凭印象硬写；④ 过几天再默一次，确认这回记牢了。 |
| 没时间 | 这题我会做，可考试时间里没来得及做完吗？ | ① 把这类题里你最慢的那一步找出来；② 退回到你能自己做对的最小一步，单独把这一步反复练熟、练快；③ 给这类题掐表，记下自己实际花多久；④ 过几天再掐一次表，看时间有没有压下来。别靠刷整套卷子假装在提速。 |

这张表很粗，先归一类就好。它只帮你把错题放进四个盒子，再给你一条对着这一类去修的路。它不会替你判断「你这个人为什么偏偏老在这一类上栽跟头」，那是更深一层的事，后面会说清楚归谁管。

后面附录里有一张空白卡：[附录E·错题调试归因四分法卡](../appendix/附录E_错题调试归因四分法卡.md)，照着填就行。

## 3. 把错题喂回下一轮

这套学习方法有个关键动作，叫「自引用」——说白了，就是这一轮的结果，会变成下一轮的材料。上一回做错的题，不是终点，是下一回的入口。

错题就是最典型的这种材料。很多人订正完就合上本子，觉得这道题翻篇了；可真正有用的做法，是把它放进下一轮，再跑一遍。

当天，只做三件事：

1. 写下题号。
2. 贴上四类标签里的一个。
3. 写下下一次重做的时间。

到了重做那天，先盖住答案，自己从头做。能独立做对，说明你这一轮真的修好了。要还是错，就再贴一次标签——标签可能没变，也可能换了一类，比如上回是「记错」，这回发现其实是「不懂」。那也正常，接着对着新标签修就是。

错题不是废纸，是下一轮的入口。

## 4. 自己跑一遍：一道错题完整的调试流程

光说四类标签还不够。这一节给你一条能自己从头跑到尾的流程，四步：**归类 → 找缺口 → 对着修 → 隔几天重做验证。** 拿一道你最近做错的题，照着走一遍。

**第一步，归类。** 别先骂自己「太粗心」。先拿四个问句过一遍：

- 题干条件有没有看岔？
- 这一步为什么这么做，我讲得出来吗？
- 用到的公式、概念、步骤，有没有记错？
- 考试时间里，是不是来不及做完？

哪个问句一问，你心里「咯噔」一下——那一类，多半就是它。只贴一个标签，先别贪多。

**第二步，找缺口。** 标签定了，就顺着上一节那张表，把缺口具体定位到点上。

- 归到「看错」，就翻回原题，把看岔的那个字、那个符号圈出来。缺口就是那一个点，不是「我整个人不认真」。
- 归到「不懂」，就找出你讲不出来的是哪一步。一直往回退，退到你能自己做对的最小一步为止——就是不查资料、自己就能独立做对的那一步。讲不出的那一段，就是缺口。
- 归到「记错」，就把你写的和课本上的并排放，圈出差在哪个字、哪个符号。
- 归到「没时间」，就把这类题里最拖时间的那一步找出来。

**第三步，对着修。** 缺口找准了，就照第二节表格里那一类的修法走，别越界去修别的。看错就练「读完题先停一秒确认条件」；不懂就从能自己做对的那一步往上、一步一步把中间补回来，每补一步都要能讲出为什么；记错就重背加默写核对；没时间就把最慢那一步单独练熟、掐表提速。一次只修一类，别想着一道题把四种病全治了。

**第四步，隔几天重做验证。** 这一步最容易被跳过，可它才是判断修没修好的唯一办法。当天就记下重做时间——隔三五天比较稳，太近了你还记着答案，验不出真假。到点了，盖住答案，从头独立做一遍：

- 做对了，这道题这一轮的调试就算收工，标个「已修好」。
- 还是错，别灰心，回到第一步重新归类。有时你会发现标签换了——上回当成「记错」，这回才看出根上是「不懂」。那就对着新标签再修一轮。

这条流程你完全可以自己跑，不需要任何人盯着。一道题、一处缺口，对着修，再隔几天复查一次——这套跑顺了，自我调试的基本功就握在你手里了。

## 5. 这件事当天怎么做

今天，从最近一次作业或试卷里挑三道错题。一道一道来，同样一次只盯一道（原因见第二章）。

每道题，照着前面那条流程走，填这四项：

| 题号 | 粗标签 | 下一轮重做时间 | 重做结果 |
|---|---|---|---|
|  | 看错 / 不懂 / 记错 / 没时间 |  | 对 / 仍错 |
|  | 看错 / 不懂 / 记错 / 没时间 |  | 对 / 仍错 |
|  | 看错 / 不懂 / 记错 / 没时间 |  | 对 / 仍错 |

填完就停。这一步看着小，可正因为它小，才扛得住天天做。

错题本之所以失败，多半不是因为孩子不会写漂亮步骤，而是每道错题都被当成一次情绪事件来处理——今天懊恼一阵，明天就忘了，下次考试又在同一个坑里栽下去。

把它压成这四项，不是图省事，是为了留下一条能复查的痕迹：哪道题、哪类毛病、哪天重跑、跑出什么结果。没有这四项，错题本再厚也很难真正进入下一轮。

你不必在当天就把自己所有的问题都想明白。当天能做到的，是把这道题从「我又错了」，变成「它现在有一个标签，也有一个下次再见的时间」。别写五百字反思，也别把一道今天的错题，扩成一整套改造计划。本章要练的，就是这么一个小动作：看见错题，先定位是哪一类，再对着那一类修，然后安排一次重做来验证。把这个动作稳稳做起来，错题本才开始真正有用。

## 6. 给家长的一段

这一环，家长能看到的信号是：孩子错题本抄得整整齐齐，同类题却反复错；一问错在哪，只会说「粗心」「没注意」「下次会了」；对错题明显烦躁，只想赶紧把订正写完交差。

该说的一句话：「这道题先别急着解释，先归一类——看错、不懂、记错，还是没时间？归完了，咱们对着那一类改，过几天再盖住答案做一遍看看。」

别说的一句话：「你怎么又这么粗心，错题抄十遍。」

家长不用去判断学科上的深层原因。你只盯一个动作：孩子有没有给错题贴上标签、对着那一类修、并安排一次重做来验证。能稳定做到这三步，方向就对了。

## 7. 这一章给到哪、剩下的归谁

把话说明白：通用的方法和能自己跑的流程，这本书都给你了。照着本章，你一个人就能完成这些——

- 把错题归到四类标签之一。
- 找到每一类的缺口在哪。
- 对着这一类用通用办法去修。
- 隔几天盖住答案重做，验证修没修好。

这一整套「归类 → 找缺口 → 对着修 → 隔几天重做验证」，你一个人就能跑通，跑通了就能解决掉大半错题。

剩下的两件事，不是看一本书就能替代的，那是问分在做的：一是针对你这一个人的持续诊断——同样是「不懂」，为什么偏偏是你、偏偏在这一类题反反复复栽，你的薄弱环节具体卡在哪条链子上，这得持续看着你才诊得准；二是按周盯着你、动态调整的长期陪跑——这周练什么、下周节奏要不要变、哪个毛病压下去又冒出来，这得有人一周一周跟着你调。

除了这两件，别的都在书里了。通用方法和自查流程，照着做就行；针对你个人的持续诊断和按周陪跑，才是问分的事。

# 第五章：内存与缓存——为什么背了就忘，以及怎么让知识在考场上取得出

**本章结论：背了就忘，很多时候不是没存进去，而是取不出来。本章拆穿「死记硬背最保险」这个迷思，帮你养成三个习惯：合上书自己默一遍、隔几天再默一次、把背错的地方挑出来改掉。核心动作就一句——把「重读」换成「合上书自测」，隔几天再测一次，把记错的内容圈出来改对。分数是这套读写习惯跑顺之后的结果，不是背得多就保证考得好。**

脑子有点像电脑：背进去的先存在「内存」里，可考场要的是随手就能取出来的那层（电脑里叫「缓存」）。下面就用「存进去/取得出」这两个词，把这件事讲清楚。

先拆穿一个迷思：背得多，就一定记得住。

最常见的情形是这样：一页笔记读了五遍，翻开时哪句都眼熟，可一合上书，只写得出两三条；考前刚背过，进了考场一紧张，脑子里空一片；等看到答案又拍大腿——「对，我当时明明会」。这其实不保险，它只是把「认得」当成了「能取出来」。考场不要你看见材料时觉得熟，它要的是你手边什么都没有，照样能把东西取出来。

## 1. 真实困境：背过了，考场取不出

很多学生复习时只做一件事——重读。读课本，读笔记，读答案，读错题。读的时候很顺，顺到最后，人就以为自己记住了。可一合上书，问题立刻露馅：概念说不全，公式差一个条件，步骤只记得开头，关键那句话怎么也写不出来。

这不是简单的「没背熟」。更准确地说，是**取不出来**——东西也许是写进脑子了，但考试那一刻就是读不出来。

## 2. 「存进去」和「取得出」是两件事

学习里的记忆，可以拆成两个动作：一个是把东西写进去，一个是再把它读出来。你听课、划线、抄笔记、重读课本，多半是在「写进去」；真正练「读出来」的，是你合上书默写、自问自答、把一段内容讲给别人听这些事。

考场说白了就是一次限时的「读出来」。能一下子就取出来的，才算真的存稳了；得翻一眼才想起来的，其实还没到考场能用的程度。所以**重读的时候手感顺，不等于记住了**——手感顺只说明你认得这一页，不说明你取得出来。

这里有个很关键、又最容易混的区别：**「认得」和「取得出」差得很远**。「认得」是你看见原文时觉得熟，「取得出」是原文不在眼前，你还能把它写出来、说出来、用出来。很多复习的假努力就卡在这儿——材料一摆出来，大脑马上说「我见过」；材料一拿走，答案就接不上了。偏偏考场拿走的就是材料，所以光练「认得」，风险很大。

判断有没有真学会，有个简单标准：不查任何资料，你能独立做对、或独立默写出来的那一步——这就是第2章说的「能自己做对的最小一步」。能稳稳做到这一步，才算真的摸到底。

还有一种情况更危险：你倒是取出来了，可取出来的是错的。公式背串了，概念的边界记反了，早就过时的旧说法还赖在脑子里——这就是**「背了，但背错了」**。这种比「想不起来」更坑人，因为你会很自信地写出一个错版本。记错的东西不挑出来改掉，越熟越危险。

## 3. 当天动作一：先合上书自己默一遍

第一步，把「再读一遍」换成「先测一遍」。今天挑一页内容，合上书，从下面三种里选一种做：

| 怎么测 | 当天怎么做 | 留下什么 |
|---|---|---|
| 合书默写 | 盖住材料，把核心点写出来 | 一段默写 |
| 自问自答 | 先给自己出个问题，再不看材料回答 | 一问一答 |
| 讲给别人 | 不看材料讲一遍，讲不顺的句子标出来 | 一条卡点 |

别一上来就追求全对。先看自己取不取得出来——取不出，才知道该回去补哪一句。这跟第3章说的判断标准是一回事：能讲给别人听懂，才不是只在自己脑子里觉得熟；也对得上全书反复讲的那条线——连续 N 次独立做对、或者能讲给别人听懂，才算过关。

## 4. 当天动作二：隔几天再测一次

当天测一次还不够。记忆最会骗人的地方，恰恰是今天刚看完、感觉最顺的时候，所以要故意隔开时间再测。

起步的节奏可以很简单：

| 时间 | 做什么 |
|---|---|
| 学完当天 | 合书自测一次 |
| 隔 1～2 天 | 再测一次 |
| 隔约一周 | 再测一次 |

做对了，下次就把间隔拉长；卡住了，下次就缩短。隔开时间再测，目的不是把日程表排满，而是趁你已经有点生疏了，看看这块内容离开材料后还取不取得出来。当天会，不代表三天后还会；三天后还会，才更接近考场上能用的状态。

这只是个通用的起步节奏，不是什么精确的遗忘曲线公式，也不必照搬死守。

## 5. 当天动作三：把背错的挑出来改掉

「背错了」这件事最容易被忽略，因为它不像「想不起来」那么扎眼——你会顺顺当当地写出一个错版本，自己还浑然不觉。挑错改对，就两步：

1. 合上书，把核心概念默写出来。
2. 跟原文一对，凡是对不上的地方圈出来，查清楚它本来该是什么样，再重背一次。

这跟第4章里「记错」那一类错题是同一个道理：背错了，别再凭印象硬背，要回去查它该长什么样。

今天就拿下面这张小表，跑一页内容：

| 核心点 | 合书能写出吗 | 是否记错 | 下次重测时间 |
|---|---|---|---|
|  | 对 / 卡住 | 对 / 记错 / 想不起 |  |
|  | 对 / 卡住 | 对 / 记错 / 想不起 |  |
|  | 对 / 卡住 | 对 / 记错 / 想不起 |  |

完整空白卡见[附录F·提取练习与间隔重复设计卡](../appendix/附录F_提取练习与间隔重复设计卡.md)。照着这张卡，你自己就能跑一页内容。

## 6. 你自己就能跑的一轮自我诊断

前面三个动作分开看都简单。把它们串起来，你一个人在家就能跑通一整轮诊断，不用等任何人。

具体怎么跑，照第4章那套自查环跑，这里把对象换成默写/提取：合上书默写一页内容，对着原文核一遍，先归类（是「取不出来」「背错了」还是「只是认得」），再找缺口（把卡住的那一句具体圈出来），对着修（取不出来的练「读出来」，背错了的查回正确版本并排对照，只认得的把材料盖死重测），最后隔几天重做验证（隔 1～2 天、再隔约一周重测同样几个点，还能独立取出来才算真修好，又卡住就回去重判一次）。

能跑通一轮，你对自己「哪种忘、忘在哪、修没修好」就已经心里有数了。

## 7. 给家长的一段

这一环，家长能看到的信号挺明显：孩子反复读课本、重抄笔记、划线划得密密麻麻，看着特别用功；可一合上书问三条核心点就卡住；考前一晚临时抱佛脚，第二天照样想不起来；嘴上说「我都看过了」，手上一个字默不出来。

一句该说的话：「别再读了，合上书，先默写三条核心，写不出来的再回去查。」

一句别说的话：「多读几遍背熟就行了，背不下来就是没用功。」

家长不用去判断孩子该背哪一段，只盯一个动作就够了：他有没有把「重读」换成「合上书自测」。

## 8. 把这一章收个尾

这一章的方法，连同你一个人就能跑的那套自查流程，都在前面写齐了：

- 合上书自己默一遍（也就是主动逼自己把东西取出来）；
- 隔几天再测一次（间隔重复的起步节奏）；
- 把背错的挑出来改掉（自查并改对）；
- 还有第6节那一轮自我诊断（照第4章那套自查环跑，把对象换成默写/提取）。

照着这些，你自己就能跑通这套方法，也能完整跑完一轮自我诊断。

真到自己反复卡在同一类点上、默写总取不出、薄弱链条始终找不准，这一科可以交给问分，帮你把卡住的那条链拎出来。

## 9. 避免玄学的硬要求

本章不讲神奇记忆术，不讲看一遍就永远忘不掉，也不把「隔几天再测」包装成什么必胜公式。

每一个说法，都得落到一个真能做的动作上：合书默写、隔几天再测、把记错的圈出来、查回正确版本。落不到动作的话，删。

# 第六章：进程调度——时间和注意力是 CPU，别让它一直在切换中空转

**本章结论：全都想抓，不等于学得更多；多学一点，也不等于学得更深。本章要建起的能力是：一次只做一件事，用「失分频率×可提升空间」先修最该修的。核心动作很简单：下一个番茄钟只做一件事，被自己中断一次就划一笔正字；再照[附录G·注意力调度优先级表](../appendix/附录G_注意力调度优先级表.md)，把要做的事排个顺序，先修排最前的一两项。**

进程调度，说白了，就是给任务排先后；CPU，就是电脑里真正干活的那块芯片，一次只能认真跑一个任务。借这个说法，不是为了讲电脑，是为了把你每天「忙了很久却没推进」这件事看清楚。

## 1. 先拆穿一个迷思

很多人以为：全都想抓，才叫努力；多学几科，才算没浪费时间。

现实经常反过来。桌上五科一起摊开，一会儿写语文，一会儿背单词，一会儿又想起数学还没做。手机摸了十几次，科目换了七八次，台灯下坐了三小时，真正推进的没几页。时间不是没花，花在重新进入状态上了。

多，不一定快。很多时候是又慢又浅。

## 2. 真实困境：你不是没时间，是一直在切换

最典型的晚上是这样：你先打开数学卷子，刚做两步，想起英语单词还没背；英语背了三分钟，又担心语文默写没复习；刚翻开语文，手机亮了一下，顺手看一眼，回来以后连刚才数学做到哪一步都忘了。

这时再回数学，不是接着刚才继续做。你要重新找题、重新读条件、重新想刚才卡在哪。这段重新热机的时间，就是上下文切换，换成大白话：注意力每换一次，都要重新把脑子搬进新状态。

切得越勤，真正干活的时间越少。你看起来一直在学，实际一直在启动、暂停、重启。

所以，「我效率低」「我坐不住」可以先改成一句更能处理的话：我把太多力气耗在来回切换上了。

## 3. 动作一：一个番茄钟只做一件事

番茄钟，就是一段你打算专心做一件事的时间，常用 25 分钟。不要先追求完美专注，先把切换看见。

下一次学习，照这个做：

1. 选一件事：一科、一类题、一个小任务。
2. 桌上只留这件事需要的材料。
3. 放一张纸在旁边。
4. 这 25 分钟里，每被自己中断一次，就划一笔正字。

什么算中断？

摸手机算一次。突然换去做另一科算一次。发呆回过神来算一次。起身找无关的东西算一次。

结束以后数几笔。不要骂自己，也不要写长反思。只记一件事：这一轮我被自己打断了几次。下一个番茄钟的目标，不是变成零，而是比上一次少一笔。

这个动作的价值，是把看不见的切换开销变成看得见的笔画。你不用做复杂记录，只数笔画就够。

数完以后，别急着给自己贴「自律差」的标签。先找最容易改的一处：手机是不是放在手边，桌上是不是同时摊着两科，水和笔是不是没提前准备好。下一轮只改一个小动作，比如手机放客厅、桌上只留一张卷子、开始前先把笔和草稿纸放好。能少一笔，就说明这次调度有进步。

## 4. 动作二：按「失分频率×可提升空间」排顺序

一次只做一件事，下一步就会冒出问题：先做哪一件？

别靠心情排。用两把尺子。

**失分频率**：这类题、这个知识点，最近是不是老错，错得多不多。

**可提升空间**：这一项现在补，好不好补，补了能不能很快见效。

两个一起看：又常错、又好提的，排最前面。很少错，或者短期很难补上的，先往后放。注意，这是排序方法，不是把一天按小时排满。你不是给自己塞满格子，你是在决定：下一个番茄钟先把注意力放到哪里。

下面是合成示范，不对应任何真实学生。

| 待修项 | 失分频率 | 可提升空间 | 这周先修顺序 | 先做的那一个动作 |
|---|---|---|---|---|
| 计算题最后一步总算错 | 高：最近一周错 4 次 | 好提：能先练验算 | 第 1 个 | 做 5 道同类题，每题最后单独验算一次 |
| 古诗默写错别字 | 中：最近错 2 次 | 好提：能当天默写核对 | 第 2 个 | 合书默写 3 首，错字单独重写 |
| 压轴题最后一问 | 低：偶尔做不出 | 难提：牵涉多块内容 | 先往后 | 先不抢今天第一个番茄钟 |

这张表示范的是排法，不是给所有人开的处方。完整空白表见[附录G·注意力调度优先级表](../appendix/附录G_注意力调度优先级表.md)。你把自己的失分项填进去，按同样两把尺子排。

没排在第一，不代表永远不做。它只是先不抢这个番茄钟。

排顺序时，宁可少排一点。一次只挑一两项，才真的能做完；一口气列十项，看起来很认真，最后往往又回到全都想抓。

别让待办清单重新变成压力清单。

## 5. 今天就跑一遍

今晚只跑一个最小闭环。

先从最近的错题、作业、默写里列出三四个失分项。照附录 G，把它们按「失分频率×可提升空间」排出来，挑排最前的一项。然后开一个番茄钟，只做这一项。中途被自己打断，就在纸上划正字。

结束时，只写三行：

| 今天排第一的事 | 这个番茄钟划了几笔 | 下次少一笔的办法 |
|---|---|---|
|  |  |  |

下次少一笔的办法要具体。比如：手机放客厅；桌上只留数学卷；做题前先倒好水。别写「提高自律」这种没法执行的话。

排出来的先修项，后面可以带进第7章的更新日志：这周我先修了哪一项，中断从几笔压到几笔，下周继续试什么。到那里再谈迭代，本章先把排序和数中断跑起来。

## 6. 给家长的一段

这一环家长能观察到的信号：孩子桌上同时摊好几科，书翻来翻去；学十几分钟就摸手机、刷一会儿视频；一会儿写语文，一会儿背单词，一会儿翻数学，没一科坐得住；坐了一晚上，问起来推进很少。

一句该说的话：「这一个番茄钟，咱就只做这一件事，手机先放客厅，被打断你自己划个正字，做完看几笔。」

一句别说的话：「你怎么一直玩手机，这么坐不住，给我都学了！」

家长不用替孩子排哪科先学，只盯一个动作：这个番茄钟里孩子是不是只做了一件事，有没有真的在数自己被打断几次，而不是又五科横跳。

## 7. 把这一章收住

一次只做一件事，按两把尺子先修最该修的，这套排法和数法都在前面写清楚了。你一个人就能照着排、照着数、照着把下一轮中断往下压。

真到自己怎么调都坐不住、优先级总排不准，问分能帮你把这一科的节奏拆开看。

本章不承诺分数。提分是注意力不再空转、失分项被持续修掉之后的结果，不是排完一张表就自动发生。

# 第七章：版本迭代——每周给自己发一个新版本，复盘是写更新日志

**本章结论：复盘不是反思一下，也不是写几句「下次更努力」感动自己。本章要建起的能力是：像发版本一样，一周给自己的学习方法更新一次，写清本版改动、修复的毛病、遗留问题、下版只试一件事，并用收敛判据判断哪一类可以收工。核心动作：今天开一份更新日志，按四栏填一版，再用[附录H·每周学习更新日志模板](../appendix/附录H_每周学习更新日志模板.md)每周续上。**

版本迭代，就是一版一版改；放到学习里，就是别每次都推倒重来，而是在上一版基础上只改一处，观察它有没有用。更新日志，就是把这次到底改了什么写下来，方便下次回头看。

## 1. 先拆穿一个迷思

很多人把复盘理解成「考完想一想」「错题抄一遍」「写一句下次要细心」。这些都可能有情绪价值，但不一定能让方法变好。

真正的问题是：下周你还会不会错同一类？你这周到底改了哪一处？这一处有没有起作用？要是这些都没留下记录，复盘就只是叹气，或者把自己感动一遍。

最常见的现象是：每次都复盘，每次都写「更努力」，一连几周错的还是同一类题。另一种更隐蔽：一不顺就整科推倒重来，前几周已经做对的东西也一起扔掉。

这不叫迭代。它只是重启。

## 2. 真实困境：你没有上一版，也没有下一版

学习方法很容易变成一团糊。今天觉得自己背得不够，明天觉得题刷少了，后天觉得时间安排有问题。每次都想改，每次都改一堆，过一周一看，不知道到底是哪一处让情况变好，也不知道是哪一处添了乱。

还有一种情况：你一道题做错，就从头翻课本；一类题没做稳，就把整章重学一遍。你没有回到最近一个做对的地方，而是把已经跑通的部分也扔了。

版本迭代要解决的就是这两件事：第一，别靠感受复盘，要留下更新日志；第二，别推倒重来，要回退到最近做对处，带着修正从那里重跑。回溯，用大白话说，就是退回最近一个你确实做对的节点，再往下走。收敛，就是练到稳了、不再忽对忽错，可以放下这一类。

## 3. 发版本：回退到最近做对处重跑

发一个学习版本，不是从头重学。

更好的做法是：先找最近一个你确实能自己做对的节点。这个说法接上[第2章·递归调用](../chapters/02_递归调用拆到会做.md)里的「能自己做对的最小一步」：不查资料、不看答案，你能独立做对的那一步。找到它以后，只带上这次要修的那一处，从那里继续往下重跑。

举个合成场景：一类数学大题，你第一步能读懂条件，第二步能列出式子，第三步变形总卡。那就不要从整章开始重学。你回到「能独立列式」这个节点，把第三步变形单独修，再从列式那里往后重跑。

这能保住前面已经做对的东西。回退还省时间：你不用把整章从头重学，只在真正卡住的那一点上花力气，前面跑通的部分直接接着用。学习最怕两种浪费：一种是错了不修，另一种是明明前半段会了，还把它一起推倒。

## 4. 硬规矩：一次只改一个变量

一版只动一处。

这个规矩很硬。你可以换一个解题方法，可以调一个学习节奏，也可以改一个习惯，但一版里只选一个。不要同时换资料、换时间、换练法、换错题本格式。

原因很简单：一次改五样，下周变好了，你不知道是哪一样起作用；下周更差，你也不知道是哪一样添了乱。一次只改一个变量，下一版回看才有判断价值。

所以更新日志里最重要的不是写得漂亮，而是写得能验证。下周回头看，只问一句：我只改的这一处，有没有让那个毛病少犯？

## 5. 每周更新日志四栏模板

每周只填一版。四栏固定，不要自己加一堆情绪栏。

| 栏目 | 填什么 |
|---|---|
| 本版改动 | 这一版我只改了哪一处。 |
| 修复的毛病 | 这一版把上一版哪个反复犯的毛病修掉了，或正在修。 |
| 遗留问题 | 这版没解决、留到以后的是哪一个卡点。 |
| 下版实验一项 | 下一版只试这一件事。 |

合成示范：

| 栏目 | 示例填写 |
|---|---|
| 本版改动 | 数学大题第二步变形前，先把已知条件重新圈一遍。 |
| 修复的毛病 | 第二步经常漏掉一个条件，导致后面式子错。 |
| 遗留问题 | 变形以后计算速度还是慢。 |
| 下版实验一项 | 下周只练变形后的计算验算动作。 |

这张表的用法很直接：一周后回看「漏条件」有没有少。如果少了，这版改动有效；还在漏，就把它继续放进遗留问题，下一版换一个更小的动作试。完整空白模板见[附录H·每周学习更新日志模板](../appendix/附录H_每周学习更新日志模板.md)。

## 6. 收敛判据：练到什么程度算够

很多人不会停。练了两遍就觉得会了，结果考试还错；另一类人一直不敢停，同一类题刷到考试前一天还在刷。

这里要用收敛判据。收敛，就是一类题练到稳了，不再忽对忽错，可以把主要精力挪走。硬定义是：连续 N 次独立做对，或者能讲给一个没学过这块的人也听懂。「讲给别人听懂」里的别人，得是没学过的人——你能把它讲到他点头，才说明你不是只在自己脑子里觉得会。N 自己定，建议 3 次起。满足任一条，就给这一类打一个「稳定版」标记，先收工。

这不是偷懒。没有收敛判据，复盘会变成死循环：错题喂进下一轮，下一轮再下一轮，永远不知道哪天该停。[第4章·调试错题](../chapters/04_调试错题当bug.md)讲错题喂下一轮，本章给停手线：该重跑的继续重跑，已经稳定的就别继续占用主要注意力。

今天就做两件事：开一份更新日志，按四栏填下这一版；再挑手头一类题，用收敛判据自测一次：连续独立做对几次了？能不能讲给别人听懂？能，就标稳定版；不能，就写进遗留问题，带进下一版。

## 7. 给家长的一段

这一环家长能观察到的信号：孩子每次考完或周末都说「复盘了」，但复盘就是对着卷子发呆、写一句「下次要更努力」；看不到他这周具体改了哪一处、修了哪个毛病；同一类题连着几周还在错，或者一不顺就把整科推倒从头来。

一句该说的话：「这周你只改了哪一处？写下来：改了什么、修掉了哪个毛病、下周只试哪一件事，下周我们对着看它有没有用。」

一句别说的话：「天天说复盘，怎么还错同一道？复盘有什么用，多做几套不就好了。」

家长不用懂这门学科，只盯一个动作：孩子有没有把「这周只改了一处、改了什么」写下来留痕，而不是空喊更努力或推倒重来。

## 8. 把这一章收住

一周发一版、写更新日志、一次只改一个变量、按收敛判据收工，这套方法前面已经写透了。你一个人就能照着一版一版往下迭代。

方法都在书里了，你自己就能一周周发版本；真到自己迭代两三轮还总卡在同一类毛病、判断不准这一版到底改好没有，可以让问分帮你看一眼。

本章不承诺分数。提分是方法一版版跑稳之后的结果，不是写完一张日志就自动发生。

# 第八章：自举与元认知——学习如何学习，让系统自己升级自己

**本章结论：学会了知识，不等于从此不用管自己怎么学。本章拆穿「学会了就不用管怎么学了」，要建起的能力是：把「我怎么学」本身也当成一道题，一轮轮调试升级。今天就对自己跑一遍元认知三问，再用五阶段自测现在在哪一档，照通用方向调一个动作；配套工具见[附录I·元认知三问与五阶段自测卡](../appendix/附录I_元认知三问与五阶段自测卡.md)。**

## 1. 先拆穿一个迷思

很多人以为，学会某个知识点之后，剩下就是多做、多背、多坚持，学习方法本身不用再管。这个迷思最隐蔽，因为它看起来很勤奋。

真实现象很容易验：这周某个方法明明没效果，你还因为「一直这么学」继续硬用；分数卡在一个台阶上很久，复盘时全在看题，却从没问过「我这套学法本身是不是该改了」。知识学会是一回事，会不会调整自己的学习方式，是另一回事。

## 2. 真实困境：方法用顺了，就没人再检查它

前面几章讲过拆题、编译、调试、提取、调度、迭代。很多学生读到这里会点头，觉得「这些我都懂了」。真正的卡点在后面：这些方法用了几周以后，谁来检查它有没有变形、有没有失效、有没有换场景就不灵？

自举，就是系统用自己升级自己。放到学习里，就是你用前面学过的同一套「拆、测、改」，去调试「我怎么学」这件事本身。自引用，就是拿上一轮的结论当下一轮的输入，比如「重读没用」这个发现，下一轮就变成「把重读改成自测」的材料。元认知，就是跳出来看自己是怎么学的。

一句大白话：学习如何学习，就是把「我怎么学」也当成一道题来调试。这是全书递归脊柱的收口：第2章把题拆到能自己做对的最小一步，第4章把错题喂回下一轮，第7章用回退和收敛判据发版本，本章把这套方法反过来用在学习方法本身。

## 3. 元认知三问：每问都要落到具体对象

三问不要空问。每次只挑一个对象：本周一个方法、今天一道题、最近一类错。问完以后，把答案写成一个能执行的改动。

| 三问 | 当堂填什么 | 问出后通用怎么调 |
|---|---|---|
| 我现在在系统的哪一环？ | 填本周卡住的这道题或这类失分，是卡在输入、编译、存储、调度、执行、调试，还是迭代。先按[第1章](../chapters/01_系统总览地图.md)定位。 | 定位到哪一环，就翻到对应章节跑当天动作。背了取不出，去[第5章](../chapters/05_内存与缓存.md)做提取练习；错题反复错，去[第4章](../chapters/04_调试错题当bug.md)先贴四类标签。 |
| 这个方法这周到底有没有用？ | 填本周一个具体方法，比如重读、刷套卷、错题重做、合书默写。用收敛判据看：连续 N 次独立做对／能讲给别人听懂。 | 有用就保留，间隔拉长；没用就别硬留，按[第7章](../chapters/07_版本迭代.md)一次只改一个变量，下周再测。 |
| 下一步我自己会怎么调？ | 写一句「下周我把 X 改成 Y」，只改一个动作。 | 把这个改动当成下一轮输入。做对了就保留，没动静就回退到最近做对处，换一个更小的动作。 |

这三问不是写反思作文。它们的作用是逼你留下证据：卡在哪一环、哪个方法有没有用、下一版只改什么。没有具体对象，就没有调试。

## 4. 五阶段自测：看清自己现在在哪一档

五阶段不是聪明排名，只是在描述你现在这套学习方法成熟到哪一步。每一档都能往上走，关键是把下一步落成动作。

| 阶段 | 一句话是什么 | 自测信号（你是不是……） | 往下一档的通用方向 |
|---|---|---|---|
| 模仿 | 主要靠照着别人步骤做。 | 你是不是例题会跟，换个样子就断，说不清每一步为什么？ | 开始给错题贴[第4章](../chapters/04_调试错题当bug.md)四类标签；讲不清的一步，按[第2章](../chapters/02_递归调用拆到会做.md)拆到能自己做对的最小一步。 |
| 调试 | 已经能看见自己错在哪一步。 | 你是不是会说「这题是不懂」或「这题是记错」，也能找出卡点？ | 把同一类反复错的题放在一起，不再只补单题，而是重写这一类题的做法，一版只改一个变量。 |
| 重构 | 开始改自己的学习方法。 | 你是不是会把「重读」改成「合书自测」，把「刷整套」改成「先拆题」？ | 每周固定跑一次元认知三问，检查这套新方法这周有没有用，并写进更新日志。 |
| 自我编程 | 能主动设计练习和节奏。 | 你是不是会自己安排练什么、怎么测、下周改哪一处？ | 把跑稳的方法迁移到新学科或新题型，用收敛判据确认它站得住。 |
| 进化 | 自我升级变成习惯。 | 你是不是遇到新卡点时，会自动问「卡在哪一环、这法子有没有用、下版改什么」？ | 保持循环，定期清掉没用的方法，把有效版本留下来，遇到新场景再跑一轮。 |

这张表最怕只用来给自己贴一个档。定档只是起点，真正有用的是后面那一列：下一步往哪儿使劲。

## 5. 合成示例：自己发现一个学习毛病

有个学生复习生物，一直用同一个动作：每天把笔记从头读三遍。他的感觉很好，读得顺，荧光笔也划得满。可一合书，三个概念之间的关系讲不出来；做题时看到相近选项，又开始凭印象猜。

他拿这套方法跑三问。第一问，卡在哪一环？不是输入，也不是努力时间不够，而是存储和提取：看过了，取不出来。第二问，这个方法这周有没有用？按收敛判据看，他不能独立讲给别人听懂，也不能连续做对同类题，所以「重读笔记」这一版无效。第三问，下周怎么调？他写成一句话：「下周我把每天读三遍，改成每天合书默写三条概念关系，再翻书核对。」

再用五阶段自测，他已经能发现方法没用，处在「调试」往「重构」走的位置。下一步不是把笔记读得更久，而是改学习方法本身：从「看得顺」改成「取得出」。一周后，他只检查这一个改动：合书默写是不是更容易说清关系、同类题是不是少靠猜。有效就保留，无效就回到最近能讲清的一小步，再换更小的动作。

这就是自引用：上一轮发现「重读没用」，下一轮就把它当成输入，改成自测。流程并不复杂，难点是你愿不愿意承认：有些老办法已经不产出结果了。

## 6. 给家长的一段

这一环家长能观察到的信号：孩子一套学法用了很久从不调整，分数卡在一个台阶上很久；你问起，他说「我一直就这么学的」；很少听他讲过「这个方法这阵子没用了，我打算换个法子」。

一句该说的话：「这阵子哪个学法你觉得没以前管用了？这周换一个动作试试，下周咱们只看这一处有没有变化。」

一句别说的话：「方法有什么好改的，你就是不够努力，照老办法多做点就行。」

家长不用懂学科，只盯一个动作：孩子会不会偶尔停下来调一调「自己怎么学」，而不是一套老办法用到底、卡住了也不换。

## 7. 把这一章收住

把「我怎么学」也当成一道题来一轮轮调试，这套方法书里已经给透了：三问怎么问，问出来怎么调，五阶段怎么看，下一档怎么够，你一个人就能照着跑。提分是方法跑稳之后可能出现的结果，不是写完一张表就自动发生。

真到一个人怎么调都上不去那个台阶，问分能陪你盯着这一科再往上够一档。

本章只管平时怎么调试自己的学习方法；三年规划看[《别等出分才开始》](../../新疆高中三年升学规划书/README.md)，志愿填报看[《别把孩子的分数浪费在志愿表里》](../../新疆家长志愿风险识别手册/README.md)。

# 第九章：给家长的运维手册——你是供电和散热，不是超频

**本章结论：你不用懂学科，也不用替孩子做题。你的活是把孩子学习的运行环境维护好：稳定供电，意思是情绪、后勤、信任别断；做好散热，意思是把家里的焦虑降下来；不超频，别一味加压加时长；不替 CPU 运算，别代替孩子思考。核心动作就一句：盯住环境，把手从题上拿开。**

## 1. 先点破一个家长误区

很多家长的直觉是：我越使劲，孩子提分越快。于是你坐旁边盯题、盯分、盯手机，临时加卷子，替他整理错题，替他决定先学哪科。

可你可能已经见过反效果：你一坐到旁边，孩子写得更慢；你把错题本整理得整整齐齐，下次同类题他还是不会；你越催，他越烦，最后学习变成你们俩对抗。问题不在于你不关心，而是你把力气用到了题上。家长该使劲的地方在环境上，不在孩子脑子里那一步推导上。

还有一种更隐蔽的场景：你陪读时，孩子明明平时能做的题也开始写错，草稿纸越擦越乱；你一离开，他反而能安静把前几步跑完。不是孩子故意和你对着干，而是你站在旁边那一刻，学习现场多了一个「随时会被评价」的压力源。你越想把他按进正确步骤，他越忙着防备你的反应，真正用于思考的那点力气就被挤走了。力气用错地方，关心也会变成干扰。

## 2. 把系统隐喻翻译成家里动作

孩子才是那台真正跑学习循环的机器。前八章写给孩子，是让他自己学会拆题、编译、存储、调度、执行、调试、迭代。家长的位置不是上去替他跑，而是做运维：让机器能稳定工作。

供电，就是情绪、后勤、信任这三路电别断。在家里具体就是：吃睡尽量稳，桌面尽量安静，孩子说自己在修一个问题时，你别每天用分数把他打回原点。电一断，机器就重启，前面刚搭起来的节奏全丢。

比如孩子考差那天晚上，原本已经愿意说「我这次主要是时间不够」，你一句「早跟你说过别玩手机」，他很可能立刻闭嘴。那一刻最该做的不是追责，而是把话缩短：「先说一件，这次最卡的是哪一步。」你把入口留小，孩子才有可能继续讲；你把入口扩大成整个人的问题，他只能防守。

散热，就是降焦虑。孩子已经因为错题、排名、作业热起来了，家长再把自己的焦虑倒上去，机器只会更慢。在家里具体就是：先把话说短，先问卡在哪一步，别在孩子卡住时连带翻旧账。

散热最常见的失败，是家长自己先急起来：孩子刚说「我不会」，你脑子里已经跑到中考、高考、以后怎么办。接下来你说出口的就不是眼前这道题，而是一整串后果。更稳的做法是先把后果放下，只问眼前：「现在是读题没看懂，还是某一步接不上？」焦虑被压回到一步，孩子才有空间处理一步。

超频，就是给机器强行加压、加时长、加任务。放到家里，就是一考差就加一套卷、报一个班、熬到更晚。短期看起来转得快，长期更容易崩。

替 CPU 运算，就是你替孩子思考。CPU 是电脑里真正干活的那块，放到学习里，就是孩子自己的脑子。你直接讲答案、替他归错题、替他决定本周先练什么，看起来帮了忙，实际把他自己那块「处理器」闲置了。

## 3. 家长该做／不该做对照表

这张表先列最常踩的坑。照着把几条做稳，就已经能少很多冲突。

| 场景 | 该做（供电/散热） | 不该做（超频/替孩子算） |
|---|---|---|
| 情绪 | 先问「卡在哪一步」，话说短。 | 一上来骂态度、骂分数、骂别人家孩子。 |
| 后勤 | 固定吃饭、睡觉、桌面、手机摆放。 | 临时加任务，孩子刚坐下又被你打断。 |
| 信任 | 让孩子说本周自己在修什么。 | 每天追问「你到底有没有学」。 |
| 一道不会的题 | 问他哪一小步会，哪一步断。 | 直接讲答案，或说「这么简单都不会」。 |
| 一次考砸 | 只挑一个最影响的 bug 说清。 | 把所有科、所有旧账一次翻出来。 |
| 报班加量 | 先看现有问题有没有被定位。 | 一焦虑就加课、加卷、加到睡眠被挤掉。 |
| 错题本 | 让孩子自己归类，你只看有没有留痕。 | 替他整理得很漂亮，替他下结论。 |
| 家长焦虑 | 自己先停十分钟，再开口。 | 把「我怕你以后吃亏」变成当场施压。 |

家长不需要判断这题到底怎么做。你只判断一件事：我现在是在供电散热，还是在超频、替他算。

有一个很实用的自测：你说完一句话以后，孩子更愿意继续讲卡点，还是更想把本子合上？前者多半是在供电，后者多半是在断电。你不需要把自己变成温柔老师，也不用把所有话都说得漂亮；先把最伤人的三句话停掉，把「你怎么又这样」换成「卡在哪一步」，家里的学习对话就会先降一档火。

## 4. 话术替换：七环节逐环换嘴

下面这张表把前八章翻成家长能说出口的话。bug，就是一做就出错的那一处；你问的是「卡在哪」，不是「你怎么又不行」。

| 环节 | 孩子常见表现 | 别说这句 | 该说这句 |
|---|---|---|---|
| 输入 | 上完课说不清今天学了什么。 | 「你上课到底听没听？」 | 「今天先说三条新东西，说不全就翻第1章那张表定位。」 |
| 编译 | 听懂了，自己一做就卡。 | 「懂了还不会，就是没认真。」 | 「先合上书讲一遍，讲到哪句卡住，我们就标哪句。」 |
| 存储 | 反复读，合上书写不出。 | 「多读几遍自然会。」 | 「先合上书写三条，写不出再翻书查。」 |
| 调度 | 学一会儿摸手机，多科横跳。 | 「你怎么这么坐不住！」 | 「这 25 分钟只做这一件，手机先放这，结束看被打断几次。」 |
| 执行 | 平时会，限时写不完。 | 「怎么又磨蹭到没时间？」 | 「哪一步最慢？先把那一步单独练顺。」 |
| 调试 | 错题抄了，同类还错。 | 「你怎么又错，抄三遍。」 | 「这道题先归一类：看错、不懂、记错，还是没时间？」 |
| 迭代 | 复盘只写「下次认真」。 | 「反思有什么用，分还这样。」 | 「这周只改了哪一个动作？下周先验这一处。」 |
| 元认知 | 一套老办法用很久，没效果还继续用。 | 「方法有什么好想的，多做就行。」 | 「这周哪个学法没用了？下周换一个动作试试。」 |

遇到不会的题，可以回指[第2章](../chapters/02_递归调用拆到会做.md)那句「拆到能自己做对的最小一步」。遇到错题，可以回指[第4章](../chapters/04_调试错题当bug.md)的四类归因。你不用重教方法，只负责把话从刺激孩子，换成帮他定位。

这张表最重要的不是「说得高级」，而是把家长的本能反应往后撤半步。孩子不会、做错、考砸的时候，你最容易抢着先开口——先忍住，让他把卡在哪说出来。你少抢一步，孩子才有机会把自己的那一步说出来。

## 5. 家庭运维例会：一周一次，十几分钟

家庭例会不是查岗会。每周一次，15 分钟以内，固定时间。孩子先说，家长主要听和记；会上不讲题、不翻旧账、不临时加任务。完整可填版在[附录J·家长运维对照表与家庭运维例会模板](../appendix/附录J_家长运维对照表与例会模板.md)。

为什么一定让孩子先说？因为真卡点通常藏在孩子自己的几句话里：他会说「我不是不会，是写到第三步就慌」，也会说「我知道该背，就是一合书就空」。家长先开口，往往会直接给结论，孩子就只剩点头、解释、辩解三种反应，真正的卡点反而出不来。你只记，不是装沉默，而是在给他留出把问题说完整的时间。

孩子说完以后，你可以只复述一句：「我听到的是，下周先改这一处。」这句话比立刻提建议更有用。它会让孩子确认自己被听见，也会把本周的动作压回到一个点上。动作越少，越容易真的做完。下周回看时，也只看这一处有没有变化，别把短会扩成全科总结。

第一次开会最容易翻车的地方，是家长忍不住把小会开成审判会：孩子刚说一个 bug，你马上追问为什么早不说；孩子刚说下周只改一个动作，你又加三件「顺便」。稳住的办法很简单：会前先写一句提醒放在手边：「今晚不讲题、不翻旧账、不加任务。」忍住这三件事，第一次小会就算成功一半。

| 环节 | 谁说 | 说什么 |
|---|---|---|
| 本周最卡 | 孩子 | 这周哪一科、哪一类题最难受。 |
| 本周修好 | 孩子 | 我修好了哪个 bug，怎么知道它比上周好。 |
| 还卡着的 | 孩子 | 哪个 bug 反复卡，自己拆不动。 |
| 下周一件事 | 孩子定，家长记 | 下周只先改哪一个动作。 |
| 后勤一件事 | 家长 | 我这周帮你做哪一件环境上的事。 |

会后只留一张小表，不写长总结。表越短，越容易下周继续开。先保证能坚持，别追求好看。

| 本周孩子说卡在哪 | 家长这周供电散热做得如何 | 下周环境上只调一件事 |
|---|---|---|
|  |  |  |

硬纪律再说一遍：例会上家长不讲题、不判对错、不把小会变成批评会。你说得越多，孩子越少说；孩子说得越少，真正的卡点越藏起来。

例会开不下去，先把它缩短。十分钟也行，甚至五分钟也行，只保留三件事：孩子说一个已经修好的 bug，一个还卡着的 bug，下周只先改一个动作。家长只记，不点评。这个形式跑稳以后，再慢慢加后勤项。短会的价值不在仪式感，而在固定给孩子一个「说卡点不会立刻挨骂」的窗口。

## 6. 把这一章收住

你的活说到底就八个字：供稳电，做好散热；别超频，别替他算。这些你照着这一章就能做起来，不用懂任何一科。提分是孩子那套学习机器跑稳之后可能出现的结果，不是你多催几句就自动发生。

你把供电和散热做稳就够了；针对孩子个人的持续诊断和长期陪跑，是问分的事。

本章只管平时学习环境；三年规划看[《别等出分才开始》](../../新疆高中三年升学规划书/README.md)，志愿填报看[《别把孩子的分数浪费在志愿表里》](../../新疆家长志愿风险识别手册/README.md)。

# 第十章：AI 是你的副驾，不是你的大脑

**本章结论：AI 可以做副驾，不能做你的大脑。它能帮你整理、查漏、讲解、出题、陪你复现错误；不能替你跑循环、替你思考、替你做诊断决策。核心动作：今天把一句「替我做」的提示词（就是你怎么问 AI 的那段话），改成「陪我做，最后我自己做对」，再用[附录K·AI 安全提示词对照表](../appendix/附录K_AI安全提示词对照表.md)跑一遍。**

## 1. 先拆穿一个迷思

有了 AI，还学什么？问它不就行了。

这个想法最危险的地方，不是懒，而是太顺手。你把题甩给 AI，它给完整步骤；你点头觉得看懂了，换一道同类题，又卡在第一步。你让 AI 总结一章重点，第二天合上页面，一条也想不起来。循环一直是 AI 在跑，你只是在旁边看。

更常见的画面是：一道难题问 AI，它把条件、公式、步骤、答案全铺好，你跟着读的时候每一句都顺。到了第二天，你遮住它的回答自己写，还是卡在昨天那一步。还有人一晚上问了十几次 AI，聊天记录里全是解释和笔记，真正不靠它独立做出来的题没几道。这种忙很容易骗过自己，因为屏幕上一直有产出，可产出不在你脑子里。

还有一种假进度更隐蔽：你让 AI 把一章内容整理成很漂亮的框架，标题清楚、层级完整，看上去比自己的笔记强很多。可一合上屏幕，让你说出三条核心点，你还是空白。那一刻要承认，整理发生在 AI 那边，提取没有发生在你这边；页面变整齐，不等于知识变成了你能取出来的东西。

## 2. 真实困境：你看见的是它会，不是你会

以前抄答案，至少还要翻书、抄步骤；现在把题拍给 AI，答案来得更快，解释也更顺。问题没变：它把你真正卡住的那一步替掉了。第4章说过，错题不是用来罚抄的，是用来定位 bug 的；bug 就是一做就出错的那一处。AI 直接给答案，最容易把那个 bug 盖过去。

最典型的误用，是把整张卷子拍给 AI，让它逐题讲完，再把步骤抄回本子。表面上作业完成了，错题也订正了，实际你没有经历读题、拆步、判断、验算这些动作。AI 讲得太顺，你就会把「它解释清楚了」误当成「我会了」。用 AI 不是问题，越权用才是问题；它一旦替你跨过卡住那一步，bug 就被盖住了，下次考试还会在同一处露出来。

真正安全的用法，是让它停在你卡住那一步前面：它可以把这一步讲清楚、把思路提示给你，但真正动手做对的那一下，得你自己完成。省掉这一下，省掉的就是真正的学习。

还有一种常见情况：你让 AI 帮你挑毛病，它顺着你夸了几句，再给一堆泛泛建议。听起来舒服，真正该改的那一步没被揪出来。AI 不是不能挑错，关键是你要把它限制在具体任务上：只看这一步，只挑三处，只问我哪里讲不通。

副驾，就是坐在旁边帮你看路、递工具、提醒你别错过路口的人；开车的还是你。放到学习里，AI 可以帮你整理材料、换种说法讲概念、出变式题、陪你复现错误。AI 降级，就是把它放回工具位，不让它替你踩油门、定方向、下判断。提示词，就是你怎么问 AI 的那段话；提示词一越界，AI 就从副驾变成代驾。

判断它有没有越位，只看一条：AI 讲完、陪完以后，你能不能不查资料、不看它的回答，自己独立做对那一步。做不到，就是它替你跑了。

## 3. 副驾能干什么，不能交什么

| 副驾能干（手脚活） | 驾驶位不能交（方向盘） |
|---|---|
| 把零散笔记整理成结构清单。 | 替你判断这章到底会不会。 |
| 把难懂概念换三种大白话讲。 | 替你把不会的题拆到底。 |
| 按你给的考点出变式题。 | 替你完成变式题。 |
| 陪你复现错题里原来那步想法。 | 替你归因到底是看错、不懂、记错，还是没时间。 |
| 根据你给的错题记录做粗分类表。 | 替你决定本周先补哪科。 |
| 提醒你遗漏了哪些材料。 | 替你下「真实根因就是 X」这种个人诊断。 |

AI 能不能信，不看它说得多顺，看它做完之后，你能不能回到[第2章](../chapters/02_递归调用拆到会做.md)那条线，找到能自己做对的最小一步，并独立做出来。

## 4. 安全提示词 vs 危险提示词

| 危险提问：让 AI 替你学 | 安全提问：让 AI 陪你学 | 差在哪一句 |
|---|---|---|
| 这道题怎么做？ | 我卡在这一步，别直接给答案，先问我两个问题，引我自己往下拆。 | 从要答案，改成陪我拆。 |
| 帮我总结这章重点。 | 我先默写这章重点，你对照我漏了哪几条。 | 从代替提取，改成先自测再查漏。 |
| 这篇作文写得怎么样？ | 别夸，挑三处最该改的，每处说清是哪类问题。 | 从求评价，改成找可修的点。 |
| 给我讲二次函数。 | 用三种大白话讲一遍，再出两道变式题考我，我做完你再看哪步错。 | 从听懂，改成讲解加陪练。 |
| 我这科为什么总考不好？ | 我把最近错题类型列给你，你只帮我整理分类，先补哪科我自己定。 | 从替我诊断，改成帮我整理材料。 |

通用心法很简单：凡是问完以后答案直接到手、循环是 AI 在跑，就危险；凡是问完以后最后那一步还得你自己独立做对，就安全。完整可复制版本见[附录K·AI 安全提示词对照表](../appendix/附录K_AI安全提示词对照表.md)。

用完 AI 以后，还要补一个验收动作：关掉回答，自己重跑一遍。能连续 N 次独立做对／能讲给别人听懂，才算这轮陪练有结果；只是听它讲得顺，不算过。这条接上[第7章](../chapters/07_版本迭代.md)的收敛判据，防止你把「AI 解释得很好」误当成「我已经会了」。

## 5. 今天就改一句提示词

今天只挑一件你平时习惯丢给 AI 的事：讲题、总结、挑错，选一个就行。先把原话写下来，再按下面三步改：

1. 删掉「直接给答案」「帮我写完」「替我总结」。
2. 加上「先别给答案」「逐步问我」「让我自己做出那一步」。
3. 跑完以后，合上 AI 的回答，自己把那一步独立做一遍。

示范：

| 原提示词 | 改安全后的提示词 |
|---|---|
| 帮我把这道题完整讲一遍。 | 我做到第二步卡住了。先别给完整答案，请问我两个问题，引我自己往下拆；我答完你再提示下一步。 |
| 帮我整理这章重点。 | 我先写出我记得的重点，你帮我指出漏掉的三条，并追问我能不能讲清。 |

AI 陪你把一个卡点照出来以后，这个卡点照样要喂回[第4章](../chapters/04_调试错题当bug.md)做归因，再带进[第7章](../chapters/07_版本迭代.md)的下一版。AI 是帮你照亮 bug，不是替你修完、验完。

## 6. 给家长的一段

这一环家长能观察到的信号：孩子一遇到不会就开 AI，复制题目、复制答案、抄到本子上，几秒钟一道；你问他懂了吗，他说「AI 讲过了」，换道同类又不会；AI 总结的东西存了一堆，没有一条是他自己默得出来的。

一句该说的话：「你让 AI 陪你做可以，但合上它的回答，这一步你自己能不查就做对吗？做不对，就让它别给答案，引你自己往下拆。」

一句别说的话：「别用 AI，用 AI 就是作弊。」

家长不用会用 AI，只盯一个动作：孩子用完 AI 以后，能不能合上它、自己独立做对那一步。能，就是副驾；不能，就是 AI 在替他学。

## 7. 把这一章收住

把 AI 用成副驾这套方法，书里已经给透了：能交给它什么，不能交给它什么，怎么把危险提问改安全，用完怎么验它有没有越位。志愿填报那本也有一章讲 AI 不能替家庭拍板，见[《别把孩子的分数浪费在志愿表里》](../../新疆家长志愿风险识别手册/README.md)，道理是同一个：AI 可以提示你查，不能替你确认。

AI 能陪你把毛病复现出来，但它没盯着你这一个人。真到自己反复卡在同一处、AI 也绕不过去，问分能接着帮你盯这一科。

提分是你自己把循环跑稳之后的结果，不是问一句 AI 就自动发生。

# 第十一章：交棒——方法跑通之后，规划与志愿在前方等你

**本章结论：看完方法不等于跑通方法，读完这本书也不等于学习这台机器已经开始运转。本章只做一件事：今天选一科，把输入、编译、存储、调度、执行、调试、迭代这一整圈走一遍，留下痕迹；然后看清自己站在平时学习、三年规划、志愿填报哪一段。**

先拆穿一个迷思：方法读懂了，不等于方法跑通了。

最常见的情况是，前面每章你都点头，可真让你拿一科从头跑一圈，到第二步就停了。你知道要拆出一个能自己做对的最小一步，知道听懂不算会，知道错题得拿来找原因，但这些动作从来没有在同一科里连成一次完整的运转。

这本书读完，不是结束。结束的标志不是「我都看过了」，而是：你能在一科上留下七个看得见的痕迹。

## 1. 真正的困境：零件都懂，整机没通电

很多学生学方法，是一章一章记的。今天记一个错题方法，明天记一个背诵方法，后天又记一个时间管理方法。每个零件单看都懂，可放回真实学习里，就是连不起来。

结果就是那几样老问题：错题还在反复错，背过的还是取不出来，一看就会一做就废，时间花了分却没动。

前十章像是把一台机器的各个零件都装好了。本章是第一次把它们接上电，整机跑一遍。

所谓一次完整的运转，意思是：你从一个具体的学习任务开始，走完七步，最后把这一轮做出来的东西，变成下一轮的起点。错题、给自己写的小结、判断「这块过没过」的标准，都不是终点——它们要喂给下一轮接着用。

这就是这本书反复说的那个「自己跑、跑完再跑」的循环。

## 2. 选一科跑通一圈

今天不要选全科，只选一科，而且只选一个小任务。比如一类函数题、一组英语短语、一个物理公式的某个用法。

| 环节 | 今天留下的痕迹 | 回指 |
|---|---|---|
| 输入（拆到会做） | 拆出一个能自己做对的最小一步：不查任何资料，就能独立做对的那一点。 | 见第2章《递归调用——拆到会做》 |
| 编译（看懂变会用） | 合上书，把一条规则复述出来，再说清它在题里怎么用。 | 见第3章《编译——学过≠会用》 |
| 存储（背了取得出） | 盖住材料默写一次，标出哪一处是「背了取不出来」。 | 见第5章《内存与缓存》 |
| 调度（坐得住时间） | 用一个番茄钟做这件事，注意力被自己带跑一次，就划一笔正字。 | 见第6章《进程调度》 |
| 执行（亲手做一遍） | 独立做一题，留下完整步骤，别让答案或 AI 替你做。 | 见第6章《进程调度》 |
| 调试（错题里找因） | 做错就先贴个粗标签：看错 / 不懂 / 记错 / 没时间。 | 见第4章《调试——错题不是用来罚抄的》 |
| 迭代（一轮喂下一轮） | 写一句小结：这一轮修了什么，下一轮要重做什么。 | 见第7章《版本迭代》 |

这张表不用填得漂亮，它只要能留下痕迹。

痕迹比感受可靠。你觉得自己努力了，机器不一定真跑过；可你留下了七个痕迹，至少说明这一轮没空转。

判断「这块过没过」的标准也在这里用上：同一类题连着好几次都能独立做对，或者你能把它讲给别人听懂，这一小块才算可以先停一停。停不是放弃，是说这一块暂时不用再占着你的主要注意力了。

## 3. 我站在产品线哪一段

平时学习跑通之后，前面还有规划和志愿。这三件事，本书不混在一起讲。

平时学习的这套方法，是问分线的起点：先自己把机器跑起来，真卡住了，再找问分做针对你个人的诊断。

| 你最近是不是…… | 你站在哪一段 | 这一段先看哪里 |
|---|---|---|
| 同一类题反复错，背了就忘，时间花了分还不动？ | 平时学习 | 本书；卡住时看[附录L·产品线交棒与服务边界入口](../appendix/附录L_产品线交棒与服务边界入口.md) |
| 选科、节奏、关键窗口说不清？ | 三年规划 | [《别等出分才开始》（三年规划）](../../新疆高中三年升学规划书/README.md) |
| 已经出分，要处理冲稳保、章程核验、志愿提交？ | 志愿填报 | [《别把孩子的分数浪费在志愿表里》（志愿填报）](../../新疆家长志愿风险识别手册/README.md) |

说白了，平时学习管的是每天学的东西怎么跑通。再往上一层，三年规划要操心的是高中三年怎么别踩那些回不了头的窗口。等到出分，志愿填报才上场，处理的是怎么在规则里把方案提交好。

本章只负责交棒，规划和志愿不在这里展开。

## 4. 何时该找问分

如果你只是读完了本书，还没亲手跑过一圈，先别急着说自己需要什么复杂方案。先跑，跑完再看卡在哪。

很多看起来「必须找人」的卡点，自己其实能先走一遍：照第4章那套自查环跑——归类→找缺口→对着修→隔几天重做验证，这里把对象换成「这一轮七个痕迹里哪一步没留下、或者留得很勉强」，那一步就是嫌疑最大的地方，缺哪补哪，一次只修一处，隔三五天拿同一类题重新独立做一次来验。

跑完这个小流程，下一步往哪走，方向也能心里有数：复述这关老过不去，就往「先讲后做、把规则讲清楚」上使劲；取不出来，就把重心挪到主动提取和间隔重做；做题总在某一步崩，就盯着那一步多攒几道同类题、把步骤拆细。这些都是通用方向，不挑人。

真到自己迭代几轮还卡在同一处，可以找问分一直盯着你这一科往下挖、按周陪你调。它会顺着两条线往下走：一条是针对你这一个人、这一科的诊断——为什么偏偏是你、偏偏在这儿反复犯，背后那条最薄弱的链条到底在哪；另一条是按周盯着你，根据你每周的实际情况动态调节奏、纠偏，一路陪到稳住，而不是自己启动一次就完事。

## 5. 给家长的一段

这一环，家长能看到的信号有这么几个：孩子方法听了一堆，却从没拿一科完整跑过这七步；一提规划或志愿就焦虑，想跳过平时学习，直接问报哪个学校、选哪个专业；做题留下的只有对错，没有复述、默写、找错、写小结这些痕迹。

该说的一句话是：「先把这一科按七步跑一圈，规划和志愿后面有专门的书，不用现在慌。」

别说的一句话是：「这些方法先别管了，赶紧想想报哪个学校、选哪个专业。」

家长此刻不用替孩子设计什么系统，只看一件事：今天有没有在一科上，留下这七个痕迹。

## 6. 避免玄学的硬要求

「整机通电」得落到七个看得见的痕迹上才算数，否则只是句说着好听的比喻；同样，「产品线地图」要能让你当场判断出，自己现在到底是在平时学习、三年规划，还是志愿填报，而不是停在一句口号上。

至于分数，本章不做承诺——提分是机器跑通之后才会出现的结果。本章里没有真实学生案例，涉及的场景都是脱敏或合成的。

# 附录 C：递归拆解卡

这是配合[第2章·递归调用](../chapters/02_递归调用拆到会做.md)使用的通用拆解卡。遇到不会的题，不是先抄答案，而是往下拆到你自己能做对的那一步，再往回拼；卡里只给框架与空格，拆出什么、拆到哪一层，由你照着填、自己判断。

## 卡一：一题拆到底主卡

先只挑一道今天真不会的题。每一层只写一句话，行不够就自己接着往下加。

| 位置 | 你要填什么 |
|---|---|
| 整道题（抄下你今天挑的这一道） | ______ |
| 第一层·这道题大致要做哪几步 | ______ |
| 我卡在哪一步（这一步我下不了笔/不确定） | ______ |
| 下一层 1·这一步又卡在哪个更小的、我已经会一点的东西上 | ______ |
| 下一层 2·继续往下拆 | ______ |
| 下一层 3·继续往下拆 | ______ |
| 一直拆到的能自己做对的最小一步（不查任何资料、不看答案，我自己能独立做对的那一步） | ______ |
| 往回拼（从这个最小一步往上，一层层接回去，整道题能不能走通） | ______ |
| 这道题真正卡住的最小一步是 | ______ |

拆几层不固定。拆到你确实能独立做对那一步为止；如果还要猜、还要看答案，就继续往下拆。

## 卡二：触底硬判据自查

判断有没有拆到底，不看感觉，看这三项。

| 这一步现在是什么状态 | 勾选 |
|---|---|
| 不查任何资料、不看答案，这一步我能独立做对（独立写出）。 | - [ ] 到底了 |
| 还要瞄一眼答案才敢写。 | - [ ] 没到底，回卡一继续拆 |
| 能写出结果，但讲不出这一步为什么成立。 | - [ ] 没到底，回卡一继续拆 |

拆到底不是拆到「看着最简单」，是拆到「你自己确实能独立做对」的那一层。每个人的那一层可能不同，这正常。

## 卡三：把卡点带进下一轮

拆出来的最小卡点，要喂进下一轮练习。错题怎么进入下一轮，见[第4章](../chapters/04_调试错题当bug.md)；练到什么时候停，见[第7章](../chapters/07_版本迭代.md)，连续独立做对几次、能讲给别人听懂，就算这一小步练通。

我这道题真正卡住的最小一步：______，下次先练它。

## 给家长的一段

信号：孩子一遇到大题或难题就卡住不动，半天不下笔；要么直接抄答案，要么一句「不会做」就整道跳过；同类题抄过一次，下次还是不会。

一句该说的话：「先别抄答案，这道题里哪一小步是你已经会的？从你会的那步，一点点往下拆看看。」

一句别说的话：「这么简单都不会？多刷几道不就会了。」

家长不用会做这道题，只盯一个动作：孩子有没有把题往下拆、找到自己会的那一步，而不是整道放弃或直接抄答案。

这套往下拆到能自己做对的最小一步、再往回拼的卡，书里都给全了，你照着就能自己跑一遍。

# 附录 D：编译通过判据清单与报错对照表

用途：配合[第3章·编译](../chapters/03_编译学过不等于会用.md)使用。学过一个知识点后，对照下面判据逐条勾；勾不满＝还没编译通过，别急着进下一题。

## ①「编译通过」硬判据清单

- [ ] 合上书和笔记，能把这个知识点或方法用自己的话复述一遍。
- [ ] 能讲给一个没学过的人听懂，对方能照着你的说法做对。
- [ ] 给一道改了数字或换了情境的变式题，能不查任何资料独立做对；这里对齐 能自己做对的最小一步 口径：能不查任何资料独立做对的最小单元。
- [ ] 能说出这个方法什么时候用，什么时候不能直接用。
- [ ] 能指出自己上一次在这一步崩在哪里，而不是只说「当时没想到」。

勾不满＝处于「源码看懂了、可执行文件还没生成」状态。回到第3章重新编译，别把「看懂」当成「会用」。

## ②「一看就会一做就废」报错对照表

| 报错现象 | 可能卡在哪（通用方向） | 当天动作 |
|---|---|---|
| 看例题秒懂，自己动笔卡在第一步 | 题没有拆到 能自己做对的最小一步 | 把这道题拆到能不查任何资料独立做对的最小单元；可接[第2章·递归调用](../chapters/02_递归调用拆到会做.md)和[附录C·递归拆解卡](附录C_递归拆解卡.md)。 |
| 跟着答案能写，合上答案就断 | 只读懂答案，没有生成可执行步骤 | 合书复述步骤，再做一道变式；对照上面的硬判据勾选。 |
| 平时觉得会，考场上想不起来 | 提取这一环不稳定 | 用自测代替重读；可接[第5章·内存与缓存](../chapters/05_内存与缓存.md)和[附录F·提取练习与间隔重复设计卡](附录F_提取练习与间隔重复设计卡.md)。 |
| 步骤大体会，数字、符号、条件老错 | 更像执行时的看错或记错 | 先按[第4章·调试](../chapters/04_调试错题当bug.md)和[附录E·错题调试归因四分法卡](附录E_错题调试归因四分法卡.md)归一个粗标签，再重做。 |
| 会做，但限时里总跑不完 | 熟练度和时间压力没有分开看 | 把这类题先拆到 能自己做对的最小一步 提速；不要用刷整套卷假装提速。 |

这张表帮你把「一做就废」定位到一个粗方向，并启动一个当天动作。

这套判据和报错对照，你自己就能照着跑。

# 附录 E：错题调试归因四分法卡

这张卡只做一件小事：每道错题先贴一个粗标签，贴完就停，不写长反思。先从最近一次作业或试卷里挑三道错题，记下题号、粗标签、下一轮重做时间和结果就行，先别急着上升到学科判断、训练安排或自我评价。卡片可以打印，也可以抄到错题本首页，当天就能用。配套用法见[第四章·调试错题当bug](../chapters/04_调试错题当bug.md)。

## 四类粗标签速查表

先把错题归类，再照「通用修法」走一遍。这四步谁都能照着做。

| 粗标签 | 一句话识别问句 | 通用修法（人人可照做） |
|---|---|---|
| 看错 | 我是不是把题干、符号、条件、单位、范围看岔了？ | 回到原题，把看岔的地方圈出来，重新完整读一遍，再确认题目到底要你求什么。 |
| 不懂 | 我能不能合上答案，把这一步为什么这样做讲出来？ | 退回到这一类里你能自己做对的最小一步（指不查任何资料、不看答案就能独立做对的那一步），把卡住的地方单独拎出来先弄懂，再往上接。 |
| 记错 | 这个公式、概念、步骤，和课本或笔记里的原样一致吗？ | 翻回课本或笔记，看它原本长什么样，当场重背一遍，别凭印象硬写；隔天再默一次，确认是真记住了，不是临时记。 |
| 没时间 | 这题我会做，但考试时间里没来得及做完吗？ | 这类题你其实会，缺的是熟练。把它拆成你能一口气做对的最小一步，反复练那一步，把卡顿的环节练顺，别靠刷整套卷假装提速。 |

## 自己能跑的一轮诊断

光贴标签还不够。下面这套流程，不用任何人帮忙，你自己就能跑一遍：先把这道题归哪一类，再找出到底缺在哪，照那一类的通用修法去补，过几天盖住答案重做一次，看修没修上。一句话：**归类 → 找缺口 → 对着修 → 隔几天重做验证。**

这里头「找缺口」最值钱，多数人也恰恰卡在这一步。几个例子帮你对照着找：

- 如果归到**看错**，缺口往往是某个固定动作没做——比如从来不圈条件、不看单位，读到一半就开始算。把这个动作补上，下次照做就行。
- 如果归到**不懂**，缺口是某一步的道理没通。一直往回退，退到你不查资料也能独立讲清楚的那一步为止，缺口就在那一步再往上一格。
- 如果归到**记错**，缺口是你记的东西和原样对不上。把你记的和课本摆在一起比一比，差在哪一眼就看出来。
- 如果归到**没时间**，缺口通常是某个环节太慢，一步拖垮了整道题。找出最慢的那一步，单独练它。

针对修完别急着算赢。隔三五天，盖住答案重做这道题：能独立做对，这一轮才算真有结果；要还是错，就把这套流程再跑一遍，缺口未必和上次在同一处。

## 空白可填卡（带诊断流程）

每道题填一行，把上面那套流程落到纸面上。一道题只贴一个粗标签。

| 题号 | 归哪一类 | 缺口在哪（一句话） | 打算怎么修 | 下一轮重做时间 | 重做结果 |
|---|---|---|---|---|---|
|  | 看错 / 不懂 / 记错 / 没时间 |  |  |  | 对 / 仍错 |
|  | 看错 / 不懂 / 记错 / 没时间 |  |  |  | 对 / 仍错 |
|  | 看错 / 不懂 / 记错 / 没时间 |  |  |  | 对 / 仍错 |

「缺口」和「怎么修」各写一句就够，别写成长篇反思，也别再加别的字段。能把这五项稳定填下来、并且真去重做，这张卡就开始起作用了。

## 这张卡到哪儿为止

通用的归类方法、找缺口的思路、四类的通用修法，再加上这套「归类 → 找缺口 → 对着修 → 隔几天重做验证」的自查流程，书里都给全了。你照着就能自己跑一轮，把错题从「抄过答案」推到「粗归因 + 针对性重做」。

# 附录 G：注意力调度优先级表

这是配合[第6章·进程调度](../chapters/06_进程调度.md)使用的两张通用卡。进程调度，就是给任务排先后；CPU，就是电脑里真正干活的那块芯片，一次只能认真跑一个任务。落到学习上，就是先修又常错又好提的，再量一量自己一个番茄钟里被自己打断几次。卡只给框架与空格，哪几项排前面、一个番茄钟划几笔，你照着填、自己判断。

## 卡一：失分项优先级排序表

失分频率＝这类题或这个点最近错得多不多；可提升空间＝这一项现在补，好不好补，补了能不能很快见效。两个一起看：又常错又好提的，排最前面先修；很少错、或者短期很难提的，先往后放。这是排顺序，不是把一天按小时排满。

| 示例：某类计算题最后一步总算错 | 高：最近一周错 4 次 | 好提：先单练验算动作 | 第 1 个 | 做 5 道同类题，每题最后单独验算一次 |
|  | 高 / 中 / 低，或一周错几次 | 好提 / 一般 / 难提 |  |  |
|  | 高 / 中 / 低，或一周错几次 | 好提 / 一般 / 难提 |  |  |
|  | 高 / 中 / 低，或一周错几次 | 好提 / 一般 / 难提 |  |  |

失分项可以从最近的错题里挑。错题怎么归类见[第4章](../chapters/04_调试错题当bug.md)，这里不展开；你只要把最常错的几类填进来排队，行不够就往下加。

## 卡二：番茄钟中断划正字自测表

番茄钟，就是一段你打算专心做一件事的时间，比如 25 分钟。上下文切换，说白了就是注意力每换一次都要重新热机；这段时间里，只要你自己中途看手机、发呆、想别的、换去做另一科，就划一笔正字，结束后数几笔。

| 这个番茄钟在做哪一件事 | 时长 | 中途被自己打断划正字 | 一共几笔 | 下一个番茄钟想改的那一个小动作 |
|---|---|---|---|---|
|  |  |  |  |  |
|  |  |  |  |  |
|  |  |  |  |  |

笔数多，说明注意力一直在切换，力气耗在重新进入状态上了，不是你不努力。先盯着把笔数往下压一点，比如手机放远、桌上一次只摊开一科的东西；这是给自己量一量、自己调，不是给自己打分判好坏。

## 给家长的一段

信号：孩子书桌上同时摊开好几科，一会儿翻这本一会儿翻那本；坐下没几分钟就摸手机、起身找东西、跟你搭话，半天没真正做进去一件事；什么都想补，结果哪科都没补扎实。

一句该说的话：「这一个番茄钟，咱们只做这一科这一件事，中途你想干别的就划一笔，结束咱们看看划了几笔。」

一句别说的话：「这么多要补的，你怎么还有空玩手机？给我全都抓紧！」

家长不用替孩子排该先做哪科，只盯一个动作：孩子有没有一次只摊开一件事、把笔数往下压，而不是几科横跳、坐着空耗。

这套「先修又常错又好提的、一次只做一件事、给自己数中断」的排序和自测，书里都给全了，你照着两张卡就能自己跑一遍。

|  |  |  |  |

## 卡三：收敛判据自查

收敛，就是一个学法反复跑几轮后越来越稳，不再忽对忽错。硬判据写死：连续 N 次独立做对／能讲给别人听懂＝这一项已经稳了，可以停，不必刷到考试为止。没收敛的遗留问题，带进下一版继续重跑。

- [ ] 这一项是不是连续 N 次不查资料独立做对了？
- [ ] 我是不是能把这一步讲给别人听懂，对方也能照着做？
- [ ] 这个毛病是不是连着几周都没再犯？
- [ ] 前三条满足后，我有没有把它从日志里结掉，换下一个卡点上来？

## 给家长的一段

信号：孩子周末复盘只写「这周不够努力、下周加油」，从不记自己具体改了哪一处学法、哪个错还在反复犯；同一类题一周周还在错，复盘本上全是情绪话。

一句该说的话：「别写『要加油』，就写一句：这周你在学法上具体改了哪一处，下周只打算再改哪一件？」

一句别说的话：「复盘有什么用，赶紧多做两套卷子去。」

这张周更日志和收敛判据，书里都给透了，你自己就能每周发一版、自己看出什么时候该停、什么时候该换卡点。

# 附录 I：元认知三问与五阶段自测卡

这是配合[第8章·自举与元认知](../chapters/08_自举与元认知.md)使用的随手卡。元认知，就是回头看自己是怎么学的，再动手调它；自举，就是把你平时用的学习方法本身，也当成一道可以拆、可以调试、可以发新版本的题。每天或每周拿一个具体对象来填，别空问「我学习状态怎么样」。

## 卡一：元认知三问随手卡

三问每次只盯一个对象：这周一个方法、今天一道题、最近一类错。问完以后，只改一个动作，接上[第7章](../chapters/07_版本迭代.md)的「一次只改一个变量」。

| 三问 | 这周拿来问的具体对象（哪个方法·哪道题） | 我的回答 | 据此我打算改的那一个动作 |
|---|---|---|---|
| 我现在在系统的哪一环？（输入 / 编译 / 存储 / 调度 / 执行 / 调试 / 迭代，先按[第1章](../chapters/01_系统总览地图.md)定位） |  |  |  |
| 这个方法这周到底有没有用？（别看感觉，看它有没有让我更能独立做对） |  |  |  |
| 下一步我自己会怎么调？写成一句「下周我把 X 改成 Y」。 |  |  |  |
| 示例：这个背诵方法这周到底有没有用？ | 生物概念复习：每天重读笔记。 | 读得顺，合书讲不出，说明卡在「存了取不出」。 | 下周把每天重读改成合书默写三条，再对照课本补漏。 |

调到什么程度算稳，别凭手感。用收敛判据：连续 N 次独立做对／能讲给别人听懂。这个判据在[第7章](../chapters/07_版本迭代.md)里已经讲过，本卡只拿来验结果。

## 卡二：五阶段自测表

五档描述的是你现在这套学习方法成熟到哪一步，不是给人贴标签。照信号对号入座，定完档就看下一步往哪走。

| 阶段 | 这一档的识别信号（你最近是不是……） | 到下一档的通用进阶方向 |
|---|---|---|
| 模仿 | 你最近是不是照着例题和老师步骤能做，题一换样子就断，还说不清为什么这么做？ | 开始给错题贴[第4章](../chapters/04_调试错题当bug.md)四类标签；讲不清的一步，按[第2章](../chapters/02_递归调用拆到会做.md)拆到能自己做对的最小一步。 |
| 调试 | 你最近是不是已经会把错题分成看错、不懂、记错、没时间，也能说出卡在哪一步？ | 把同一类反复错的题放在一起，不再只补单题，而是重写这一类题的做法，一版只改一个变量。 |
| 重构 | 你最近是不是开始改自己的学法，比如把重读改成自测，把刷整套改成先拆题？ | 每周固定跑一次三问，只改一个动作，并把改动写进更新日志。 |
| 自我编程 | 你最近是不是能自己设计练习、安排节奏、回看效果，不必每一步等人教？ | 把已经跑稳的方法迁移到新学科或新题型，用收敛判据确认它是否站得住。 |
| 进化 | 你最近是不是遇到新问题时，会自动问「我现在卡在哪一环、这办法有没有用、下版改什么」？ | 保持这套循环，定期清掉没用的方法，把有效做法留下来。 |

## 合成小例子

有个学生复习生物一直靠重读笔记。他觉得「我读得很顺」，一合书却讲不出三个关键词之间的关系。照三问走：第一问，卡在存储和提取，东西像是存进去了，考试时取不出来；第二问，重读这周没让他更能独立说出来，方法无效；第三问，下周把「读三遍」改成「合书默写三条关系，再翻书核对」。按五阶段看，他已经不只是模仿，能发现问题，处在「调试」往「重构」走的位置。下一步不是再读更多遍，而是把复习法本身改掉一个动作。

## 给家长的一段

这一环家长能观察到的信号：孩子很努力，却从不回头看自己是怎么学的；你问这周方法有没有用，他只会说「还行」「不知道」，说不出哪道题因此做对了；一套老办法用了很久，错了也不换。

一句该说的话：「这周你换了什么学法没有？挑一个方法，说说它这周到底帮你做对了哪道题。」

一句别说的话：「别瞎琢磨怎么学了，老老实实多做题就行。」

家长不用懂学科，只盯一个动作：孩子有没有偶尔停下来看自己怎么学，并动手调一个动作，而不是一条道闷头走到黑。

这两张卡你自己就能用起来：自己问三问、自己定档、自己看下一步往哪走。真到反复卡在同一档上不去、自己调了几轮还在原地，问分会针对你这一科持续诊断，按周陪你往上调。

# 附录 J：家长运维对照表与家庭运维例会模板

这是配合[第9章·给家长的运维手册](../chapters/09_给家长的运维手册.md)使用的家长卡。家长不负责把题做对，只负责维护孩子能安心运行的环境，再开一个每周 15 分钟以内、不查岗的短会。bug，就是一做就出错的那一处；家长不用会修题，只帮孩子把 bug 说出来。

## 卡一：家长该做／不该做运维对照表

| 环节（这一环在管什么） | 家长该做（具体动作） | 家长别做（具体踩坑） | 别说这句 → 换成这句 |
|---|---|---|---|
| 输入：今天学了什么 | 让孩子说出今天三条重点。 | 只问作业写完没。 | 「今天又学了多少」→「今天你能说出哪三条新东西？」 |
| 编译：学过能不能会用 | 让孩子合上书讲一遍。 | 听他说懂了就算过。 | 「懂了还错？」→「合上书讲一遍，卡在哪句我们标出来。」 |
| 存储：背了能不能取出 | 让孩子先默写，再翻书核对。 | 逼他反复读很多遍。 | 「多读几遍自然会」→「先合上书写三条，写不出再查。」 |
| 调度：注意力放哪 | 帮他清桌面、手机离手，一个时间块只做一件事。 | 一边催一边临时加任务。 | 「别玩手机赶紧学」→「这 25 分钟只做这一件，手机先放这。」 |
| 执行：限时里能不能跑完 | 问最慢的是哪一步，让孩子单独练那一步。 | 盯着他整套卷越写越慌。 | 「怎么又没写完」→「最拖时间的是哪一步，先练那一步。」 |
| 调试：错题怎么处理 | 让孩子先归到看错、不懂、记错、没时间。 | 抄答案、罚抄、骂粗心。 | 「你怎么又错」→「这道题的 bug 在哪一步？先归一类。」 |
| 迭代：这周修了什么 | 每周只看这周改了哪一个动作。 | 翻旧账、把所有问题一次说完。 | 「这周怎么还没进步」→「这周你修好了哪一个 bug？」 |
| 整体心态：家里怎么供电散热 | 保持吃睡、安静桌面、少用分数开火。 | 用焦虑催孩子加时长。 | 「你再不努力就完了」→「今晚先把一个卡点说清，别扩大。」 |

收敛就看一句：连续 N 次独立做对／能讲给别人听懂。家长不用替孩子判题，只听他说得清不清。

用这张表时，家长先别追求一次全改。先挑自己最常说错的一句，把它换掉；再挑家里最容易引爆的一件事，比如手机、错题、考试后说话方式，只改这一处。能少吵一次，孩子多说出一个真实卡点，这张卡就开始起作用了。先稳住一处，再谈下一处。

## 卡二：家庭运维例会轻量模板

每周一次，15 分钟以内，固定时间。孩子主讲，家长只听、记、定自己的后勤；不评判对错，不当场纠题，不临时加码。

| 环节 | 时长 | 谁说 | 说什么（一句话引导） |
|---|---|---|---|
| 开场 | 2 分钟 | 孩子 | 这周哪一科、哪一类题最难受。 |
| 战报 | 4 分钟 | 孩子 | 这周修好了哪个 bug，怎么修的。 |
| 卡点 | 4 分钟 | 孩子 | 还有哪个 bug 反复卡，自己拆不动。 |
| 下周一件事 | 3 分钟 | 孩子定，家长记 | 下周只先改哪一个动作。 |
| 后勤 | 2 分钟 | 家长 | 我这周帮你做哪一件后勤：接送、作息、手机、桌面。 |

| 日期 | 本周修好的一个 bug | 还卡着的一个 bug | 下周只改的那一个 | 家长负责的后勤一件 |
|---|---|---|---|---|
|  |  |  |  |  |
|  |  |  |  |  |
|  |  |  |  |  |

## 给家长的一段

家长能观察到的信号：一聊学习孩子就闭嘴或顶嘴；你一开口就是分数、名次、别人家孩子；家里没有一个固定的、不吵架的聊学习时间。

一句该说的话：「这周咱们开个 15 分钟小会，你说这周修好了哪个 bug，我就听，帮你记，不查你。」

一句别说的话：「考这么点分还有脸开会，我看你就是不上心。」

家长不用懂题，照着对照表换嘴、照着模板每周开一次短会。孩子能把卡点讲出来，才有机会修得动。

这套对照表和例会模板，家长自己就能用起来；想要有人针对孩子某一科一直盯着往下诊断、按周陪你们一起调，那是问分在做的事。

# 附录 K：AI 安全提示词／危险提示词对照表

这是配合[第10章·AI副驾](../chapters/10_AI副驾.md)使用的提示词卡。提示词，就是你发给 AI 的那段话；原则只有一条：AI 帮你练、帮你查、帮你讲，但不替你做对那一步。

同一个 AI，问法决定它是副驾还是代笔。让它直接给答案，练到的是「它会」；让它出变式、追问你、陪你复现卡点，练到的才是「你会」。判据是：不靠 AI、不看答案，自己独立做对／能讲给别人听懂。

## 卡一：危险提示词 vs 安全提示词

| 场景 | 危险问法 | 安全问法 | 为什么这样改 |
|---|---|---|---|
| 一道题不会 | 直接把这道题答案和完整步骤给我。 | 先别给答案，把题拆成几步，逐步问我会不会，卡住时只提示下一小步。 | 危险问法把卡住那步替掉了。 |
| 一类知识没懂 | 帮我把这章总结成笔记，我背就行。 | 我先讲我的理解，你指出哪句错、哪句漏，再追问我两个问题。 | 先自测，再让 AI 查漏。 |
| 想多练 | 这种题帮我做十道。 | 按这个考点出三道变式，我做完你只判对错，先别讲解。 | 题要你做，不是它做。 |
| 错题订正 | 这道我错了，帮我改对。 | 先别告诉我正确答案，陪我复现我原来那一步，问我当时为什么这么想。 | 先找崩在哪一步。 |
| 拿主意 | 我该先补哪科，你帮我定计划。 | 我给你最近错题类型，你帮我归类列出来，先补哪科我自己定。 | AI 做整理，不替你拍板。 |

## 卡二：可直接复制的安全提问范式

拆题陪练时用：

```text
我在【题目/步骤】卡住了。先别直接给答案，请把它拆成几个更小的问题，逐步问我会不会。每次只提示一点，让我自己做出能自己做对的最小一步。
```

出变式题时用：

```text
请按【考点】给我出【3】道同类变式。先只给题，不给答案。我做完后你只判对错；我错了，再提示我卡在哪一步。
```

查漏时用：

```text
我先复述【知识点】。请听完后只指出我讲错、讲漏、讲不清的地方，再追问我两个问题，别替我重写整份笔记。
```

错题归类时用：

```text
我会给你【几道错题的题号和我错的地方】。请只帮我按看错、不懂、记错、没时间归类整理，不替我决定先补哪科。
```

## 卡三：AI 输出怎么用

| AI 给你的东西 | 能直接拿来用吗 | 你该怎么处理 |
|---|---|---|
| 整理、归类、讲解 | 可作草稿 | 自己核对一遍，确认是你听懂了。 |
| 变式题、陪练追问 | 可以练 | 合上 AI，自己独立做对一遍才算过。 |
| 替你拿主意、下诊断 | 不能直接用 | 个人判断自己定；需要长期跟踪的事不要靠一句提示词。 |

AI 不是不能用，是不能越权用。它能帮你练、帮你查，不能替你做对那一步，也不能替你拿主意。

## 给家长的一段

这一环家长能观察到的信号：孩子做作业时把整道题甩给 AI，抄下答案就算完；让 AI 整理好笔记直接背；做得飞快，换道同类还是不会。

一句该说的话：「让 AI 先别给答案，让它一步步问你，你自己做出来；能自己做对那步，才算你学会了。」

一句别说的话：「别用 AI，用 AI 就是偷懒作弊。」

家长不用懂学科，也不用会用 AI，只盯一个动作：孩子是让 AI 替他做完，还是让 AI 陪他把卡住那步自己做出来。

这张卡你自己就能用起来。
