# 附录 A：计算机概念→学习动作→留痕对照总表

**每个计算机概念在这里只占一行，对应一个当天能做的动作和一种留痕方式；查到哪行，今天就去做那一个动作。填不出当天动作的概念，本书不收。**

| 计算机/递归概念 | 学习动作（动词开头·当天可做） | 留痕方式 |
|---|---|---|
| 递归调用（拆解为子问题） | 拆：把今天一道不会的题拆成几个更小问题，写下最先卡住的一步。 | 一张子问题拆解记录；可接[附录C·递归拆解卡](附录C_递归拆解卡.md)。 |
| 能自己做对的最小一步 | 拆：拆到能不查任何资料独立做对的最小单元，写下这个最小单元。 | 一行 能自己做对的最小一步 记录；可接[附录C·递归拆解卡](附录C_递归拆解卡.md)。 |
| 自引用 | 写：把上一轮错题、默写或练习结果改成下一轮输入，写下下一次重做对象。 | 一行「下一轮输入」记录，能翻出来重跑。 |
| 回溯 | 标：从做错处往前找最近一个独立做对的位置，标出从哪里重跑。 | 一个回退点标记，写清题号、步骤或知识点。 |
| 收敛 | 讲：把这块讲给别人听懂，或连续 N 次独立做对，到了就停。 | 一条收敛记录：连续 N 次独立做对／能讲给别人听懂。 |
| 编译 | 讲：合上书把本节知识讲成考场可执行步骤，再标出卡住的句子。 | 一段口述提纲和卡点标记；可接[附录D·编译通过判据与报错对照表](附录D_编译通过判据与报错对照表.md)。 |
| 内存/缓存 | 默：合上书默写本节核心概念，标出默不出来或写错的地方。 | 一段默写痕迹和错漏标记；可接[附录F·提取练习与间隔重复设计卡](附录F_提取练习与间隔重复设计卡.md)。 |
| 进程调度 | 标：一个番茄钟只做一件事，被自己中断一次就划一笔正字。 | 一组正字记录；可接[附录G·注意力调度优先级表](附录G_注意力调度优先级表.md)。 |
| 调试 | 标：给今天一道错题贴「看错/不懂/记错/没时间」四类粗标签之一。 | 一个粗标签和下一轮重做时间；可接[附录E·错题调试归因四分法卡](附录E_错题调试归因四分法卡.md)。 |
| 版本迭代 | 写：用四行记录本周改了什么、修了哪个 bug、遗留什么、下次试什么。 | 一条每周更新日志；可接[附录H·每周学习更新日志模板](附录H_每周学习更新日志模板.md)。 |
| 自举/元认知 | 写：用一句话回答本周哪个方法有效、哪个方法无效、下一次观察什么。 | 一组三问自检记录；可接[附录I·元认知三问与五阶段自测卡](附录I_元认知三问与五阶段自测卡.md)。 |

所以读这张表的方式很简单：查到哪一行，今天就照那一格的动作去做，做完按那一格的方式留下一条痕迹，别的概念先放着。后面各章遇到任何一个计算机比喻，都先回到这张表，找它对应的当天动作和留痕，找不到的就当它没出现过。

本表把「概念→当天动作→留痕」整套写在这里，照着做就能自己跑起来：哪一科该先做哪一行、痕迹里怎么读出根因、接下来往哪个方向进阶，都在书里讲透。通用框架在表里，针对某个孩子某一科的个性化诊断找问分。平时学习方法跑通以后，再交棒给[新疆高中三年升学规划书](../../新疆高中三年升学规划书/README.md)和[新疆家长志愿风险识别手册](../../新疆家长志愿风险识别手册/README.md)。
