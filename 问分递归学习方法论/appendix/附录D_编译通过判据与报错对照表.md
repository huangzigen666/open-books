# 附录 D：编译通过判据清单与报错对照表

用途：配合[第3章·编译](../chapters/03_编译学过不等于会用.md)使用。学过一个知识点后，对照下面判据逐条勾；勾不满＝还没编译通过，别急着进下一题。

## ①「编译通过」硬判据清单

- [ ] 合上书和笔记，能把这个知识点或方法用自己的话复述一遍。
- [ ] 能讲给一个没学过的人听懂，对方能照着你的说法做对。
- [ ] 给一道改了数字或换了情境的变式题，能不查任何资料独立做对；这里对齐 能自己做对的最小一步 口径：能不查任何资料独立做对的最小单元。
- [ ] 能说出这个方法什么时候用，什么时候不能直接用。
- [ ] 能指出自己上一次在这一步崩在哪里，而不是只说「当时没想到」。

勾不满＝处于「源码看懂了、可执行文件还没生成」状态。回到第3章重新编译，别把「看懂」当成「会用」。

## ②「一看就会一做就废」报错对照表

| 报错现象 | 可能卡在哪（通用方向） | 当天动作 |
|---|---|---|
| 看例题秒懂，自己动笔卡在第一步 | 题没有拆到 能自己做对的最小一步 | 把这道题拆到能不查任何资料独立做对的最小单元；可接[第2章·递归调用](../chapters/02_递归调用拆到会做.md)和[附录C·递归拆解卡](附录C_递归拆解卡.md)。 |
| 跟着答案能写，合上答案就断 | 只读懂答案，没有生成可执行步骤 | 合书复述步骤，再做一道变式；对照上面的硬判据勾选。 |
| 平时觉得会，考场上想不起来 | 提取这一环不稳定 | 用自测代替重读；可接[第5章·内存与缓存](../chapters/05_内存与缓存.md)和[附录F·提取练习与间隔重复设计卡](附录F_提取练习与间隔重复设计卡.md)。 |
| 步骤大体会，数字、符号、条件老错 | 更像执行时的看错或记错 | 先按[第4章·调试](../chapters/04_调试错题当bug.md)和[附录E·错题调试归因四分法卡](附录E_错题调试归因四分法卡.md)归一个粗标签，再重做。 |
| 会做，但限时里总跑不完 | 熟练度和时间压力没有分开看 | 把这类题先拆到 能自己做对的最小一步 提速；不要用刷整套卷假装提速。 |

这张表帮你把「一做就废」定位到一个粗方向，并启动一个当天动作。

这套判据和报错对照，你自己就能照着跑。
