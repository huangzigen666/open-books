# 附录 C：递归拆解卡

这是配合[第2章·递归调用](../chapters/02_递归调用拆到会做.md)使用的通用拆解卡。遇到不会的题，不是先抄答案，而是往下拆到你自己能做对的那一步，再往回拼；卡里只给框架与空格，拆出什么、拆到哪一层，由你照着填、自己判断。

## 卡一：一题拆到底主卡

先只挑一道今天真不会的题。每一层只写一句话，行不够就自己接着往下加。

| 位置 | 你要填什么 |
|---|---|
| 整道题（抄下你今天挑的这一道） | ______ |
| 第一层·这道题大致要做哪几步 | ______ |
| 我卡在哪一步（这一步我下不了笔/不确定） | ______ |
| 下一层 1·这一步又卡在哪个更小的、我已经会一点的东西上 | ______ |
| 下一层 2·继续往下拆 | ______ |
| 下一层 3·继续往下拆 | ______ |
| 一直拆到的能自己做对的最小一步（不查任何资料、不看答案，我自己能独立做对的那一步） | ______ |
| 往回拼（从这个最小一步往上，一层层接回去，整道题能不能走通） | ______ |
| 这道题真正卡住的最小一步是 | ______ |

拆几层不固定。拆到你确实能独立做对那一步为止；如果还要猜、还要看答案，就继续往下拆。

## 卡二：触底硬判据自查

判断有没有拆到底，不看感觉，看这三项。

| 这一步现在是什么状态 | 勾选 |
|---|---|
| 不查任何资料、不看答案，这一步我能独立做对（独立写出）。 | - [ ] 到底了 |
| 还要瞄一眼答案才敢写。 | - [ ] 没到底，回卡一继续拆 |
| 能写出结果，但讲不出这一步为什么成立。 | - [ ] 没到底，回卡一继续拆 |

拆到底不是拆到「看着最简单」，是拆到「你自己确实能独立做对」的那一层。每个人的那一层可能不同，这正常。

## 卡三：把卡点带进下一轮

拆出来的最小卡点，要喂进下一轮练习。错题怎么进入下一轮，见[第4章](../chapters/04_调试错题当bug.md)；练到什么时候停，见[第7章](../chapters/07_版本迭代.md)，连续独立做对几次、能讲给别人听懂，就算这一小步练通。

我这道题真正卡住的最小一步：______，下次先练它。

## 给家长的一段

信号：孩子一遇到大题或难题就卡住不动，半天不下笔；要么直接抄答案，要么一句「不会做」就整道跳过；同类题抄过一次，下次还是不会。

一句该说的话：「先别抄答案，这道题里哪一小步是你已经会的？从你会的那步，一点点往下拆看看。」

一句别说的话：「这么简单都不会？多刷几道不就会了。」

家长不用会做这道题，只盯一个动作：孩子有没有把题往下拆、找到自己会的那一步，而不是整道放弃或直接抄答案。

这套往下拆到能自己做对的最小一步、再往回拼的卡，书里都给全了，你照着就能自己跑一遍。
