## 2. 本附录定位（一句话）

这是一张**极简早筛索引卡**：把九条特殊招生路线的名字、一个官方查询入口、一句话「大概适合谁」摆出来，让家长知道「有这么几条路、去哪查官方信息」，**但不替家庭判断走不走、来不来得及、怎么准备**——那些是第 8 章随第二阶段发布的完整版的事。

## 3. 必含结构（清单页体例：本附录结论加粗 + 分节 + 一张九行表，不套八段）

1. **开篇版本声明（必写、显眼）**：用一段话写明「**本附录为 MVP 极简版，只做路线名 + 官方查询入口 + 一句话适配方向的早筛索引；完整分流卡（含触发信号、提前布局年限、标准准备动作、何时必须人工复核）随第 8 章在第二阶段发布。**」
2. **本附录结论**（加粗一句，格式 `本附录结论：**……**`，冒号在加粗外）：例如——特殊招生路线只先做「知道有这条路、去哪查官方」，**要不要走、来不来得及、怎么准备一律此处建议人工复核**，别照这张卡自己拍板。
3. **怎么用这张卡（2-4 句）**：先用第 1 章时间轴定位自己站在哪一段；这张卡只帮你认门、找官方入口；任何一条想真正评估，去找一对一（指向附录 M）。
4. **九条路线早筛索引表**（一张 Markdown 表，**九行**，列固定为：`路线 | 官方查询入口（方向，非具体网址） | 一句话适配方向（示意） | 早筛动作`）：
   - 九条路线：**强基计划、学科竞赛、艺术类、体育类、军警公安类、医学类、公费师范（含优师专项）、南疆/国家专项计划、综合评价**。
   - 「官方查询入口」列只写**入口方向**（如「新疆教育考试院官网及目标高校本科招生网当年简章（需复核：以当年官方发布为准）」「阳光高考平台特殊类型招生专栏」），**不要编造任何具体网址、文号、报名时间**。
   - 「一句话适配方向」列只写**示意性方向**（如「适合学科特别拔尖、愿意走基础学科的孩子」），**禁止写门槛数字、年限、分数、奖项级别**。
   - 「早筛动作」列**统一只写一句**：`查官方入口 +（此处建议人工复核）`，不得写「提前 X 年布局」「满足 X 信号就启动」「先去集训/报名/政审」等任何可套用动作。
5. **共同提醒（极简，3-5 条短句）**：每条路线都有独立资格/流程/义务，逐年逐校变动；本卡不含达标标准；想评估具体某条路线请走一对一。
6. **需复核与官方优先声明**：政策、时间、资格门槛、报名窗口一律以新疆教育考试院及目标高校当年官方发布为准，行内统一标「（需复核：以当年官方发布为准）」。
7. **交棒第一本**：一句话——「到了志愿季逐条核验某条专项/特殊类型路线的资格、章程、政审、体检达标标准，不在本书范围」，链接到第一本[《别把孩子的分数浪费在志愿表里》第 8 章·专项场景不能套通用模板](../../新疆家长志愿风险识别手册/chapters/08_专项场景不能套通用模板.md)。

## 4. 红线（违反即不通过，本条目写死）

- **MVP 必须极简**：**严禁写「提前布局年限」「触发信号」「具体准备动作清单」「集训/报名/政审/材料怎么准备」等任何可直接套用的细节**——这些属于第 8 章完整版，第二阶段才发布。若你写出的内容能让家庭照着自己排出某条路线的准备时间表，就是写过头了，收回到「路线名 + 官方入口 + 一句话方向」三件套。
- **不替家庭判断走不走**：不给「你家该不该走强基/竞赛/某专项」的结论或倾向；不做「孩子是 XX 型→建议走 XX 路线」的映射。
- **每条都要带「（此处建议人工复核）」**：九行的「早筛动作」列每一行都必须出现「（此处建议人工复核）」字样，一条不漏。
- **不编造官方数据**：不杜撰具体网址、文号、报名/确认时间、分数线、户籍学籍年限、奖项级别、体检/政审达标标准；不确定一律标「（需复核：以当年官方发布为准）」或只写入口方向。
- **不写玄学/不承诺结果**：禁用「保上名校」「低分捷径」「内部名额」「百分百」等表达；不得把任一路线描述成「稳/保底/捷径」。
- **志愿季逐条核验一律交棒第一本**，本卡不展开资格/章程/政审/体检/体测的达标细则。
- **案例脱敏**：不出现可识别学生/家庭/学校/获奖信息。
