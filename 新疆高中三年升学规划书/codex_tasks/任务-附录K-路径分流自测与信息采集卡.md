## 2. 本附录定位（一句话）

这是一张**领资钩子**：用极少的几个问题，帮家庭**粗判「现在大概该重点关注哪 1-2 个里程碑」**，然后把人导向一对一——它只回答「你大概要先盯哪个方向」，**不回答「你家该走哪条路、按什么节奏、准备什么」**。同页附一张家庭信息采集卡，收集后续咨询要用的基础信息。

## 3. 必含结构（清单/表单页体例，不套八段）

1. **本附录结论**（加粗一句，格式 `本附录结论：**……**`，冒号在加粗外）：这张卡只帮你「粗判该先重点关注哪 1-2 个里程碑」，**具体走哪条路、怎么排节奏，一律此处建议人工复核**，别拿这张卡当三年方案。
2. **怎么用这张卡（2-3 句）**：先用第 1 章时间轴定位站在哪一段；自测只做方向粗判；想真正评估，带着信息采集卡走附录 M 找一对一。
3. **路径分流自测（几个问题 → 只输出方向粗标）**：设计 5-8 个简单是/否或选择题（如孩子年级、是否有明显学科特长/艺体特长、是否考虑过特殊路线、户籍学籍是否可能涉及专项、家庭是否已开始系统了解升学），**判定结果只能落到「方向粗标 + 转人工」两段**：
   - 输出形式统一为：「你现在大概要重点关注的里程碑是【选科定科 / 方向倒推 / 路线分流 / 资格早筛 / 院校梯度初定 中的 1-2 个】，建议结合孩子具体情况人工复核。（此处建议人工复核）」
   - **严禁输出**：具体特殊路线名称推荐（如「建议走强基/竞赛」）、任何时间节点/年限、任何准备动作清单、任何「你属于 X 型→该怎么做」的路线方案。
4. **家庭信息采集卡**（一张可填空白表）：采集后续咨询要用的基础信息——孩子年级、成绩趋势（近几次主要考试/单科）、选科倾向或已定组合、目标专业大方向、身体条件与受限项、户籍/学籍年限情况、家庭预算与底线、孩子本人想法、最想解决的一个问题。**只设字段、留空白，不替家庭填、不据此给结论。**
5. **需复核与官方优先声明**：凡涉及选科要求、专项资格、户籍学籍年限等会逐年变的，标「（需复核：以当年官方发布为准）」。
6. **转人工与交棒**：结尾明确「自测只到方向粗判为止，要不要走某条路、来不来得及、怎么准备，必须一对一」，链接[附录 M·服务边界与咨询入口](附录M_服务边界与咨询入口.md)；出分后志愿季相关交棒第一本[《别把孩子的分数浪费在志愿表里》](../../新疆家长志愿风险识别手册/README.md)。

## 4. 红线（违反即不通过，本条目写死）

- **输出边界写死**：自测卡的判定结果分支**不得超过「方向粗标 + 转人工」两段**，只输出「你大概要重点关注哪 1-2 个里程碑 +（此处建议人工复核）」。**严禁输出具体路线名称、具体时间节点/年限、具体准备动作清单、六/九条路径各自的完整三年规划或节奏。** 强自检：若家庭照这张卡就能排出自己的路线选择与准备时间表，就是写过头了，必须收回到「方向粗标 + 转人工」。
- **不替家庭判断**：不做「孩子是 XX 型 → 建议走 XX 路线」的映射；不给「该不该走某路线/该选哪科/要不要复读」的结论或倾向。
- **信息采集卡只设字段不下结论**：留空白给家庭填，不替填、不据此输出任何个性化判断。
- **不编造官方数据**：不杜撰具体分数线、位次、选科要求、资格门槛、年限、报名时间；不确定标「（需复核：以当年官方发布为准）」。
- **不写玄学/不承诺结果**：禁用「保上名校」「内部名额」「低分捷径」「百分百」等表达。
- **不泄露产品壁垒**：不写问路工作台的字段结构、评分算法、复核规则。
- **案例脱敏**：不出现可识别学生/家庭/学校/获奖信息。
